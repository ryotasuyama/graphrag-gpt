### ループ構造方針
- **Wall.Fore × Deck の DL 系**
  - `DL00〜DL05` をキーにして抽出
  - `Wall.Fore.DLxx.*` は **Deck.*.DLxx.F / FP のみ**を接続候補とする
  - 本スクリプトでは例えば
    - `profile62 = HK.Casing.Wall.Fore.DL01.CDP`
    - `profile61 = HK.Casing.Deck.D.DL01.FP`
    のような対応が正例

- **Wall.Aft × Deck の DL 系**
  - `DL00〜DL05` をキーにして抽出
  - `Wall.Aft.DLxx.*` は **Deck.*.DLxx.A / AP のみ**を接続候補とする
  - 代表例:
    - `profile75 = HK.Casing.Wall.Aft.DL01.CDP`
    - `profile23 = HK.Casing.Deck.D.DL01.AP`

- **Wall.Side × Deck の FR 系**
  - `FR07〜FR15` を軸に、
  - 同一 FR の Deck FR profile、またはその FR 近傍の Deck plate / Deck DL profile を候補化
  - Side Wall は Fore/Aft 制約なし
  - ただし **Deck 側 profile の存在確認**は必須
  - さらに今回の再分析では、`FR14F415` のような特殊横通材に対しては、成功済み実績がない限り標準 `1501` を安易に使わない

- **失敗候補修正ルール**
  - `Side.*.BCP/OAP/ABP` と `Deck.*.FRxxF***` のような特殊 FR 補助材の組合せは、まず
    1. 参考スクリプトで同型の成功例があるか
    2. `FL × FL` の標準面種で整合するか
    3. 実際に交点・取付平面が成立するか
    を確認する
  - 不成立なら
    - 同 FR の通常 Deck profile (`FRxxP`, `DL00.F`, `DLxx.FP`) に差し替える
    - もしくは `Plate PLS × Profile FL` の `1505` に切り替える

- **BracketType のループ選択**
  - `Profile FL × Profile FL` は `1501`
  - `Plate PLS × Profile FL` は `1505`
  - Side 壁用で参考スクリプトに `Sf1EndElements` があるものは必ず同様に適用

- **実装上の反復単位**
  - 1つの `wall_profile` ごとに
    1. 対応 `deck_profile` 候補抽出
    2. Fore/Aft 整合チェック
    3. FR/DL キー一致または近傍ルール確認
    4. `Surfaces1/Surfaces2` の面種判定
    5. `BracketType` 決定
    6. `BaseElement` を wall 側 profile に固定
    7. 必要なら `Sf1EndElements` 付加
    8. 特殊 FR 補助材なら候補除外または plate 案へ置換

### パラメータ依存関係
- **BracketType**
  - `1501`
    - 2面ブラケット
    - `Profile FL × Profile FL` の標準形
    - Wall/Deck の通常 profile 同士で採用
  - `1505`
    - 2面ブラケット
    - `Plate PLS × Profile FL` の標準形
    - Side Wall と Deck plate/solid の接続で採用
    - `BracketParams=["200"]` の付与パターンを踏襲

- **今回の失敗から得られる追加判断**
  - `1003` の特殊 Deck FR 補助材 (`FR14F415` 等) は、名前上 profile であっても、標準 `1501 / FL×FL` で安全に作れるとは限らない
  - 参考スクリプトで成功している FR 1003 の主な組合せは
    - `Deck.FR09P ↔ Side.FR09.CDP`
    - `Deck.FR13P ↔ Side.FR13.CDP`
    - `Deck profile の WF/WB ↔ FR profile の FL`
  - よって `FR14F415` のような部材は、成功済み同型パターンがない場合は **候補から落とすか、plate 基準に変更**するのが安全

- **Thickness**
  - `ProfileType 1002` 同士の標準 DL 接続 → **8**
  - Side wall profile と plate 接続 → **10**
  - `ProfileType 1201` を含む DL02 系 → **12**
  - Deck FR profile 接続 → **8 or 9**
  - 今回失敗した `FR14F415` 接続は、厚さよりも**面定義や成立平面の問題**の可能性が高く、厚さ変更だけでは解消しないと判断

- **Sf1 / Sf2 Dimension**
  - `1501` の標準 profile-profile:
    - `Sf1DimensionType=1531`
    - `Sf2DimensionType=1531`
    - `Sf1DimensonParams=["200","15"]`
    - `Sf2DimensonParams=["200","15"]`
  - FR girder 系では
    - `["250","15"]` や `["300","15"]`
  - `1505` の plate-profile:
    - `Sf1DimensionType=1541`
    - `Sf1DimensonParams=["0","100"]` または `["0","80"]`
    - `Sf2DimensionType=1531`
    - `Sf2DimensonParams=["200","15"]` または `["300","15"]`

- **FlangeType / FlangeParams**
  - 1201 系・WF/WB 接続ブラケットで採用
  - `DL02` 周辺では
    - `FlangeType=262`
    - `FlangeParams=["100","30","29.999999999999996","30","30","1"]`
    - または `["75","30","29.999999999999996","30","50","1"]`
  - 失敗した FR14F415 接続はこの特殊 flange パターンにも属しておらず、参考準拠性が低い

- **Sf1EndElements / Sf2EndElements**
  - **Side Wall の Plate PLS × Profile FL** のとき重要
  - 参考スクリプトで成功した Side 用 plate ブラケットは、必ず `Sf1EndElements` を持つ
  - 代替候補 #41, #42 はこの成功型に寄せて構成するのが適切
  - `1501` の通常 Wall/Deck profile 同士では不要

- **BaseElement**
  - 原則 **Wall 側 profile**
  - Side-Deck FR 接続でも Side 側を基準にする
  - 今回の失敗候補でも `BaseElement=profile21[0]` 自体は不自然ではないが、接続先選定が不適切だったと見る

### 注意事項
- **Fore/Aft 方向整合ルールは必須**
  - `Wall.Fore.DLxx.*` は `Deck.*.DLxx.F / FP` のみ接続可
  - `Wall.Aft.DLxx.*` は `Deck.*.DLxx.A / AP` のみ接続可
  - 今回もこのルールは再確認済み
  - 方向違反候補は候補表に残していても、最終実装対象から除外する

- **片側 Deck のみ存在する DL は反対壁から除外**
  - `.F/.FP` しかない DL は Aft 側と結ばない
  - `.A/.AP` しかない DL は Fore 側と結ばない

- **Side Wall（FR 系）は方向制約なし**
  - ただし接続対象となる Deck profile / Deck plate の**実在確認**は必須
  - さらに **FR 名が一致しない候補は優先度を落とす**
  - 特に `FR14F415` のような特殊部材は、FR が一致してもそのまま安全とは限らない

- **今回の失敗候補の扱い**
  - `profile21[0]+",FL"` × `profile49[0]+",FL"` は **除外**
  - 理由:
    1. 実行時 COM 例外が発生済み
    2. 参考スクリプトの成功型と一致しない
    3. `Deck.A.FR14F415` が標準ブラケット面として扱えない可能性が高い
  - よって「失敗候補のみ修正・除外」という要件に従い、**この1件のみ除外**した

- **代替候補の優先順位**
  1. 成功済み bracket 1〜19 と同型のもの
  2. `Side.CDP ↔ Deck.FRxxP` の同FR組合せ
  3. `Side profile ↔ Deck.DLxx.F/FP` の実在 profile 組合せ
  4. `Plate PLS × Profile FL` の `1505` 構成
  5. `FRxxF***` の特殊補助材は最後に検討

- **面の種類を混同しない**
  - `1501` は基本 `profile + ",FL"` 同士
  - `1505` は `["PLS", ... , solidX]` と `profile + ",FL"`
  - `WF`, `WB`, `PLS` の混在は、参考スクリプトで成功実績がある場合のみ採用

- **DL02 の 1201 断面は特殊**
  - `profile18`, `profile19`, `profile59`, `profile88`, `profile97`, `profile120` などは flange/body を持つ
  - bracket 生成では `[0]` / `[1]` の使い分けが必要
  - PLS 参照は flange/body 側に揃える

- **候補数要件の維持**
  - 前回 40 件の枠組みを維持しつつ、
  - 失敗した #20 のみ除外し、
  - #40〜#42 の代替候補を補充したため、
  - **30個以上**という要件は維持される

以上より、この元スクリプトに対しては、**成功済み 1〜19 を維持し、失敗した `Side.FR14.BCP ↔ Deck.A.FR14F415` を除外、代わりに FR14 の実在 Deck profile または Plate 基準候補へ置換する方針**が最適です。
