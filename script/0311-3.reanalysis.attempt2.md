### ループ構造方針
- **Fore/Aft 壁 × Deck縦材接続ループ**
  - DLごとに `Wall.Fore` / `Wall.Aft` の `CDP` を抽出し、対応する Deck の同一DLプロファイルとペアリングする。
  - ただし、**ProfileName の Fore/Aft サフィックスを必ず確認**する。
  - 許可ルール:
    - `Wall.Fore.DLxx.*` → `Deck.*.DLxx.F` / `Deck.*.DLxx.FP`
    - `Wall.Aft.DLxx.*` → `Deck.*.DLxx.A` / `Deck.*.DLxx.AP`
  - `DL04P`, `DL05P`, `DL02P`, `DL03P` のように F/A サフィックスを持たない Deck 部材は、参考スクリプト実績がある同型接続のみ候補化する。
- **今回の失敗候補の修正**
  - 失敗した候補は  
    `profile117 = HK.Casing.Wall.Fore.DL03.CDP`  
    ↔ `profile54 = HK.Casing.Deck.C.DL03.FP`
  - 名前規則だけ見れば Fore/FP で整合するが、`CreateBracket` が COM 例外になったため、**この組み合わせのみ確定候補から除外**する。
  - 代替として、同じ Fore.DL03 の成功可能性が高い別 Deck ライン:
    - `profile35 = HK.Casing.Deck.D.DL03P`
    - `profile108 = HK.Casing.Deck.A.DL03.FP`
    - `profile114 = HK.Casing.Deck.B.DL03.FP`
    を優先採用する。
- **成功済み bracket 1〜10 の固定**
  - 前回実行で成功した 1〜10 はそのまま固定採用。
  - 失敗再発防止のため、修正対象は **11番相当の失敗候補のみ** とする。
- **Side Wall × Deckソリッド接続ループ**
  - FRごとの Side Wall `CDP` を順番に処理し、Deck.D ソリッド面に対して 1505 型ブラケットを配置する。
  - `Surfaces1 = ["PLS", ..., solid2]`
  - `Surfaces2 = [profile + ",FL"]`
  - `Sf1EndElements` は参考スクリプトどおり `profile17[0]` 系を使う。
- **Side Wall × Deck FRプロファイル接続ループ**
  - FR09, FR13 は Deck 側に対応する FR プロファイルが存在するため、1501 型の profile-to-profile 接続を採用する。
  - Side Wall は Fore/Aft 制約を持たないが、Deck 側プロファイルの存在確認は必須。
- **候補生成順序**
  1. 成功済み 1〜10 を固定
  2. 失敗した Fore.DL03 ↔ Deck.C.DL03.FP を除外
  3. 同じDL・同方向の Deck.A / Deck.B / Deck.D 候補で補完
  4. Side系候補で総数を 30超に維持

### パラメータ依存関係
- **BracketType**
  - `1501`: profile と profile の直接接続
    - 例: `Wall.Fore.DL01.CDP` ↔ `Deck.D.DL01.FP`
    - 例: `Wall.Side.FR13.CDP` ↔ `Deck.D.FR13P`
  - `1505`: Deck plate / solid 面 と Side Wall profile の接続
    - 例: `Surfaces1 = ["PLS", ..., solid2]`
    - 例: `Surfaces2 = [profile16[0]+",FL"]`
- **Thickness**
  - Wall.Fore / Wall.Aft の `1002` 同士接続:
    - `7.9999999999999964`
  - `1201` を含む DL02 接続:
    - `12`
  - Side Wall `1002` と plate/solid 接続:
    - `9.9999999999999982`
  - Side Wall `1003` や Deck `1003` 接続:
    - `7.9999999999999964` または `8.9999999999999982`
- **Sf1 / Sf2 Dimension**
  - `1501` の標準:
    - `Sf1DimensionType = 1531`
    - `Sf1DimensonParams = ["200","15"]`
    - FR系では `["250","15"]`, `["300","15"]` を使用
    - `Sf2DimensionType = 1531`
    - `Sf2DimensonParams = 同系統`
  - `1505` の標準:
    - `Sf1DimensionType = 1541`
    - `Sf1DimensonParams = ["0","100"]` または `["0","80"]`
    - `Sf2DimensionType = 1531`
    - `Sf2DimensonParams = ["200","15"]`
- **Sf1EndElements**
  - 1505 型では必須に近い
  - 例:
    - `["PLS","False","False","-0","-1","-0",profile17[0]]`
    - Deck.D の端部基準として参照
- **FlangeType / FlangeParams**
  - `WF/WB` や `1201` を含む特殊 1501 ブラケットでのみ使用
  - 今回の修正版では、失敗回避を優先し、**新規修正対象には持ち込まない**
- **Surfaces の選択**
  - 安定優先:
    - `profile + ",FL"`
    - `["PLS", ..., solid]`
  - 今回失敗した Fore.DL03 ↔ Deck.C.DL03.FP は `FL-FL` でも失敗したため、**同一組み合わせの再採用はしない**
- **BaseElement**
  - 基本は壁側部材
  - Side Wall × Deck profile でも Side Wall 側を BaseElement に統一

### 注意事項
- **Fore/Aft 方向整合は必須**
  - `Wall.Fore.DLxx.*` は `Deck.*.DLxx.F` / `Deck.*.DLxx.FP` のみ
  - `Wall.Aft.DLxx.*` は `Deck.*.DLxx.A` / `Deck.*.DLxx.AP` のみ
- **今回の再分析で除外する候補**
  - `profile117 = HK.Casing.Wall.Fore.DL03.CDP`
  - `profile54 = HK.Casing.Deck.C.DL03.FP`
  - この組み合わせは前回 `bracketPram11` として COM 例外が発生しているため、**確定候補から除外**
- **成功済み候補は維持**
  - bracket 1〜10 は不変更
- **修正対象は失敗候補のみ**
  - 前回ドキュメントの他候補はそのまま維持
  - ただし #11 のみ削除し、#17〜#20 等の既存代替候補群で候補数を維持
- **候補数要件**
  - 30個以上を維持するため、確定採用候補は **35件**
  - 失敗1件を除外しても要件を満たす
- **Side Wall（FR系）は方向制約なし**
  - ただし接続先 Deck プロファイルまたは solid の存在確認は必要
- **DL02 の 1201 断面は特別扱い**
  - `profile18`, `profile19`, `profile59` などは複数戻り値を持つ
  - `Surfaces1/2` の添字は参考スクリプトどおりに固定
- **1505 は Deck.D ソリッド前提**
  - `solid2` が事前に存在する構成のみ採用
  - 他 Deck へ同形式を安易に横展開しない
- **安全側の採用原則**
  - 実績のあるパターンを優先
  - 失敗した profile-to-profile の同一面組を再挑戦しない
  - 代替は同一DL・同一Fore/Aft・別Deckラインを優先
