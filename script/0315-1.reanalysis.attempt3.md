### ループ構造方針
- **Wall.Fore × Deck の DL 系**
  - `DL00〜DL05` をキーに抽出
  - `Wall.Fore.DLxx.*` は **Deck.*.DLxx.F / FP** にのみ接続
  - 例
    - `profile62 = HK.Casing.Wall.Fore.DL01.CDP`
    - `profile61 = HK.Casing.Deck.D.DL01.FP`
  - Fore 側に `.F/.FP` が存在しない DL は候補除外

- **Wall.Aft × Deck の DL 系**
  - `DL00〜DL05` をキーに抽出
  - `Wall.Aft.DLxx.*` は **Deck.*.DLxx.A / AP** にのみ接続
  - 例
    - `profile75 = HK.Casing.Wall.Aft.DL01.CDP`
    - `profile23 = HK.Casing.Deck.D.DL01.AP`
  - Aft 側に `.A/.AP` が存在しない DL は候補除外

- **Fore/Aft 方向整合の再確認結果**
  - `Wall.Fore.DL01/03/04/05` に対しては `Deck.*.DLxx.F / FP` のみ採用
  - `Wall.Aft.DL01/03/04/05` に対しては `Deck.*.DLxx.A / AP` のみ採用
  - 今回の修正で除外対象となった Side 候補は Side 系なので方向違反ではないが、**Deck 側 DL03.FP との FL×FL 直結が不安定**と判断

- **Wall.Side × Deck の FR 系**
  - Side Wall は Fore/Aft 制約なし
  - ただし **Deck 側実在確認**は必須
  - 優先順は次の通り
    1. `Side.CDP ↔ Deck.FRxxP`
    2. `Side.* ↔ Deck.DLxx.F / FP`
    3. `Deck plate(PLS) × Side profile(FL)` の `1505`
  - `FRxxF***` のような特殊補助材は、参考スクリプトの成功型と一致しない限り**直接接続候補から除外**

- **今回の失敗候補修正ルール**
  - `profile111[0] = HK.Casing.Wall.Side.FR11.CDP`
  - `profile108[0] = HK.Casing.Deck.A.DL03.FP`
  - この `1501 / FL×FL` は実行失敗済みのため除外
  - さらに、同系統の `Side.FR11.CDP ↔ Deck.C.DL03.FP` も安全側で除外
  - FR11 系の差替えは以下へ寄せる
    - `Side.FR11.OAP ↔ Deck.A.DL01.FP`
    - `Side.FR11.ABP ↔ Deck.A.DL01.FP`
    - `Deck.A plate(P﻿LS) ↔ Side.FR11.*`
  - つまり、**FR11 では DL03.FP 直結を捨て、DL01.FP または plate 基準へ寄せる**

- **BracketType のループ選択**
  - `Profile FL × Profile FL` は `1501`
  - `Plate PLS × Profile FL` は `1505`
  - Side 用 `1505` では `Sf1EndElements` を必ず設定する

- **BaseElement の固定方針**
  - 原則として **Wall 側 profile**
  - Fore/Aft/Side のいずれでも同様
  - plate を使う場合も `BaseElement` は side/wall 側 profile のままとする

### パラメータ依存関係
- **BracketType**
  - `1501`
    - 標準 2面ブラケット
    - `FL × FL` の profile-profile 接続
    - DL 系 wall-deck、FR 系 side-deck の通常構成
  - `1505`
    - `PLS × FL` の plate-profile 接続
    - Side Wall 系で特に安定
    - `BracketParams=["200"]` を付与する参考スクリプト型を踏襲すべき構成

- **今回の失敗からの追加判断**
  - `profile108 = HK.Casing.Deck.A.DL03.FP` 自体は Fore 系 Deck profile として実在する
  - しかし `profile111 = HK.Casing.Wall.Side.FR11.CDP` との **1501 / FL×FL** は、今回実行で成立しなかった
  - したがって、部材名の整合だけでなく、
    - 交差位置
    - 面の向き
    - ブラケット生成平面
    - 実際の取付可能な稜線
    を満たす必要がある
  - 今回は `FR11.CDP ↔ Deck.A.DL03.FP` を **実行失敗済みの不適合組合せ**として扱う

- **特殊補助材の扱い**
  - `profile49 = HK.Casing.Deck.A.FR14F415`
  - `profile27 = HK.Casing.Deck.C.FR14F415`
  - `profile70 = HK.Casing.Deck.B.FR14F415`
  - これらは通常の `FRxxP` ではなく特殊補助材
  - 直接ブラケット接続先としては不安定
  - それらに従属して生成された `DLxx.FP` も、Side からの直結では個別に成立性検証が必要

- **Thickness**
  - `1002` 同士の標準 DL 接続 → **8**
  - Side と deck plate の `1505` → **10**
  - `1201` を含む DL02 系 → **12**
  - Deck FR 主部材との接続 → **8 または 9**
  - 今回の失敗原因は厚さではなく、**接続面と交差条件の不適合**

- **Sf1 / Sf2 Dimension**
  - `1501` の標準:
    - `Sf1DimensionType=1531`
    - `Sf2DimensionType=1531`
    - `Sf1DimensonParams=["200","15"]`
    - `Sf2DimensonParams=["200","15"]`
  - FR 主部材周りでは
    - `["250","15"]`
    - `["300","15"]`
    の拡大型もありうる
  - `1505` の plate-profile:
    - `Sf1DimensionType=1541`
    - `Sf1DimensonParams=["0","100"]` または `["0","80"]`
    - `Sf2DimensionType=1531`
    - `Sf2DimensonParams=["200","15"]` または `["300","15"]`

- **FlangeType / FlangeParams**
  - DL02 の 1201 系、または `WF/WB` を使う FR 主部材ブラケットで適用
  - 今回の追加代替候補では、失敗回避を優先して**標準 1501 または 1505 に寄せる**
  - 失敗した FR11×DL03.FP に対して特殊 flange パターンは流用しない

- **Sf1EndElements / Sf2EndElements**
  - Side の `1505` では重要
  - 参考スクリプトで成功している plate-profile 構成は、`Sf1EndElements` 付き
  - 今回追加した #50, #51 もこの方針に合わせる

- **BaseElement**
  - 常に Wall 側 profile を優先
  - `Side.FR11.CDP ↔ Deck.A.DL03.FP` の失敗でも `BaseElement=profile111[0]` 自体は妥当
  - 問題は `Surfaces2` 側の組合せ成立性と判断する

### 注意事項
- **Fore/Aft 方向整合ルールは必須**
  - `Wall.Fore.DLxx.*` → `Deck.*.DLxx.F / FP` のみ
  - `Wall.Aft.DLxx.*` → `Deck.*.DLxx.A / AP` のみ
  - 今回の修正版でもこのルールを再確認済み

- **片側 Deck のみ存在する DL は反対側壁から除外**
  - `.F/.FP` のみなら Aft から除外
  - `.A/.AP` のみなら Fore から除外

- **Side Wall は方向制約なし**
  - ただし Deck 側の実在は必須
  - さらに **FR一致優先**
  - FR不一致や特殊補助材従属ラインへの直結は、成功実績がない限り除外寄り

- **今回の失敗候補の扱い**
  - `profile111[0]+",FL"` × `profile108[0]+",FL"` は**除外確定**
  - 理由
    1. COM 例外発生済み
    2. `1501 / FL×FL` が成立しなかった
    3. Deck.A.DL03.FP は特殊 `FR14F415` 系列に従属し、Side 直結が不安定
    4. 既成功型の `CDP ↔ Deck.FRxxP` または `PLS × FL` に比べて信頼性が低い

- **波及除外**
  - 同系統の `Side.FR11.CDP ↔ Deck.C.DL03.FP` も、今回の失敗パターンと近いため除外
  - 失敗候補のみ修正・除外する方針に沿い、成功済み1〜19は変更しない

- **成功済み候補の維持**
