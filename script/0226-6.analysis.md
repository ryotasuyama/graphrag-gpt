## ブラケット配置分析

### 対象部材とループ変数
- **Deck（板）系 Surfaces（Sheet）**
  - `extrude_sheet1`: HK.Casing.Deck.B（SweepDirection=+X、SheetAlignNormal=-0,0,1）
  - `extrude_sheet2`: HK.Casing.Deck.A（同上）
  - `extrude_sheet4`: HK.Casing.Deck.C（同上）
  - `extrude_sheet5`: HK.Casing.Deck.D（同上）
  - `extrude_sheet7`: HK.General.Deck.UpperDeck（SweepDirection=+X、SheetAlignNormal=0,0,-1）
- **Wall（板）系 Surfaces（Sheet）**
  - `extrude_sheet3`: HK.Casing.Wall.Aft（SweepDirection=+Z、SheetAlignNormal=1,0,0）
  - `extrude_sheet6`: HK.Casing.Wall.SideP（SweepDirection=+Z、SheetAlignNormal=0,-1,0）
  - `extrude_sheet9`: HK.Casing.Wall.SideS（SweepDirection=+Z、SheetAlignNormal=0,-1,0）
  - `extrude_sheet8`: HK.Casing.Wall.Fore（SweepDirection=+Z、SheetAlignNormal=1,0,0）
- **Profile（主に Stiffener/FB/Angle 等、BracketのBaseElement候補）**
  - `profile*` の多くが `ProfileType=1002/1003/1201/1007` で生成されており、Bracketの `BaseElement` になり得る。
  - 特に「壁（Fore/Aft/Side）」側の縦通材/フレーム相当：例 `profile62(HK.Casing.Wall.Fore.DL01.CDP)`, `profile75(HK.Casing.Wall.Aft.DL01.CDP)`, `profile65(HK.Casing.Wall.Side.FR09.CDP)` など。
  - 「デッキ」側の梁/縦通材相当：例 `profile23(HK.Casing.Deck.D.DL01.AP)`, `profile18(HK.Casing.Deck.D.DL02P)` など。
- **Solid（板厚付け後のソリッド、BracketのSurfacesにPLSとして使われることがある）**
  - `solid2`: HK.Casing.Deck.D（thicken2）
  - `solid1`: HK.Casing.Deck.C（thicken1）
  - `solid3`: HK.Casing.Deck.B（thicken3）
  - `solid4`: HK.Casing.Deck.A（thicken4）
  - `solid5`: HK.Casing.Wall.Aft.CD（thicken5）
  - `solid10`: HK.Casing.Wall.Fore.CD（thicken10）
  - `solid12/solid15/solid16` 等：Sideの各状態（CDP/BCP/ABP）
- **ループ変数化候補（原スクリプト側の構造）**
  - 原スクリプトは**大量の profile を個別生成**しており、ブラケット対象も「FR/ DL の離散位置」で繰り返し発生する構造。
  - 参考スクリプトのパターンから、以下を**グルーピングしてループ対象**にできる：
    - Side壁のフレーム位置：`FR07/08/09/10/11/12/14/15`（`profile16, profile12, profile65, profile44, profile111, profile52, profile113, profile3` などの「.CDP」系）
    - Fore/Aft壁のデッキライン位置：`DL01/02/03/04/05`（`profile62, profile59, profile117, profile100, profile32` 等）
    - Aft壁のDL系：`profile75 (DL01)`, `profile19 (DL02)`, `profile36 (DL03)`, `profile87 (DL04)`, `profile72 (DL05)` 等

### ループ構造方針
- **(A) Side壁フレーム群（1505）をループ**
  - 対象：`profile16(FR07)`, `profile12(FR08)`, `profile44(FR10)`, `profile111(FR11)`, `profile52(FR12)`, `profile113(FR14)`, `profile3(FR15)`  
  - 共通仕様（参考踏襲）：
    - `BracketType=1505`
    - `Surfaces1`：`["PLS","False","False","0","-0","-1", solid2]`（Deck.D のソリッド面）
    - `Surfaces2`：`[profileX[0]+",FL"]`
    - `Sf1DimensionType=1541` / `Sf1DimensonParams=["0","100"]`
    - `Sf1EndElements`：Deck側の基準材（参考では `profile1[0]` 等）に相当する部材を設定（原スクリプトでは Deck.D 側の基準プロファイルとして `profile17(DL05P)` 等から選定）
    - `Sf2DimensionType=1531` / `Sf2DimensonParams=["200","15"]`
- **(B) Fore/Aft のDL系（1501）をループ**
  - Fore：DL01/02/03/04/05（`profile62, profile59, profile117, profile100, profile32` など）
  - Aft：DL01/02/03/04/05（`profile75, profile19, profile36, profile87, profile72` など）
  - 共通仕様（参考踏襲）：
    - `BracketType=1501`
    - `Surfaces1=[wallProfile+",FL"]`
    - `Surfaces2=[deckProfile+",FL"]`（同一DL番号のデッキ側プロファイル）
    - `Sf1DimensionType=1531` / `Sf1DimensonParams=["200","15"]`
    - `Sf2DimensionType=1531` / `Sf2DimensonParams=["200","15"]`
  - DL02のように `ProfileType=1201`（フランジ付き）を絡める場合のみ、参考同様に：
    - `FlangeType=262` と `FlangeParams=[...]` を付与する分岐を置く

### パラメータ依存関係
- **Thickness（板厚）**
  - 参考スクリプトでは、ブラケット板厚は接続材の板厚/部材タイプに応じて概ね固定値で使い分け：
    - 一般（1002系相当）: `Thickness="7.9999..."`（≒8mm）
    - Sideフレーム系（1505）: `Thickness="9.9999..."`（≒10mm）
    - Flange付き（1201絡み、DL02等）: `Thickness="12"`（12mm）
  - 原スクリプトでも同様に、**壁/デッキで使っているプロファイルのt感**に合わせて、上記3段階で割り当てるのが参考踏襲として妥当。
- **BracketType と寸法指定**
  - `BracketType=1501`（一般）  
    - `Sf1DimensionType=1531` / `Sf1DimensonParams=["200","15"]` が基本  
    - 大きめ（DL02など）では `["800","15"]` の例が参考にある（DL02 Fore/Aft）
  - `BracketType=1505`（片側基準/高さ指定）  
    - `BracketParams=["200"]`（=ブラケットの代表寸法として運用されている）
    - `Sf1DimensionType=1541` / `Sf1DimensonParams=["0","100"]`（Deck側の位置決め）
    - `Sf1EndElements` を必ず入れる（参考では Deck側の端部基準要素を指定して寸法決めを安定化）
- **Surfaces指定の型（重要）**
  - `Surfaces1/Surfaces2` は**面ペア**で、面種別は以下が登場：
    - Profile面：`,FL`（フランジ面）、`,WF`（ウェブ/フランジ?）、`,WB`（ウェブ背面?）  
    - PLS面：`["PLS", "False","False", nx, ny, nz, element]`
  - 原スクリプトには `solid2` 等のDeckソリッドがあるため、1505では参考同様 `PLS + solid` を用いるのが再現性が高い。
- **End1/End2 の指定**
  - ブラケット作成（CreateBracketParam）では、参考スクリプトに **End1/End2 を直接指定する例は基本無い**（Profile作成時にはEnd要素が大量に出る）。  
  - ただし 1505 の `Sf1EndElements` / 1501 の `Sf1EndElements` が登場するケースがあり、**寸法方向の基準要素（端部拘束）として必須**扱い。

### 注意事項
- **(1) BaseElement は「片側の代表部材」**
  - 参考スクリプトでは、`BaseElement` は多くの場合 `Surfaces2` 側の profile（または壁側profile）に設定されている。  
  - 原スクリプトでも、ブラケット名称規則（例：`...Wall...Deck...DP`）に合わせ、**壁側プロファイルをBaseElement**にするのが整合的。
- **(2) 1505 を使う場合、Surfaces1 は PLS + solid を推奨**
  - 参考スクリプトの Side FRxx のデッキ取り合いは、`Surfaces1` が `PLS` 指定（法線ベクトル + 要素）で、要素に `solid1`（Deck.Dソリッド）が使われている。  
  - 原スクリプトで Deck.D は `solid2` なので、同様に **`solid2` を PLS の参照要素にする**。
- **(3) フランジ付き（ProfileType=1201）絡みは FlangeType を付ける分岐**
  - 参考の DL02 ブラケット（Fore/Aft）では `FlangeType=262` と `FlangeParams` を付与している。  
  - 原スクリプトでも DL02 周り（`profile18` や `profile59` / `profile19` 等）では、同様の条件分岐を入れないと形状が不完全になる可能性が高い。
- **(4) 寸法タイプ 1531 / 1541 の使い分け**
  - 1531：両辺寸法（例：`["200","15"]`）で汎用的に使われる  
  - 1541：端部基準 + オフセット（例：`["0","100"]`）で、`Sf1EndElements` を伴う  
  - Side壁→Deck のように「Deck面上の取り付け位置基準が必要」な場合は 1541 が安定。
- **(5) 原スクリプトにはブラケット後処理（BodyDivideByCurves）が無い**
  - 参考スクリプトでは、ブラケット作成後に `BodyDivideByCurves` で分割し、形状調整・切り欠き相当を実施している箇所が多い。  
  - 原スクリプトへ同等の品質を求めるなら、ブラケットにも同様の分割処理（同一の `skt_pl4` 等を利用）を計画に含める必要がある。
