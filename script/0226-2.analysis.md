## ブラケット配置分析

### 対象部材とループ変数
- **Wall（板）系 Sheet**
  - `extrude_sheet8`: HK.Casing.Wall.Fore（Fore壁板）
  - `extrude_sheet3`: HK.Casing.Wall.Aft（Aft壁板）
  - `extrude_sheet6`: HK.Casing.Wall.SideP（片舷側壁板）
  - `extrude_sheet9`: HK.Casing.Wall.SideS（反対舷側壁板）
- **Deck（板）系 Sheet / Solid**
  - `extrude_sheet5`: HK.Casing.Deck.D（Deck Dの板要素）
  - `extrude_sheet4`: HK.Casing.Deck.C
  - `extrude_sheet2`: HK.Casing.Deck.A
  - `extrude_sheet1`: HK.Casing.Deck.B
  - `solid2`: HK.Casing.Deck.D（板`extrude_sheet5`から厚み付けしたソリッド）
  - `solid1`: HK.Casing.Deck.C（板`extrude_sheet4`から厚み付けしたソリッド）
  - `solid3`: HK.Casing.Deck.B（板`extrude_sheet1`から厚み付けしたソリッド）
  - `solid4`: HK.Casing.Deck.A（板`extrude_sheet2`から厚み付けしたソリッド）
- **Profile（補強材）**
  - 端部処理/スニップ・スカラップ付きで作られており、ブラケットの **接続相手面（Surfaces）** になり得る。
  - 代表例（参考スクリプトのブラケット接続に現れる型）:
    - `...,FL`（フランジ面）
    - `...,WF`（Web/Flange系の面種別：WF）
    - `...,WB`（Web/Flange系の面種別：WB）
    - `...,FR`（End要素指定で使われる “FR” 面種別）
- **ループ変数の候補（ブラケットを「付けたい対象」）**
  - Side壁の各フレーム位置の縦通材：`profile16 (FR07)`, `profile5 (FR08)`, `profile44 (FR10)`, `profile10/profile111 (FR11)`, `profile24/profile52 (FR12)`, `profile21/profile113 (FR14)`, `profile3 (FR15)` など
  - Fore/Aft壁のデッキライン位置の水平材：`profile62 (DL01 Fore)`, `profile59/profile97 (DL02 Fore)`, `profile117/profile25 (DL03 Fore)`, `profile100/profile13/profile29 (DL04 Fore)`, `profile32/profile6/profile67 (DL05 Fore)` と、Aft側の対応材 `profile75/profile105/profile115 (DL01 Aft)`, `profile19/profile120/profile2/profile11 (DL02 Aft)`, `profile36/profile63/profile9 (DL03 Aft)`, `profile87/profile14/profile1 (DL04 Aft)`, `profile72/profile7/profile66 (DL05 Aft)` 等

### ループ構造方針
- **方針1：Side壁フレーム位置（FRxx）ごとの“縦材→Deck.D”ブラケットをループ**
  - 対象：`FR07/08/10/11/12/14/15` の Side壁縦材（`profile16, profile12, profile44, profile111, profile52, profile113, profile3` 等）
  - ブラケット型：原則 **BracketType=1505**（参考スクリプトの Side FRxx は 1505 が多い）
  - Surfaces1：Deck.D 下面を **PLS** で指定（参考では `solid1` がDeck.Dのソリッド。元スクリプトは `solid2` が該当）
  - Surfaces2：対象縦材の `,FL`
  - ループ用のリスト例（概念）：`side_profiles = [profile16[0], profile12[0], profile44[0], profile111[0], profile52[0], profile113[0], profile3[0]]`
- **方針2：Fore/Aft 壁のDL位置（DL01〜DL05）ごとの“壁水平材→Deck.D”ブラケットをループ**
  - 対象：Fore側 `profile62/profile59/profile117/profile100/profile32` と Aft側 `profile75/profile19/profile36/profile87/profile72` など
  - ブラケット型：原則 **BracketType=1501**（FL-FL接続）
  - Surfaces1：壁側材 `,FL`
  - Surfaces2：Deck.D側材（DLxxP/FP/AP） `,FL`
- **方針3：Deck内部（FR09/FR13 など）の“Web面(WF/WB)→FR材(FL)”ブラケットは個別定義**
  - `Sf1DimensionType=1541` と `Sf1EndElements` を要求するケースがあり、単純ループ化しにくい。
  - 代表：FR13P/FR09P の WF 面接続（参考：bracketPram3, bracketPram12）。

### パラメータ依存関係
- **BracketType**
  - 1501：基本の三角ブラケット（FL-FL接続、またはWF/WBとFL接続で使用）
  - 1505：Side壁フレーム位置で多用されるタイプ（`BracketParams=["200"]` が付く）
- **Thickness（板厚）**
  - 参考スクリプトでは概ね以下に分かれる：
    - 一般的な小型：`7.9999999999999964`（≒8mm）
    - Side壁1505系：`9.9999999999999982`（≒10mm）
    - FB/Flange付き（ProfileType=1201）周り：`12`
  - 元スクリプト側でも、接続先の profile の板厚・種別（1002/1003/1201）に合わせてこの3クラスに寄せるのが整合。
- **Scallop**
  - 多くは `Scallop1Type=1801`。
  - `Scallop1Params` は “0” または “50” が使い分けられている（大型/フランジ付き周辺で 50 が出やすい）。
- **Surfaces1/Surfaces2 の指定法**
  - **Profile面指定**：`[profileX[0]+",FL"]`、`[profileY[0]+",WF"]`、`[profileZ[0]+",WB"]`
  - **PLS指定**（板ソリッド面を法線ベクトルで参照する）：
    - 例：`["PLS","False","False","0","-0","-1", solid2]` のように **方向ベクトル + 対象要素** を与える
    - Side壁1505ブラケットはこの形式が必須
- **Sf1DimensionType / Sf2DimensionType**
  - 1531：一般寸法（例：`["200","15"]`、`["250","15"]`、`["300","15"]`、`["800","15"]`）
  - 1541：端基準の寸法（`["0","80"]` や `["0","100"]`）  
    - 1541使用時に `Sf1EndElements` が必要になるパターンがある（参考：bracketPram3/4/7/12/15/17 等）
- **FlangeType / FlangeParams**
  - Flange付きブラケットは `FlangeType=262` と `FlangeParams=[...]` が付与される（特に DL02 など FB系プロファイル周辺）。
  - 元スクリプトでも `ProfileType=1201` が絡む DL02 のブラケットはこのパターンを踏襲するのが整合。

### 注意事項
- **ブラケットは「Surfaces1 / Surfaces2 の面ペア」で成立**し、単に BaseElement を指定するだけでは不足する。  
  - Profile同士なら `,FL` を基本に、必要に応じて `,WF` / `,WB` を使い分ける。
  - Deck下面を参照する場合は **PLS指定**（ベクトル + 対象ソリッド）を使う（参考スクリプトの Side FRxx が該当）。
- **BracketType=1505 では `BracketParams=["200"]` が必須**（参考スクリプト踏襲）。加えて `Sf1DimensionType=1541` と `Sf1EndElements` をセットするケースが多い。
- **DL02（ProfileType=1201）絡みは例外が多い**  
  - Moldが “-”、板厚12、FlangeType=262 を伴うブラケットが使われる（参考：bracketPram2/5）。
  - Surfaces を単純に `profile...,FL` とせず、**PLSで面を明示**している点に注意。
- **配置後の BodyDivideByCurves による分割が参考スクリプトには多数登場**するが、これは形状トリムの後処理。  
  - 「ブラケットを付ける」最低要件は CreateBracket だが、同様の切断線（`skt_pl4` 等）で分割が必要な設計ルールがある場合、ブラケット作成直後に同処理を追加する前提で仕様化する。
