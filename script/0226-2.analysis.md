## ブラケット配置分析

### 対象部材とループ変数
- **Deck（Sheet）**
  - `extrude_sheet1`: HK.Casing.Deck.B（PL相当の板）
  - `extrude_sheet2`: HK.Casing.Deck.A（PL相当の板）
  - `extrude_sheet4`: HK.Casing.Deck.C（PL相当の板）
  - `extrude_sheet5`: HK.Casing.Deck.D（PL相当の板）
  - `extrude_sheet7`: HK.General.Deck.UpperDeck（PL相当の板）
- **Wall（Sheet）**
  - `extrude_sheet3`: HK.Casing.Wall.Aft（PL相当の板）
  - `extrude_sheet8`: HK.Casing.Wall.Fore（PL相当の板）
  - `extrude_sheet6`: HK.Casing.Wall.SideP（側壁）
  - `extrude_sheet9`: HK.Casing.Wall.SideS（側壁）
- **Profile（主にブラケットの取り付け母材/相手材になる）**
  - 縦通材・補強材（多くが `ProfileType=1002/1003`）: `profile3,5,9,10,12,16,20,21,23,24,25,26,33,34,35,36,44,45,51,52,55,57,58,65,66,67,69,72,73,75,76,79,80,82,83,84,86,87,89,91,92,99,100,101,102,103,104,105,106,107,111,113,114,115,116,117,118,119,121,122,123...` 等
  - フランジ付き/FB等（`ProfileType=1201,1007`）: `profile2,11,18,19,31,56,59,88,97,98,120...`
- **Bracket（参考スクリプトから抽出される典型パターン）**
  - ブラケットは **必ず Surfaces1 / Surfaces2 の“面ペア”**で定義される
  - 参照面の型は大きく2系統  
    - **Profile面参照**: `profileX[0]+",FL"` / `profileY[0]+",WF"` / `profileZ[0]+",WB"` など  
    - **PLS（平面/任意面）参照**: `["PLS","False","False",nx,ny,nz, element]` 形式（例：デッキ板 `solid1` に対して法線 `0,0,-1` など）

### ループ構造方針
参考スクリプトの実装から読み取れる「漏れなく配置する」ための方針は以下。

- **(A) DLライン（縦通）系ブラケット：1501を基本**
  - 対象: Fore壁DL材、Aft壁DL材、Deck側DL材（例：`DL01～DL05`）
  - 方針:  
    - `DLxx` ごとに「壁側profile」と「デッキ側profile」の **FL面同士**を `Surfaces1/2` に割当  
    - `BaseElement` は原則「壁側のprofile」
  - ループ案（概念）:  
    - `for dl in [DL01, DL02, DL03, DL04, DL05]: create 1501 bracket between wall_profile(dl) and deck_profile(dl)`

- **(B) FRフレーム（横材）系ブラケット：1505（Deck板×側壁FR材）を基本**
  - 対象: Side壁の `FR07/FR08/FR10/FR11/FR12/FR14/FR15` 等、Deck板へ立ち上がる補強材
  - 方針:  
    - `Surfaces1` は `PLS` で deck solid（参考では `solid1`=Deck.Dのような“厚み付け後のSolid”）を参照し、法線を `0,0,-1` としてデッキ面を指定  
    - `Surfaces2` は対象FR profile の `FL`  
    - `Sf1DimensionType=1541` を使い、`Sf1EndElements` で「端の基準要素（多くはDL材）」を与えることで配置位置を安定化
  - ループ案（概念）:  
    - `for fr_profile in side_fr_profiles: create 1505 bracket between deck_solid and fr_profile`

- **(C) ガーダ/フランジ付き部材接続：1501 + FlangeType(262) の派生**
  - 対象: `ProfileType=1201`（フランジ付き）周辺、または `WF/WB` 参照が必要なガーダ接合
  - 方針:  
    - 1501を基本にしつつ、接合相手が `WF/WB` 面を持つ場合は `Surfaces1` に `",WF"` / `",WB"` を採用  
    - 参考では FR09/FR13 系で `Sf1EndElements=[profile4[1]+",FR"]` のように端要素で拘束する

### パラメータ依存関係
参考スクリプトに基づく、ブラケットパラメータの決定ロジック（元スクリプトへ適用する際の基準）。

- **Thickness（板厚）**
  - 1501（一般三角）  
    - 例: `7.9999999999999964`（実質8mm）、`12`（DL02フランジ付き周辺）など  
    - 接続相手が `ProfileType=1201`（板厚12mm級の部材）だと `Thickness="12"` が選ばれる傾向
  - 1505（補強三角/パラメトリック）  
    - 例: `9.9999999999999982`（実質10mm）  
- **BracketType**
  - `1501`: Profile面同士（FL-WF-WBを含む）の“基本ブラケット”
  - `1505`: `BracketParams=["200"]` を伴い、Deck面（PLS）とProfile(FL)を結ぶ用途が中心
- **Scallop（逃げ）**
  - `Scallop1Type=1801` が基本
  - `Scallop1Params=["0"]` または `["50"]`  
    - フランジ付き/大きめブラケットで `50` が選ばれるケースがある
- **Surfaces1 / Surfaces2 の指定方法**
  - Profile面参照: `"<element>,FL"` / `WF` / `WB`
  - PLS参照（平面定義）: `["PLS","False","False",nx,ny,nz, element]`
- **Dimension（ブラケット寸法入力）**
  - `Sf1DimensionType=1531`（例: `["200","15"]`, `["250","15"]`, `["800","15"]`）  
    - “辺長 + 端オフセット”のような2パラメータ運用
  - `Sf1DimensionType=1541`（例: `["0","100"]` や `["0","80"]`）  
    - 端基準（EndElements）とセットで使われる
  - `Sf1EndElements / Sf2EndElements`  
    - 参考では **1505やWF参照ブラケット**で登場し、配置の端基準を与えている

### 注意事項
- **ブラケットは Surfaces1/Surfaces2 の“面ペア”で成立**するため、元スクリプト側で「どの部材面を参照するか」を明確に分ける必要がある（FL/WF/WB/PLS）。
- **Deck面を直接 sheet で参照せず、solid（thicken後）を PLS 参照に使う**パターンが参考スクリプトでは多い。  
  - 例: `Surfaces1=["PLS",...,(0,0,-1), solid1]` のように「法線＋対象solid」で面を表現。
- **1505 は `Sf1EndElements` が実質必須**になりやすい（端基準が無いと配置が不安定になりやすい構造で使われている）。
- **フランジ付きブラケット（FlangeType=262）**は、フランジ付き部材（`ProfileType=1201`）が絡む箇所で使用される。元スクリプト側でも DL02等の1201材まわりはこの派生を優先的に検討する。
- 参考スクリプトでは、ブラケット作成後に **`BodyDivideByCurves`（分割）**が多数適用され、形状を後処理している。元スクリプトへブラケット追加する場合も、同様の分割工程が必要になる可能性が高い（ただし本分析では“ブラケット配置仕様”に限定し、分割は後工程扱い）。
