### ループ構造方針
- **Wall.Fore × Deck の DL 系**
  - `DL01〜DL05`, `DL00` をキーにして
  - Wall.Fore の `CDP/BCP/ABP/OAP` 系から、Deck の `F/FP` 側のみを対応付ける
  - 代表例:
    - `HK.Casing.Wall.Fore.DL01.*` → `HK.Casing.Deck.*.DL01.F / FP`
- **Wall.Aft × Deck の DL 系**
  - `DL01〜DL05`, `DL00` をキーにして
  - Wall.Aft の `CDP/BCP/ABP/OAP` 系から、Deck の `A/AP` 側のみを対応付ける
  - 代表例:
    - `HK.Casing.Wall.Aft.DL01.*` → `HK.Casing.Deck.*.DL01.A / AP`
- **Wall.Side × Deck の FR 系**
  - `FR07,08,09,10,11,12,13,14,15` を軸に、
  - 同じ FR 位置にある Deck 側部材、もしくはその近傍 Deck plate を候補化
  - Side Wall は Fore/Aft 制約なし
  - ただし接続先 Deck プロファイルの存在確認は必須

- **BracketType のループ選択**
  - `Profile FL × Profile FL` は `1501`
  - `Plate PLS × Profile FL` は `1505`
  - 参考スクリプトで `Sf1EndElements` が使われている Side 壁用 plate ブラケットは必ず同様に適用

- **実装上の反復単位**
  - 1つの `wall_profile` ごとに
    1. 対応する `deck_profile` 候補抽出
    2. Fore/Aft 整合チェック
    3. Surfaces1/Surfaces2 形式判定
    4. `BracketType` 決定
    5. `BaseElement` を wall 側 profile に固定
    6. 必要なら `Sf1EndElements` を付加
  - Side 系は `side_profiles_cdp` などの配列でループし、漏れなく全 FR を処理

### パラメータ依存関係
- **BracketType**
  - `1501`
    - 2面ブラケット
    - `Profile FL × Profile FL` の標準形
    - 参考スクリプトでも Wall/Deck の一般ブラケットに多数採用
  - `1505`
    - 2面ブラケット
    - `Plate PLS × Profile FL` の標準形
    - Side Wall と Deck plate/solid の接続で採用
    - `BracketParams=["200"]` の付与パターンを踏襲するのが妥当

- **Thickness**
  - 参考スクリプトでは接続 profile の板厚・Web/Flange 厚に対応
  - 実質的には以下の粒度で使い分け
    - 小型 L/T 系 profile 接続: `7.9999999999999964` ≒ 8
    - Side Wall の一般 stiffener 接続: `9.9999999999999982` ≒ 10
    - FB/1201 系の厚板接続: `12`
    - Deck FR profile 接続: `8.9999999999999982` ≒ 9
  - よって仕様上は
    - `ProfileType 1002` 同士の標準 DL 接続 → **8**
    - Side wall profile と plate 接続 → **10**
    - `ProfileType 1201` を含む DL02 系 → **12**
    - `ProfileType 1003` の FR13/FR09 deck girder 系 → **8 or 9**
  - と整理できる

- **Sf1 / Sf2 Dimension**
  - `1501` の標準 profile-profile:
    - `Sf1DimensionType=1531`
    - `Sf2DimensionType=1531`
    - `Sf1DimensonParams=["200","15"]`
    - `Sf2DimensonParams=["200","15"]`
  - FR girder 系では
    - `["250","15"]` や `["300","15"]`
  - `1505` の plate-profile:
    - `Sf1DimensionType=1541`
    - `Sf1DimensonParams=["0","100"]` または `["0","80"]`
    - `Sf2DimensionType=1531`
    - `Sf2DimensonParams=["200","15"]` または `["300","15"]`

- **FlangeType / FlangeParams**
  - 一部の 1201 系・WF/WB 接続ブラケットで採用
  - 参考スクリプトでは
    - `FlangeType=262`
    - `FlangeParams=["100","30","29.999999999999996","30","30","1"]`
    - または `["75","30","29.999999999999996","30","50","1"]`
  - 元スクリプトに 1201 profile が複数あるため、DL02 周辺のブラケットにはフランジ付き仕様が妥当

- **Sf1EndElements / Sf2EndElements**
  - **Side Wall の Plate PLS × Profile FL** のとき重要
  - 参考スクリプトでは、Deck plate 側の寸法基準終端として `Sf1EndElements` を設定
  - 例:
    - `["PLS","False","False","-0","-1","-0",profile1[0]]`
  - 元スクリプトでも Side 用 plate ブラケットには同パターンを適用すべき
  - `1501` の標準 Wall/Deck profile 同士では通常不要

- **BaseElement**
  - 原則として **Wall 側 profile** をベースにする
  - Side-Deck FR 接続でも `BaseElement` は Side 側 profile が自然
  - Deck FR profile を基準にするケースは参考スクリプトで限定的なので、統一のため Wall 側固定が安全

### 注意事項
- **Fore/Aft 方向整合ルールは必須**
  - `Wall.Fore.DLxx.*` は `Deck.*.DLxx.F / FP` のみ接続可
  - `Wall.Aft.DLxx.*` は `Deck.*.DLxx.A / AP` のみ接続可
  - たとえば
    - `HK.Casing.Wall.Fore.DL01.CDP` と `HK.Casing.Deck.D.DL01.AP` は **禁止**
    - `HK.Casing.Wall.Aft.DL01.CDP` と `HK.Casing.Deck.C.DL01.FP` も **禁止**
- **片側 Deck のみ存在する DL は反対壁から除外**
  - ある DL で `.F/.FP` しかないなら Aft 側壁とのブラケット候補にしない
  - `.A/.AP` しかないなら Fore 側壁とのブラケット候補にしない
- **Side Wall（FR 系）は方向制約なし**
  - ただし接続対象となる Deck profile または Deck plate が実在することを確認する
- **面の種類を混同しない**
  - `1501` は基本 `profile + ",FL"` 同士
  - `1505` は `["PLS", ... , solidX]` と `profile + ",FL"`
  - 参考スクリプトには `WF`, `WB`, `PLS` の混在例があるため、母材ごとに使い分けが必要
- **DL02 の 1201 断面は特殊**
  - `profile18`, `profile19`, `profile59`, `profile88`, `profile97`, `profile120` などは flange/body を持つため
  - bracket 生成時に `[0]` / `[1]` のどちらを使うかを明確にする必要がある
  - 参考スクリプトでは `profile12[1]`, `profile11[1]`, `profile4[1]` のように flange/body 側を PLS 参照に使っている
- **30個以上の配置は Side 系を含めるのが合理的**
  - Fore/Aft の DL 接続だけでは数が不足しやすい
  - Side.FR07〜15 の複数部材と Deck A/B/C/D の組み合わせを含めることで、30超の配置が安定して確保できる
- **参考スクリプト準拠の安全な組み方**
  - Wall/Deck profile 同士 → `1501`
  - Side wall profile と deck plate/solid → `1505 + Sf1EndElements`
  - 1201 profile 周辺 → 必要に応じて `FlangeType=262`
- **名称から候補抽出する際の実務ルール**
  - `DLxx`, `FRxx`, `Fore/Aft`, `F/FP/A/AP` のトークンを必ず分解して判定
  - 名前が近くても suffix が一致しないものは候補除外する
  - `BCP`, `CDP`, `ABP`, `OAP` は profile種別として扱い、方向判定には使わない

以上より、この元スクリプトに対しては、**Fore/Aft の DL 系ブラケット + Side FR 系ブラケットを組み合わせて 40 件前後のブラケット候補を定義する方針**が最適です。
