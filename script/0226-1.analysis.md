### ループ構造方針
- **DL（縦方向）取り合いブラケット（BracketType=1501中心）**
  - Fore壁DL群、Aft壁DL群をそれぞれ「壁側Profile」と「Deck側Profile」のペアで辞書化し、順に作成するのが最も漏れがない。
  - 例（概念）:
    - Fore: `(profile62, profile61)`, `(profile59, profile18)`, `(profile117, profile35)`, `(profile100, profile86)`, `(profile32, profile17)`
    - Aft: `(profile75, profile23)`, `(profile19, profile18)`, `(profile36, profile35)`, `(profile87, profile86)`, `(profile72, profile17)`
- **Side FR取り合い（BracketType=1505中心）**
  - Deck側参照面を `solid2`（HK.Casing.Deck.Dソリッド）に統一し、FRごとに `BaseElement=profileXX[0]`、`Surfaces2=profileXX[0]+",FL"` の形で反復生成。
  - `Sf1EndElements` が必要な型（参考: 1505で `Sf1EndElements` を指定している）は、**基準となるDeck側の端要素**（例: DL05のDeck側縦材 `profile17[0]` 相当）を必ず同様に付与する方針とする。

### パラメータ依存関係
- **共通**
  - `DefinitionType=1`
  - `MaterialName="SS400"`
  - `UseSideSheetForPlane=False`
  - `Scallop1Type=1801`（多用）、`Scallop2Type=0`
- **BracketType=1501（一般ブラケット）**
  - 厚み `Thickness` は接続先Profileの板厚相当を踏襲する傾向  
    - 参考では `7.9999...`（=8mm相当）、`12`、`8.9999...`（=9mm相当）など
  - 面ペアは以下が代表:
    - `Surfaces1=[profileA[0]+",FL"]` と `Surfaces2=[profileB[0]+",FL"]`（FL×FL）
    - もしくは Web接続: `Surfaces1=[someProfile+",WF"]` / `",WB"` + `Surfaces2=[otherProfile+",FL"]`
  - 寸法指定:
    - `Sf1DimensionType=1531`, `Sf1DimensonParams=["200","15"]`（または `["250","15"]`、`["300","15"]`）
    - `Sf2DimensionType=1531`, `Sf2DimensonParams` 同様
  - **Flange付きブラケット**
    - 参考: `FlangeType=262`, `FlangeParams=[...]` を使用するケースがある（DL02等の大きい取り合いで出現）
- **BracketType=1505（Deck板—Side FR等での定型）**
  - `BracketParams=["200"]`（高さ/投影寸法の代表値として固定的）
  - `Surfaces1` は `PLS` + 参照要素（Deckソリッド）で与え、`Surfaces2` はProfileの `",FL"`
  - `Sf1DimensionType=1541`, `Sf1DimensonParams=["0","100"]` のような「オフセット＋長さ」系が使われる
  - 参考では `Sf1EndElements` を与えて基準端を拘束している（例: `profile1[0]` 等）。同様の拘束が必要な場合は、対象Deckの基準部材（DL05縦材など）を選定して踏襲。

### 注意事項
- **Surfaces1/Surfaces2は“面ペア”で定義**され、Profileの場合は `FL/WF/WB` の面指定が必須。板・ソリッド側は `PLS` 指定（法線＋参照要素）を使うパターンがある。
- **DL02（1201: フランジ付きProfile）絡みのブラケット**は、参考スクリプトでも `Surfaces1/Surfaces2` が `PLS` 形式になっており、さらに `FlangeType=262` が付く。元スクリプトでも同様の取り合い（`profile18`, `profile19`, `profile59`, `profile97/98/88` 系）が多いため、DL02周りは1501でも「PLS×PLS + Flange付き」になる可能性が高い。
- 参考スクリプトでは、ブラケット作成後に `BodyDivideByCurves` で形状分割しているケースが多い（干渉回避・仕上げ意図）。元スクリプト側で同等の仕上げが必要なら、分割用スケッチ平面（例: `skt_pl4`）を流用して同様の後処理を設計に含める。
- MirrorCopyが多用されているため、**P/S（左右）で同一ブラケットを複製**する運用が想定される。ブラケットも左右対象なら、片舷生成→MirrorCopy、または左右それぞれのProfileに対し同一ロジックで生成する方針を採る。
- `BaseElement` は参考スクリプトで一貫して「取り付け基準となるProfile（または対象Profile）」が設定されているため、元スクリプトでも **壁側ProfileまたはSide FR ProfileをBaseElementに統一**すると整合が取りやすい。
