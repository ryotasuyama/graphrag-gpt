**補足（Surfaces表記について）**
- `profileX[0]+",FL"` のように **Profile要素は断面の面種（FL/WF/WB/FR等）**で指定される。
- `PLS,...,solid1` のような指定は、参考スクリプトでは `Surfaces1=["PLS","False","False", nx,ny,nz, element]` 形式で、**法線ベクトル + 参照要素**をセットにして面を定義している（元スクリプトでも同方式を踏襲する想定）。
- `Sf1EndElements` は、Type=1541 のときに「端基準要素」を入れて寸法の基準を確定させる目的で使われている。

### ループ構造方針
- **方針A：Side FRxx → Deck ソリッド（1505）を一括生成**
  - 対象：`profile16(FR07)`, `profile12(FR08)`, `profile111(FR11)`, `profile52(FR12)`, `profile44(FR10)`, `profile113(FR14)`, `profile3(FR15)` 等  
  - 仕様共通点（参考より）  
    - `BracketType=1505`  
    - `Surfaces1`: `PLS`でDeckソリッド（本件では主に `solid2`=Deck.D）  
    - `Surfaces2`: `profile?,FL`  
    - `Sf1DimensionType=1541` + `Sf1DimensonParams=["0","100"]`  
    - `Sf1EndElements`: `PLS`で「端基準（例: Deck側縦通材 profile1[0] / DL05の部材など）」を指定
    - `Sf2DimensionType=1531` + `Sf2DimensonParams=["200","15"]`
- **方針B：DLxx（Wall）↔ DLxx（DeckのFP/AP/中間材）（1501）を一括生成**
  - 対象：DL01/DL03/DL04/DL05 など（例：`profile62 ↔ profile61`, `profile117 ↔ profile35`, `profile87 ↔ profile86`, `profile32 ↔ profile17`, `profile72 ↔ profile17`, `profile75 ↔ profile23`）
  - 仕様共通点（参考より）
    - `BracketType=1501`
    - `Surfaces1=[Wall側 or Side側 profile ,FL]`
    - `Surfaces2=[Deck側 profile ,FL]`
    - `Sf1DimensionType=1531` / `Sf2DimensionType=1531`（両側とも同一タイプ）
    - `DimensonParams` は多くが `["200","15"]`、一部 `["250","15"]`
- **方針C：大型FRxx（1003）× Deck側DL02（1201）のWF/WB面取り合い（1501）**
  - 対象：`profile42(FR13P)`, `profile64(FR09P)`, `profile60(FR13C)`, `profile22(FR09C)` 等
  - 仕様共通点（参考より）
    - `Surfaces1` に `profile18[0]+",WF"` または `profile18[0]+",WB"` を使用（DL02Pの「Web/Face」面）
    - `Surfaces2` は FRxx の `,FL`
    - `Sf1DimensionType=1541` を使用し、`Sf1EndElements=[profile18[1]+",FR"]` のように端要素を入れるケースがある

### パラメータ依存関係
- **BracketType と形状**
  - **1501**: 一般的な三角ブラケット系（FL同士、WF/WB×FLなどの接続に使用）
  - **1505**: 片側が板ソリッド（PLS）参照になる「立ち上がり（または片持ち）ブラケット」系。`BracketParams=["200"]` がセットされる例がある（参考: bracketPram4/7/15/17/19/21/22）
- **Thickness（板厚）**
  - 参考スクリプトでは接続先の部材に合わせた板厚が明示される：
    - DL01/DL03/DL04/DL05系（ProfileType=1002/小形）: `Thickness="7.9999..."`（≒8mm）
    - Side FRxx（1002）→ Deck: `Thickness="9.9999..."`（≒10mm）
    - DL02（ProfileType=1201）関連: `Thickness="12"`（12mm）
    - FR09/FR13（1003）関連: `Thickness="8.9999..."`（≒9mm）や `7.9999...`
- **寸法指定（Sf1DimensionType / Sf2DimensionType）**
  - **1531**: 両側に `["200","15"]` や `["250","15"]`、`["300","15"]`、`["800","15"]` 等を与えるパターンが多い  
    - 1つ目がブラケットの「脚長（または高さ/幅）」、2つ目がオフセット/余長のように使われている挙動
  - **1541**: `["0","80"]` や `["0","100"]` のように「一方を0固定＋もう一方を指定」  
    - この場合に `Sf1EndElements` が併用され、端基準を確定している
- **Scallop（切欠き）**
  - `Scallop1Type=1801`
  - `Scallop1Params` は `["0"]` と `["50"]` が混在（配置箇所で統一されている傾向）
- **FlangeType**
  - DL02（1201）でPLS面参照のブラケットでは `FlangeType=262` と `FlangeParams=[...]` が追加される（参考: bracketPram2/bracketPram5/bracketPram3/bracketPram12 など）

### 注意事項
- **Surfaces1/Surfaces2は「面ペア」でブラケットを定義する**  
  - Profile相手は `",FL" / ",WF" / ",WB"` 等、面種を必ず指定する。
  - 板（ソリッド）相手は `PLS`定義（法線ベクトル＋参照要素）を使うケースがある（1505で顕著）。
- **DefinitionType=1 を踏襲**  
  - 参考スクリプトのブラケットは全て `DefinitionType=1` で作成されている。
- **BaseElementの選定**  
  - 参考では多くが「Surfaces2側のprofile」や、ブラケット名の主語になる部材を `BaseElement` にしている。配置安定性のため、**接続の主部材（例：Wall側profileやSide FRxx）をBaseElementに固定**するのが一貫している。
- **Sf1EndElements / Sf2EndElements の必要性**  
  - `Sf1DimensionType=1541` を使うブラケット（特に1505）では、端基準が曖昧になりやすいため、参考スクリプト同様に `Sf1EndElements` を設定している。元スクリプトにブラケットを追加する場合も、このパターンは踏襲すべき。
- **後処理（BodyDivideByCurves）**  
  - 参考スクリプトでは作成したブラケットに対し `BodyDivideByCurves` で分割している例が多数ある。  
  - 元スクリプトの編集指示は「ブラケットをつけてください」なので必須とは限らないが、**他ソリッドと干渉する場合や製作境界を揃える場合に分割工程が必要になる**可能性が高い。
