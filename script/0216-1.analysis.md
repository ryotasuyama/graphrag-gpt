  - `extrude_sheet9`: HK.Casing.Wall.SideS（板）
- **縦通材／補強材（Profile）**
  - DL系（甲板方向Y基準の縦通）: `profile17(DL05P@Deck.D)`, `profile18(DL02P@Deck.D)`, `profile23(DL01.AP@Deck.D)`, `profile33(DL00.A@Deck.D)`, `profile35(DL03P@Deck.D)`, `profile86(DL04P@Deck.D)` など

  - FR系（横フレーム方向X基準の縦材）: `profile16(FR07.CDP)`, `profile12(FR08.CDP)`, `profile44(FR10.CDP)`, `profile111(FR11.CDP)`, `profile52(FR12.CDP)`, `profile43(FR13.CDP)`, `profile113(FR14.CDP)`, `profile3(FR15.CDP)` 等

### ループ構造方針
- **ループA（Side FR系：BracketType=1505で統一）**
  - 対象: `profile16(FR07.CDP)`, `profile12(FR08.CDP)`, `profile44(FR10.CDP)`, `profile111(FR11.CDP)`, `profile52(FR12.CDP)`, `profile113(FR14.CDP)`, `profile3(FR15.CDP)`
  - 各要素に対し同一テンプレートで `CreateBracketParam()` を作成し、以下を固定:
    - `BracketType=1505`
    - `BracketParams=["200"]`
    - `Surfaces1`: `PLS` + Deck.Dソリッド（参考では `solid1`）
    - `Surfaces2`: 対象profileの `",FL"`
    - `Sf1DimensionType=1541`, `Sf1DimensonParams=["0","100"]`
    - `Sf1EndElements`: `PLS` + 参照用の縦通/端部要素（参考では `profile1[0]`）
    - `Sf2DimensionType=1531`, `Sf2DimensonParams=["200","15"]`
- **ループB（DL縦通：BracketType=1501でFL-FL）**
  - 代表: DL01/DL03/DL04/DL05 などで、**Wall側DL(CDP)** と **Deck側DL(P/FP/AP)** の組合せをペアで処理
  - `Surfaces1=[WallProfile+",FL"]`, `Surfaces2=[DeckProfile+",FL"]`
  - `BracketType=1501`, `Sf1DimensionType=1531`, `Sf2DimensionType=1531`（両側とも同じ寸法タイプで揃える）
- **ループC（DL02の特殊：PLS-PLS + フランジ）**
  - DL02だけは参考スクリプトで「PLSベクトル指定 + FlangeType=262」が入るパターンがあるため、DL02は別分岐で扱う。
  - `Thickness="12"`, `Scallop1Params=["50"]`, `FlangeParams=["100","30","30","30","30","1"]`（参考相当）

### パラメータ依存関係
- **BracketType の使い分け**
  - `1501`: 一般的なブラケット（多くが **FL-FL**、または **WF/WB-FL** の取り合い）
  - `1505`: Deckソリッド等から作る **PLS面** と profile FL を結ぶタイプ（Side FR縦材で多用）
- **Thickness（板厚）**
  - 参考スクリプトでは概ね以下の実績値:
    - DL01/DL03/DL04/DL05 系の多く: `Thickness="8"`（実値は 7.9999…）
    - Side FR系(1505): `Thickness="10"`（実値は 9.9999…）
    - DL02系(フランジ付・PLS指定): `Thickness="12"`
  - 依存ロジックとしては「接続先の部材（ProfileTypeやt）」連動が理想だが、参考スクリプトは**固定値運用**が多く、元スクリプトに追加する際も固定値踏襲が最も整合する。
- **寸法指定（Sf1DimensionType / Sf2DimensionType）**
  - `1531`: 多くのFL-FLブラケットで使用。`Sf?DimensonParams=["200","15"]` や `["250","15"]`, `["800","15"]` が出現。
  - `1541`: PLSを使う/端部要素を使う場合に出現。`Sf1DimensonParams=["0","100"]` や `["0","80"]`。
  - **Sf1EndElements / Sf2EndElements**
    - 1505や一部1501で、形状の終端（FR面など）を与えるために使われる（例: `Sf1EndElements=[profile4[1]+",FR"]`）。
    - 元スクリプトでも、FRで終端を揃えたいブラケット（大型桁周り）には導入が必要。

### 注意事項
- **Surfaces1/Surfaces2 は「面ペア」で定義される**
  - `profileX[0]+",FL"` のように **要素＋面種**（FL/WF/WB/FR等）で指定するケースと、  
    `["PLS","False","False", nx,ny,nz, element]` のように **擬似平面(PLS)** をベクトルと要素で定義するケースが混在。
- **DL02ブラケットは別格（PLS指定＋フランジ付が出やすい）**
  - 参考スクリプトで `FlangeType=262` が付くのはDL02の壁・桁取り合いで、通常のFL-FL(1501)とパラメータセットが異なる。
- **Bracket作成後の後処理（BodyDivideByCurves）が多用される**
  - 参考スクリプトでは作成したブラケットを `BodyDivideByCurves` で分割している例が多数あり、最終形状の整合（干渉回避やディテール表現）に影響する。  
  - 元スクリプトへブラケットを追加する場合も、同一基準線（例: `skt_pl4` の線）で分割する運用を合わせないと、モデルの結果が参考スクリプトと一致しにくい。
- **BaseElement の選定**
  - 参考スクリプトでは原則「ブラケットが取り付く側の profile」を `BaseElement` にしている（例: Side FR系はFR縦材 profile がBase、DL系はWall側profileがBaseになっているケースが多い）。  
  - 元スクリプトでもこの規則で統一すると向き・モールド（Mold +/-）の整合を取りやすい。
