### ループ構造方針
参考スクリプトには明示的な Python ループはなく、**配置箇所ごとに個別の BracketParam を作成**している。ただし配置パターンが明確に繰り返されるため、仕様としては以下のループ単位に整理できる。

- **ループA（Wall DLx ↔ Deck DLxP の 1501）**
  - 対象DL: `DL01, DL03, DL04, DL05`（元スクリプトに存在）
  - ペア例:
    - Fore側：`Wall.Fore.DLx.(CDP)` の `profile...` と `Deck.D.DLx(P/FP/AP)` の `profile...`
    - Aft側：`Wall.Aft.DLx.(CDP)` と `Deck.D.DLx(P/FP/AP)`
  - ループ変数（概念）: `dl = [DL01, DL03, DL04, DL05]`
  - 各 `dl` で `Surfaces1=[wall_profile+",FL"]`, `Surfaces2=[deck_profile+",FL"]`, `BracketType=1501`
- **ループB（Side FRxx ↔ Deck.D plate の 1505）**
  - 対象FR: `FR07, FR08, FR10, FR11, FR12, FR14, FR15`（元スクリプトに Side profile がある）
  - 面定義: `Surfaces1` は **PLS形式で Deck.D 板（`solid2`）**を参照、`Surfaces2` は `profileFR[0]+",FL"`
  - `Sf1DimensionType=1541` + `Sf1EndElements` を併用（参考スクリプト踏襲）
- **ループC（Side FRxx ↔ Deck.D FRxxP の 1501）**
  - FR09/FR13など、Deck側に同FR系 Profile（FR09P/FR13P）が存在する場合
  - `Surfaces1=[side_profile+",FL"]`, `Surfaces2=[deck_FR_profile+",FL"]`

### パラメータ依存関係
参考スクリプトから読み取れるブラケットパラメータ設計規約（元スクリプト追加時に踏襲すべきロジック）。

- **共通（CreateBracketParam / DefinitionType=1）**
  - `BracketName`: 接続元要素名 + 接続先要素名（例: `"HK.Casing.Wall.Fore.DL01.Deck.DP"`）
  - `MaterialName="SS400"`
  - `BaseElement`: 原則として **Surfaces2 側の profile** または壁側 profile（参考ではケースにより異なるが、概ね「主部材（Profile）」）
  - `UseSideSheetForPlane=False`
  - `Mold`: 接続系の向きにより `"+"` / `"-"` が分かれる（参考では Fore/Aft、DL02 などで `"-"` を使用）
- **BracketType と Surfaces指定**
  - **1501（Profile×Profile）**
    - `Surfaces1=[profileX+",FL"]`
    - `Surfaces2=[profileY+",FL"]`
    - `Sf1DimensionType=1531`, `Sf1DimensonParams=["200","15"]`（小型）
    - `Sf2DimensionType=1531`, `Sf2DimensonParams=["200","15"]`
    - DL02や大型部では `["800","15"]` など大きい値を使用する例あり
  - **1505（PLS×Profile）**
    - `Surfaces1` は PLS配列形式（`["PLS","False","False", nx,ny,nz, solid_or_profile]`）
    - `Surfaces2=[profile+",FL"]`
    - `Sf1DimensionType=1541`, `Sf1DimensonParams=["0","100"]`（Plate側の取り合い寸法）
    - `Sf1EndElements` を指定する例が多い（PLS形式でエンド参照要素を与える）
    - `Sf2DimensionType=1531`, `Sf2DimensonParams=["200","15"]`
- **Thickness（板厚）の決め方**
  - 参考スクリプトでは主に以下のように **接続Profileの板厚に整合**させている。
    - 小型FP/1002系: `Thickness ≈ 8`（例: `"7.9999999999999964"`）
    - Side縦材/1002で補強: `Thickness ≈ 10`（例: `"9.9999999999999982"`）
    - フランジ付1201系(DL02等): `Thickness="12"`
    - 大型1003系と取り合い: `Thickness ≈ 9`（例: `"8.9999999999999982"`)
  - よって元スクリプトでは、ブラケット厚は原則「**BaseElement となる profile の板厚（t）**」に合わせる（厳密な自動算出が無い場合、上記の代表値を採用）
- **Scallop（スカラップ）**
  - `Scallop1Type=1801`
  - `Scallop1Params=["0"]` または `["50"]` が混在
    - 1501の一般: `0` が多い
    - 大型/フランジ付/形状都合: `50` を使用する例あり
- **Flange（ブラケット自体のフランジ付）**
  - 参考では一部（DL02 Deck取り合い等）で `FlangeType=262` を使用
  - 元スクリプトの「ブラケットをつけてください」要求が一般的補強であれば、まずは **フランジ無し**（1501/1505のみ）を基本とし、DL02等で必要な箇所のみ 262 を検討

### 注意事項
- **Surfaces1 / Surfaces2 は「面ペア」で定義される**  
  - Profileの場合: `profileX[0]+",FL"` のように **FL/WF/WB/FR面**を明示する必要がある。
  - Plateの場合: `PLS` 配列形式を使い、法線ベクトル（`nx,ny,nz`）と参照要素（`solid` や `profile`）を渡す。参考では `["PLS","False","False","0","-0","-1",solid1]` のような形式が使われる。
- **Sf1EndElements / Sf2EndElements の踏襲**
  - 参考スクリプトの 1505 では `Sf1EndElements` を指定して、Plate側の取り合い位置（端部基準）を安定させている。元スクリプトでも **Deck板×Side縦材**のようなケースでは同等の指定を推奨。
- **DL02（ProfileType=1201）周りは例外が多い**
  - 参考では DL02 のブラケットで PLS指定＋フランジ付（262）＋寸法800など、他より複雑。
  - 元スクリプトにも DL02 の 1201部材が多数あるため、単純1501で不足する場合は、参考の bracketPram2/5 の構造（PLS×PLS、FlangeType=262）を適用する。
- **左右対称（MirrorCopy）を含む「漏れない配置」**
  - 元スクリプトは Profile/Solid を大量に MirrorCopy しており、ブラケットも同様に **P側とS側の両方**に必要。
  - 参考スクリプトでも、最終的に多くの要素を `BodyDivideByCurves` して整形しているため、ブラケット追加後に形状干渉が出る場合は同様の分割/トリム工程が必要になりうる。
- **BracketTypeの選択基準**
  - Profile同士（FL×FL）: **1501**
  - Plate(PLS)とProfile(FL): **1505**
  - それ以外（R付き、D指定、3面）は今回ナレッジグラフにあるが、参考スクリプト実装の主流が 1501/1505 のため、まずこの2種で標準化するのが安全。
