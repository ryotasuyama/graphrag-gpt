### ループ構造方針
- **参考スクリプトの特徴**
  - ブラケットは個別に `bracketPramN` を都度生成し、`BaseElement` と `Surfaces1/Surfaces2` を明示して `CreateBracket()`。
  - 1505では `Surfaces1` に **PLS指定（"PLS", ..., solid1）** を使い、**Sf1EndElements** で端部基準（例えば profile1[0] など）を与えて位置決めしている。
  - 1501では `Surfaces1=[profileX+",FL"]` / `Surfaces2=[profileY+",FL"]` のような **FL面同士**、あるいは **PLS×PLS** の面指定がある。

- **元スクリプトへ適用するループの考え方（方針）**
  1. **Side FRxx ループ（BracketType=1505）**
     - ループ対象: `side_frames = [profile16[0], profile12[0], profile44[0], profile111[0], profile52[0], profile113[0], profile3[0]]`
     - 固定要素:  
       - `Surfaces1` は DeckのSolid（例: `solid2`=DeckD）を PLSで指定  
       - `Surfaces2` は各フレームの `",FL"`  
       - `Sf1EndElements` は Deck縦通（例: `profile17[0]` DL05P など）をPLSで指定して統一
     - 左右: MirrorCopyされている場合は `mirror_side_frames` を同様に回す
  2. **Fore/Aft DL ループ（BracketType=1501）**
     - Fore: `(profile62↔profile61), (profile59↔profile18), (profile117↔profile35), (profile100↔profile86), (profile32↔profile17)`
     - Aft: `(profile75↔profile23), (profile19↔profile18), (profile36↔profile35), (profile87↔profile86), (profile72↔profile17)`
     - これを左右対称（Mirror側）も含めて倍化
  3. **Deck別（A/B/C/D）で同一ルールを再利用**
     - デッキ要素が `extrude_sheet1/2/4/5` として分かれているため、同種のDL/FRプロファイルをグルーピングして同じブラケット規則を適用し、数量を増やす。

### パラメータ依存関係
- **Thickness（板厚）**
  - 参考スクリプトでは概ね以下の対応：
    - BracketType=1501（一般三角/標準）: `Thickness` は 8〜12mmが多い  
      - 例: 7.999..., 8.999..., 12
    - BracketType=1505（補助パラメータ `BracketParams=["200"]` を持つタイプ）: `Thickness=9.999...` が多い
  - 元スクリプト側のProfileParamsから推測される板厚例：
    - 1002/1003等のプロファイルで板厚相当が `ProfileParams` 中に含まれる（例 `"9.000..."` や `"7"` が出現）
  - **設計指針（参考踏襲）**
    - Side FR系（1505）: 10mm
    - DL系（1501）: 8mm（軽量）または 12mm（フランジ付き/強度必要箇所：DL02系）

- **BracketType選択ロジック**
  - 参考スクリプトより：
    - **Deck板（Solid/Plate）に取り付け、端部基準が必要** → 1505 + `Sf1EndElements` を使う
    - **Profile同士の接合（FL面同士、またはPLS同士で直交）** → 1501

- **Sf1DimensionType / Sf2DimensionType**
  - 参考では主に2種：
    - `1531` : `["200","15"]` や `["250","15"]` や `["800","15"]`（幅/逃げのようなセット）
    - `1541` : `["0","80"]` や `["0","100"]`（端部基準オフセット系、1505で多い）
  - **元スクリプト適用の指針**
    - 1501（DL接合）: Sf1=1531(200,15) / Sf2=1531(200,15) を基本  
      - Deck FR13/FR09等の大きい部材は 250,15 / 300,15 に拡大
    - 1505（Side FR）: Sf1=1541(0,100)、Sf2=1531(200,15) を基本（参考 bracketPram4/7/15/17/22）

- **End1/End2要否**
  - 参考スクリプトのブラケット自体にはEnd1/End2は基本出てこない（ブラケットは面と寸法で定義）。
  - ただし **Sf1EndElements**（1505、または1501の一部で使用）が重要で、端部の参照要素を与えないと位置が不安定になる。

### 注意事項
- **Surfaces1/Surfaces2は「面ペア」定義であり、型が混在する**
  - `profileX[0]+",FL"` のように **断面の面（FL/WF/WB）** を明示するケース
  - `["PLS","False","False", nx, ny, nz, element]` のように **PLS（平面＋法線＋参照要素）** を与えるケース  
  - 元スクリプトでブラケットを50個以上付ける際、**同じ要素でも「FL指定」で安定する箇所と「PLS指定」が必要な箇所**がある。参考スクリプトではDL02等でPLS指定が頻出。

- **1505は `BracketParams=["200"]` を伴う**
  - 参考スクリプトでは1505のとき必ず `BracketParams` が設定されている（未設定だと形状が決まらないリスク）。
  - さらに `Sf1EndElements` をセットしている（これも踏襲必須）。

- **左右（P/S）展開で数量を稼ぐ場合、MirrorCopy後の要素参照を確実に保持**
  - 参考スクリプト同様、MirrorCopyの戻り値（配列）を変数保持し、その要素をSurfaces/BaseElementに使う必要がある。
  - 片舷だけ付けて終わると「50個以上」に届きづらいので、**Mirror側にも同数を付ける**前提の設計が自然。

- **後処理（BodyDivideByCurves）**
  - 参考スクリプトでは作成したブラケットを `BodyDivideByCurves` で分割している例が多い。  
  - 元スクリプトに同処理を入れるなら、分割用スケッチ平面（例: `skt_pl4`）とカッティング線を用意し、作成したブラケットごとに実行する設計になる（ただし本タスクでは仕様書化のみ）。
