### 対象部材とループ変数
本「元のスクリプト」には **BracketParam / CreateBracket 呼び出しが一切存在せず**、ブラケットは0個です。よって、編集指示「50個以上ブラケットをつけてください」を満たすには、参考スクリプトに見られるブラケット作成パターン（BracketParamの組み立て → CreateBracket → 必要なら BodyDivideByCurves）を、元スクリプト内で生成済みの部材同士の取り合いに対して適用する必要があります。

**50個以上にする実務的内訳（仕様としての量産対象）**
- 上記の **Side FRxx → Deck面（BracketType=1505）** を、FR07/08/10/11/12/14/15 の **左右舷（Mirror含む）** に展開するだけで 7×2 = 14個以上

- さらに Deck.B / Deck.A / Deck.C それぞれにも同様の “壁⇔甲板” / “Sideフレーム⇔甲板” を展開すれば、合計50個は容易に超えます（参考スクリプトでは同一ロジックで20個超＋分割処理を多数実施）。

### ループ構造方針
元スクリプトは「要素を個別に profileN として直列作成」しているため、ブラケット追加は次の2段ループが最も安全です。

- **方針1: Sideフレーム群をリスト化してループ（BracketType=1505）**
  - 対象例（CDP系でDeckに当てたいもの）  
    `side_frames_cdp = [profile16[0], profile12[0], profile44[0], profile111[0], profile52[0], profile113[0], profile3[0]]`  
    さらに Mirror で生成されている相当部材（例: mirror_copied2, mirror_copied17 など）を含めて左右舷分を拡張。
  - 各フレームに対し、  
    - Surfaces1: `PLS ... solid2`（Deck.Dソリッド等、対象Deckに応じて切替）  
    - Surfaces2: `frame + ",FL"`  
    - Sf1EndElements: 甲板側基準要素（例: profile17[0] = Deck.DL05P のような「同一Deckの基準縦通」）  
    を共通で適用して量産。

- **方針2: Wall(DLxx.CDP) と Deck(DLxxP/FP/AP) のペアを辞書化してループ（BracketType=1501）**
  - 例:  
    - Fore: (profile62 ↔ profile61), (profile117 ↔ profile35), (profile100 ↔ profile86), (profile32 ↔ profile17) …  
    - Aft:  (profile75 ↔ profile23), (profile36 ↔ profile35), (profile87 ↔ profile86), (profile72 ↔ profile17) …
  - Surfaces は基本 `[..., ",FL"]` 同士の組み合わせで統一。

### パラメータ依存関係
参考スクリプトから読み取れる「BracketParamの典型依存」は以下。

- **共通**
  - `DefinitionType = 1`
  - `UseSideSheetForPlane = False`
  - `Scallop1Type = 1801`（Params は `"0"` または `"50"` が多い）
  - `Scallop2Type = 0`（未使用）
  - `Sf1DimensionType / Sf2DimensionType`
    - 1501（一般ブラケット）: 1531 を多用し `["200","15"]` や `["250","15"]`、大型は `["800","15"]`
    - 1505（トライアングル/簡略ブラケット）: Sf1DimensionType=1541（`["0","100"]` 等）＋ `BracketParams=["200"]` のような固定長指定

- **Thickness（板厚）**
  - 1501: 接続対象の板厚/形鋼tに寄せて設定されており、例として 8.0mm, 12mm, 9.0mm が出現
  - 1505: Sideフレーム系で 10mm 前後が多い（例: `"9.9999..."`）
  - 元スクリプトでは profile の板厚が明示されている（例: ProfileParams内の t=9,7,12 等）ため、**ブラケット厚は「ベース要素（BaseElement）のt」または「壁/甲板側の板厚10（Thicken）」に合わせる**のが整合的。

- **Sf1EndElements / Sf2EndElements**
  - 参考では、1505で特に **Sf1EndElements を指定してブラケットの端基準を拘束**している。  
    例: `Sf1EndElements=["PLS",... , profile1[0]]`
  - 元スクリプトにおいても、Side→Deck で形状安定化のため **Sf1EndElements を必須**とする方がエラー回避に有利。

- **Surfaces1/Surfaces2 の型**
  - 形鋼の面指定: `",FL"`, `",WF"`, `",WB"`, `",FR"` など
  - PLS参照面指定: `["PLS","False","False", nx, ny, nz, element]` 形式  
    → 元スクリプトで「板（Solid/Sheet）側の法線方向を明示したい」場合に有効。

### 注意事項
- **元スクリプトはブラケット0個**のため、「50個以上」という編集指示は、単純な追記ではなく **“ブラケット生成ブロックそのものの新設”**が必須。
- 参考スクリプトではブラケット作成後に `BodyDivideByCurves` を大量に実行しており、これは主に
  - ブラケット/部材を任意スケッチ線（skt_pl4 の線）で分割し、干渉回避や形状整理を行う
  - 生成物の見た目・部材境界を合わせる
  目的です。元スクリプトでも、ブラケットを50個以上付けると干渉・貫通が増えるため、**必要に応じて分割処理（BodyDivideByCurves）を同じ基準スケッチ平面で適用する設計**が望ましい。
- `Surfaces1/2` は「面ペア」で定義され、参照面の指定方法が2系統（`element+",FL"` 形式 / `["PLS", ... , element]` 形式）ある。  
  取り合いが不安定な箇所（Deckソリッド等）は **PLS形式で法線を固定**した方が失敗しにくい（参考 bracketPram2/4/5/7/15/17/19/21/22 など）。
- MirrorCopy された要素をブラケット対象に含める場合、**ミラー後の要素参照（mirror_copiedX[0]）を BaseElement/Surfaces に正しく入れる**必要がある（元要素を参照すると反対舷に付かない）。
