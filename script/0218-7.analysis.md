### ループ構造方針
参考スクリプト・元スクリプトともに、ブラケットは基本的に **個別生成**（`bracketPramN` を都度作る）で管理している。

ただし仕様化するなら、論理的には以下の“テンプレート＋対象リスト”で整理可能：

- **テンプレートA（Deckソリッド ↔ Side profile の 1505）**
  - 対象：`solid1/2/3/4` と Side系 `profile??[0]`（`,FL`）の組合せ
  - 端基準：`Sf1EndElements` に Deck側の基準Profile（例：Deck.Cは `profile8[0]`、Deck.Bは `profile68[0]`、Deck.Dは `profile17[0]`、Deck.Aは `profile47[0]`）を必ず与える
- **テンプレートB（Profile FL ↔ Profile FL の 1501）**
  - 対象：Fore/Aft/Side/Deck側で作成した profile 同士（`[xx+",FL"]`）
  - 端要素（EndElements）は原則不要（元スクリプトでは未指定）
- **テンプレートC（Flange付き・PLS面指定の特殊1501）**
  - 対象：`profile59` と `profile18` のように、**梁/ガーダ系（1201）部材の別面**を `PLS` 参照で与える
  - `FlangeType`, `FlangeParams`, `Scallop1Params=["50"]`, 大きめ寸法（800）など、別テンプレートで扱う

### パラメータ依存関係
- **BracketType**
  - `1505`：Deckソリッド（Plate相当）と profile（主にSide縦材）の取り合いに多用。`Surfaces1` が `["PLS", ..., solidX]` で与えられる。
  - `1501`：profile同士（FL面同士）や、梁系の面（PLS参照）同士で成立する接続に使用。
- **Thickness（板厚）**
  - Side↔Deck（1505）ではほぼ **10mm**（`9.999999...`）。
  - profile↔profile（1501）では **8mm**（`7.999999...`）が多い。
  - 特殊（#8）は **12mm**。
  - 元スクリプト上は、部材寸法から自動算出している痕跡はなく、**固定値を選定**している。
- **BracketParams（主に高さ/サイズの代表値）**
  - `1505` では `BracketParams=["200"]` が固定。
  - `1501` では `BracketParams` を明示しないケースが多い（参考スクリプトでも1501は未設定が多い）。
- **寸法指定（Sf1/Sf2 Dimension）**
  - `Sf1DimensionType=1541` + `["0","100"]`：Deck側（PLS+solid）基準のオフセット/位置決め。
  - `Sf2DimensionType=1531` + `["200","15"]`：profile FL側のブラケット取り合い寸法。
  - `1501` の profile↔profileでは `Sf1DimensionType=1531`/`Sf2DimensionType=1531` が基本（例：#3, #5, #9, #11, #13, #14）。
- **End要素指定の要否**
  - Deckソリッドを `Surfaces1` に使う `1505` では、参考スクリプト同様に **`Sf1EndElements` の指定が重要**（ブラケットの端基準を確定させ、位置が不定にならないようにしている）。
  - profile↔profile（1501）では、元スクリプトでは **EndElementsを必須としていない**。

### 注意事項
- **Surfaces1/Surfaces2 は「面ペア」で定義**される。特に `Surfaces1=["PLS", ... , vector..., solidX]` 形式は、板（solid）側の参照面を「PLS＋法線ベクトル」で指定しているため、**法線方向（例：0,0,-1）の符号が変わるとブラケットが反転**し得る。
- 1505の多くは `Sf1EndElements` を必ず持つ（参考スクリプトでも同様）。元スクリプトでこの指定を落とすと、Deck上のどの端を基準にするかが曖昧になり配置が破綻しやすい。
- #8 のように `Surfaces1/2` をともに `PLS` 参照で与え、かつ `FlangeType` を使うケースは、通常の「solid＋profile」型とは別物。**同じテンプレで増設しない**。
- 参考スクリプトでは、左右対称配置は `MirrorCopy(...,"PL,Y","")` を多用して漏れを防いでいる。元スクリプトはブラケットに対するミラーが未実装のため、**P/S両舷に必要なブラケットはMirrorCopyを追加する設計**が適切（ただし本タスクは“仕様書作成”なので、実装方針として明記するに留める）。
