- **壁（Sheet）**

  - `extrude_sheet3`: HK.Casing.Wall.Aft（Aft壁：Sheet）

  - `extrude_sheet9`: HK.Casing.Wall.SideS（Side.S壁：Sheet）

- **ブラケットの主な取り付け対象（Profile）**
  - Fore側縦通/ロンジ系（DLxx）: `profile62 (DL01.CDP)`, `profile59 (DL02.*)`, `profile117 (DL03.CDP)`, `profile100 (DL04.CDP)`, `profile32 (DL05.CDP)` など
  - Aft側縦通/ロンジ系（DLxx）: `profile75 (DL01.CDP)`, `profile19 (DL02.CDP)`, `profile36 (DL03.CDP)`, `profile87 (DL04.CDP)`, `profile72 (DL05.CDP)` など
  - Side側フレーム系（FRxx）: `profile16 (FR07.CDP)`, `profile12 (FR08.CDP)`, `profile65 (FR09.CDP/1003)`, `profile44 (FR10.CDP)`, `profile111 (FR11.CDP)`, `profile52 (FR12.CDP)`, `profile43 (FR13.CDP/1003)`, `profile113 (FR14.CDP)`, `profile3 (FR15.CDP)` 等
- **ループ変数方針（推奨）**
  - 元スクリプトは「個別に profile を作成している」ため、ブラケットも **個別定義の列挙**が最も整合する。
  - ただし仕様化では、以下の「グループ単位でのループ」が可能：
    - `DL系(Fore/Aft)`: `[(fore_profile, deck_profile_or_longi_profile), ...]`
    - `FR系(Side)`: `[(side_profile, deck_solid_or_deck_sheet, endElement), ...]`

### ループ構造方針
- **方針A（元スクリプト踏襲：個別定義）**
  - 元スクリプトは `profileXX` が大量に個別生成されているため、ブラケットも `bracketPram1..N` を個別に作成し、対応する `profile` を直接参照するのが最も安全。
- **方針B（仕様上のループ化：グループ配列を作る）**
  - 例：Side FR系（BracketType=1505）のみループ化しやすい  
    - `targets = [profile16[0], profile12[0], profile44[0], profile111[0], profile52[0], profile113[0], profile3[0], ...]`
    - 各要素に対し `Surfaces1=PLS(Deck solid)`, `Surfaces2=[target+",FL"]`, `Sf1EndElements=PLS(..., 縦通/甲板補強 profile)` を共通適用
  - DL系（BracketType=1501）は、接続相手が「同DLのDeck側profile」「別部材（FPなど）」と1対1関係が強いため、ペア配列でループ化  
    - `pairs = [(profile62[0], profile61[0]), (profile72[0], profile17[0]), ...]`

### パラメータ依存関係
- **共通（参考スクリプトから抽出されるブラケットパラメータ規約）**
  - `DefinitionType = 1`
  - `UseSideSheetForPlane = False`
  - `RevSf1/RevSf2/RevSf3 = False` を基本（必要時のみ反転）
  - `Scallop1Type = 1801` が基本、`Scallop1Params` は `["0"]` または `["50"]` が多い
  - 寸法指定は面ごとに `Sf1DimensionType / Sf2DimensionType` をセット
- **BracketType別**
  - **1501（標準三角ブラケット系）**
    - `Thickness`: 接続先の板厚やprofileの板厚に合わせて選択されている傾向  
      - 例: 7.999...（≒8mm）、12mm、8.999...（≒9mm）
    - `Sf1DimensionType=1531` + `Sf1DimensonParams=["200","15"]` が頻出（DL系小ブラケット）
    - 大きいブラケットでは `["800","15"]`（DL02など）
    - `Surfaces1/2` は多くが `profileX[0]+",FL"` のように **部材面（FL/WF/WB）指定**
    - フランジ付きブラケットの場合：
      - `FlangeType=262`
      - `FlangeParams` はケースにより `["100","30","30","30","30","1"]` や `["75","30","30","30","50","1"]`
  - **1505（スロット/片側基準ブラケット系：Side×Deckで多用）**
    - `BracketParams=["200"]`（代表値）を追加
    - `Surfaces1` を `PLS` 指定（Deckのsolidを参照して法線を与える）
    - `Sf1DimensionType=1541` + `Sf1DimensonParams=["0","100"]`（オフセット＋長さ系）を使用
    - `Sf1EndElements` を設定し、端部基準（縦通など）を与えて位置決めしている（参考 bracketPram4/7/15/17/19/21/22）
- **End要素（Sf1EndElements / End1/End2）**
  - ブラケット自体では `End1/End2` は通常設定されず、代わりに  
    - 1505では `Sf1EndElements` により“どこまで/どこから”を拘束する使い方が重要
  - 1501で `Sf1EndElements` が使われるのは、`Sf1DimensionType=1541` のような特殊寸法時（参考 bracketPram3,12）

### 注意事項
- **Surfacesの指定方法を混在させない**
  - `profile+",FL"` のような “部材面名” 指定と、`["PLS", ...]` の “幾何参照面(Plane Surface)” 指定は用途が異なる。参考スクリプトでは **1505はPLSが多く、1501は面名指定が多い**。
- **BaseElementは必ず接続部材（多くはProfile）に置く**
  - 参考スクリプトでは `BaseElement=profileXX[0]` が基本。Deck側solidをBaseにしていない。
- **厚み(Thickness)は接続先の板厚整合**
  - 例として、DL02周りは 12mm、その他は 8〜10mm帯が多い。元スクリプト側で板厚が明示されない場合でも、参考と同じ値（8/10/12）に寄せる。
- **Mold（+/-）の整合**
  - 参考スクリプトでは同じ取り合いでも `Mold="+"` と `Mold="-"` が混在し、左右や基準面方向で反転している。元スクリプトでも、Fore/AftやMoldが異なるprofile（Mold="-"の1201など）では注意が必要。
- **後工程（BodyDivideByCurves）相当の分割**
  - 参考スクリプトはブラケット作成後に `BodyDivideByCurves` で不要部をカットしている箇所が多い。元スクリプトにも同様の分割用スケッチ（`skt_pl4` 等）があるため、ブラケットを作るなら“分割をどの平面で行うか”もセットで検討が必要。
