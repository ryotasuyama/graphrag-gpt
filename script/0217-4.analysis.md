### ループ構造方針
- **(A) 壁DLxx ↔ Deck.D DLxx の一括生成（BracketType=1501）**
  - 対象ペア（例）:
    - Fore: `(profile62 ↔ profile61)`, `(profile32 ↔ profile17)`, `(profile117 ↔ profile35)`, `(profile100 ↔ profile86)`
    - Aft: `(profile75 ↔ profile23)`, `(profile72 ↔ profile17)`, `(profile36 ↔ profile35)`, `(profile87 ↔ profile86)`
  - ループ変数案: `pairs = [(baseElm, sf1, sf2, mold, t), ...]` のテーブル駆動
- **(B) Sideフレーム FRxx ↔ Deck.D（PLS）一括生成（BracketType=1505）**
  - 対象: `profile16, profile12, profile44, profile111, profile52, profile113, profile3` 等
  - `Surfaces1` は `["PLS", False, False, 0, 0, -1, solid2]` 型（参考の `solid1` 相当を `solid2` に置換）
  - `Surfaces2` は `[profileX[0] + ",FL"]`
- **(C) Deck.D 内のFR材 ↔ DL02P（T形）など、フレーム交差部のブラケット（1501）**
  - `Surfaces1` に `WF/WB` を使うケース、`Sf1EndElements` を指定するケースを、個別定義で追加

### パラメータ依存関係
- **BracketTypeの使い分け（参考踏襲）**
  - **1501**: 2つの部材（多くは profile 同士）の **面ペア（FL、WF、WB、または PLS）**で定義する汎用ブラケット
  - **1505**: デッキ面（PLSでSolid参照）と Sideフレーム（profile FL）を結ぶ“デッキ取合い”型。`BracketParams=["200"]` が必須パターン
- **Thickness（板厚）**
  - 参考スクリプトでは概ね以下で固定運用:
    - L形/FB相当の細材: `t ≈ 8.0`（"7.9999..."）
    - Sideフレーム取合い（1505）: `t ≈ 10.0`
    - T形/フランジ付き（DL02関連）: `t = 12`
    - 大型FR（ProfileParamsで板厚9系）: `t ≈ 9.0`
  - 元スクリプトの ProfileParams から厳密算出していないため、**参考と同じ固定値運用**が最も整合的
- **Sf1/Sf2 Dimension**
  - 1501の標準: `Sf1DimensionType=1531`, `Sf1DimensonParams=["200","15"]`（小型）、FRやSideなどで `["250","15"]`、DL02系で `["800","15"]` など
  - 1505の標準: `Sf1DimensionType=1541`, `Sf1DimensonParams=["0","100"]`（デッキ側のオフセット/幅指定系）
- **EndElements（Sf1EndElementsなど）**
  - 参考で必須になっているケース:
    - 1505: `Sf1EndElements` にデッキ縦材（例 `profile1[0]`）を PLS形式で指定し、寸法方向の端基準を与える
    - 1501（WF指定）: `Sf1EndElements=[profile4[1]+",FR"]` のように、FR端を基準にする指定が入る場合がある（FR09P/FR13P系）
- **FlangeType / FlangeParams**
  - DL02（ProfileType=1201）絡みのブラケットで多用（参考 bracketPram2/5）
  - `FlangeType=262`, `FlangeParams=["100","30","30","30","30","1"]` もしくは小さめ `["75","30","30","30","50","1"]` が確認できる

### 注意事項
- **Surfaces指定は「面ペア」であることが最重要**
  - profile系は `",FL"` が基本だが、T形/フランジ付きでは `WF/WB` が必要になることがある（参考 bracketPram3/8/12/16）。
- **PLS形式のSurfaces**
  - 参考では `["PLS","False","False", nx, ny, nz, element]` という形で**法線ベクトル付きの参照面**を作っている。  
  - 1505（デッキPLS）やDL02系（PLS×PLS）でこの形式が出るため、元スクリプトでも同等形式で統一するのが安全。
- **BaseElementは必ず壁側（または主部材側）のprofileを採用**
  - 参考スクリプトは「壁側縦材＝BaseElement」になっているケースが多い（Fore/Aft DL材、Side FR材）。
- **後工程の BodyDivideByCurves はブラケット形状調整目的**
  - 参考では作成したブラケットを `skt_pl4` の線で分割している例がある。元スクリプトで同等の外形制御が必要なら、ブラケット作成後に同様の分割処理を追加する前提で計画する。
