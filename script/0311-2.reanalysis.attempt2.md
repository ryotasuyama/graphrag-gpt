#### 修正・除外した候補
- **除外: 前回 #24**
  - `[profile77[0]+",FL"]` ↔ `[profile41[0]+",FL"]`
  - `HK.Casing.Wall.Side.FR13.ABP` ↔ `HK.Casing.Deck.A.FR13P`

#### 除外理由
- 実行時に `CreateBracket` で COM 例外が発生しているため、候補として不成立
- `profile77` は `FR13.ABP`、`profile41` は `Deck.A.FR13P` で、名称上は FR 一致でも、**参考スクリプト内にこの組合せの 1501 ブラケット成立実績がない**
- 一方で FR13 系の成立実績は以下に限定される
  - `profile43 ↔ profile42`（Side.FR13.CDP ↔ Deck.D.FR13P）
  - `profile43 ↔ profile78`（Side.FR13.CDP ↔ Deck.B.FR13P）
  - `profile101 ↔ profile110`（Side.FR13.BCP ↔ Deck.C.FR13P）
- したがって、**FR13 だから自動的に Deck.A.FR13P に接続可能とはみなせない**
- 特に `ABP` 側部材は、幾何拘束上 `Deck.A.FR13P` との FL-FL の 1501 が安定しない可能性が高い

#### 代替提案
- 代替は新規追加ではなく、**すでに成功実績のある FR13 系候補を維持することで候補数要件を満たす**
  - `#19` Side.FR13.CDP ↔ Deck.D.FR13P
  - `#25` Side.FR13.BCP ↔ Deck.C.FR13P
  - `#27` Side.FR13.CDP ↔ Deck.B.FR13P
- これにより、失敗候補を除外しても **42候補** を維持でき、**30個以上**の要件を十分満たす

### ループ構造方針

前回方針を維持しつつ、**失敗した Side.FR13.ABP ↔ Deck.A.FR13P のような未実績 1501 直結を禁止**します。

#### ループ1: Wall Fore/Aft の DL 系ブラケット
- Wall を `Fore` と `Aft` に分離
- DL番号単位で候補化
- Deck候補は Deck A/B/C/D の DL プロファイル
- 必須チェック
  - `Wall.Fore.DLxx.*` → `Deck.*.DLxx.F / FP`
  - `Wall.Aft.DLxx.*` → `Deck.*.DLxx.A / AP`
- この系列は前回成功候補をそのまま維持

#### ループ2: Side Wall ↔ Deck 板ブラケット
- Side.CDP 系で Deck.D に届くものを対象
- 形式固定
  - `BracketType=1505`
  - `Surfaces1=["PLS",...,solid2]`
  - `Surfaces2=[side_profile+",FL"]`
  - `Sf1EndElements=["PLS",... ,profile17[0]]`
- 参考スクリプトの成立実績が最も多く、安全性が高い

#### ループ3: Side Wall ↔ Deck FR プロファイルブラケット
- 直接 1501 接続は**参考スクリプトに成立実績のある組のみ**
- 採用可
  - `profile65 ↔ profile64`
  - `profile43 ↔ profile42`
  - `profile57 ↔ profile90`
  - `profile95 ↔ profile96`
  - `profile106 ↔ profile112`
  - `profile101 ↔ profile110`
  - `profile43 ↔ profile78`
- 採用不可
  - `profile77 ↔ profile41`
  - FR一致のみを根拠にした未実績接続
  - `ABP ↔ Deck.A.FR` のように参考スクリプトでブラケット成立が確認できない組

### パラメータ依存関係

#### 1. BracketType の使い分け
- `1501`
  - Wall.DL ↔ Deck.DL
  - Side.FR ↔ Deck.FR
  - ただし Side↔Deck.FR は**成立実績のある組だけ**
- `1505`
  - Side.FR ↔ Deck 板面
  - `BracketParams=["200"]`
  - `Sf1EndElements` 必須

#### 2. Thickness
- `1002` 同士の DL 接続: `8`
- `1201` を含む DL02 系: `12`
- Side↔Deck板 `1505`: `10`
- `1003` を含む FR 接続: `8～9`
- 今回失敗した `profile77 ↔ profile41` は厚み値自体より、**接続型の不整合**が主因と判断

#### 3. Surface型
- 通常 profile 接続
  - `[profileX[0]+",FL"]`
- Deck FR 支持ブラケット
  - `[profile18[0]+",WF"]`
  - `Sf1EndElements=[profile18[1]+",FR"]`
- 板面基準
  - `["PLS","False","False","0","-0","-1",solid2]`
- `1201` 系では `[0] / [1]` の使い分け必須

#### 4. DimensionType
- 一般 profile 接続
  - `1531 / ["200","15"]`
- FR大物
  - `1531 / ["250","15"]` または `["300","15"]`
- 板面基準 1505
  - `1541 / ["0","100"]`
  - `Sf1EndElements` 必須
- DL02 の大型接続
  - `1531 / ["800","15"]`

#### 5. 今回の失敗要因の整理
- `1501` の FL-FL 接続であっても、**参考スクリプトに同型成立実績がない組合せは危険**
- `Side.FR13.ABP` は Deck.A.FR13P と名称対応していても、**幾何面の取り方や交差位置が 1501 の前提に合わない**可能性がある
- よって、Side FR 系では
  - **CDP↔Deck.D/Deck.B**
  - **BCP↔Deck.C**
  - **FR09/FR13 の成立実績ペア**
  を優先し、未実績ペアは除外する

### 注意事項

- **Fore/Aft 整合ルールは必須**
  - `Wall.Fore.DLxx.*` → `Deck.*.DLxx.F / FP` のみ
  - `Wall.Aft.DLxx.*` → `Deck.*.DLxx.A / AP` のみ
- 片側方向の Deck プロファイルしかない場合、反対側 Wall 候補は除外する

- **Side Wall は方向制約なし**
  - ただし Deck側相手の存在確認は必須
  - さらに、**成立実績のある接続型かどうか**を必ず確認すること

- **成功済み bracket 1〜23 はそのまま維持**
  - 今回の修正対象は `bracketPram24` のみ

- **失敗候補 `profile77 ↔ profile41` は除外**
  - `HK.Casing.Wall.Side.FR13.ABP` ↔ `HK.Casing.Deck.A.FR13P`
  - 理由は COM 例外発生、および参考スクリプト上の成立実績不足

- **FR13 系の代替は既存成立候補で吸収**
  - `Side.FR13.CDP ↔ Deck.D.FR13P`
  - `Side.FR13.BCP ↔ Deck.C.FR13P`
  - `Side.FR13.CDP ↔ Deck.B.FR13P`
- 新たな未実績候補を無理に補わず、**成功実績ベースで候補数要件を確保**する

- **1505 型では Sf1EndElements を省略しない**
  - 参考スクリプト準拠で必須
  - 位置決め安定性と COM 例外回避の観点で重要

- **候補数要件は維持**
