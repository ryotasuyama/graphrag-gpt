### ループ構造方針
- **方針1：Side FR系（1505）の一括生成**
  - 対象を `[(profile16,"FR07"), (profile12,"FR08"), (profile44,"FR10"), (profile111,"FR11"), (profile52,"FR12"), (profile113,"FR14"), (profile3,"FR15")]` のような配列にまとめ、順番に `CreateBracketParam` を作成して `CreateBracket`。
  - `Surfaces1` は **Deck側のソリッド（例：`solid2`=Deck.D）をPLSで指定**するパターンを踏襲。
  - `Sf1DimensionType=1541` と `Sf1EndElements` を、参考同様「端を決める要素（例：デッキ側の縦通 `profile60[0]` 相当）」で統一して漏れを防ぐ。
- **方針2：Fore/Aft DL系（1501）の一括生成**
  - Fore: `[(profile62, profile61), (profile59, profile18 or その関連要素), (profile117, profile35), (profile100, profile86), (profile32, profile17)]`
  - Aft: `[(profile75, profile23), (profile19, profile18 or その関連要素), (profile36, profile35), (profile87, profile86), (profile72, profile17)]`
  - 原則は **FL–FL接続（Surfaces1=[base+",FL"], Surfaces2=[other+",FL"]）**。
  - ただし `ProfileType=1201` が絡むDL02は、参考のように **PLS指定＋FlangeType=262** の派生パターンを優先（形状破綻回避）。

### パラメータ依存関係
- **共通**
  - `bracketPram.DefinitionType = 1`
  - `MaterialName = "SS400"`
  - `UseSideSheetForPlane = False`
  - `Scallop1Type = 1801`
  - `RevSf1/RevSf2/RevSf3` は原則 False（ただしMold符号により反転が必要なケースあり）
- **BracketType選定ロジック（参考スクリプト踏襲）**
  - **1501**: Profile同士（例：壁CDPのFLと、デッキ縦通/FP/APのFL）の“基本ブラケット”
    - `Sf1DimensionType = 1531`, `Sf1DimensonParams=["200","15"]`（小型）または `["250","15"]`（FR系1003のとき）や `["300","15"]`（大きめ）
    - `Sf2DimensionType = 1531`, `Sf2DimensonParams` 同等
  - **1505**: デッキ板（ソリッド/PLS）とSideのCDP（FL）を結ぶ“片持ち/シート基準”ブラケット
    - `BracketParams=["200"]`（参考スクリプトで固定的に使用）
    - `Sf1DimensionType = 1541`, `Sf1DimensonParams=["0","100"]`（デッキ面からのオフセット/投影系）
    - `Sf1EndElements` を指定して位置決め（参考スクリプトでは `["PLS",... , profile1[0]]` のように PLS で端要素を与える）
- **Thickness（板厚）**
  - 参考スクリプトの傾向：
    - 一般的な小型ブラケット（1501, DL01/DL03/DL04/DL05周り）: **約8mm（"7.9999..."）**
    - Side FR系の1505: **約10mm（"9.9999..."）**
    - DL02（1201が絡む）: **12mm**
    - FR09P/FR13Pのような1003（大型）で1501: **約9mm（"8.9999..."）** が使われるケースあり
  - 元スクリプト側でも、**接続相手が1201なら12mm**、それ以外は **1501=8mm / 1505=10mm / 大型FR(1003)絡み=9mm** を基本割当とするのが整合的。
- **Flange付ブラケット（参考スクリプトのDL02系）**
  - `FlangeType=262`
  - `FlangeParams=["100","30","30","30","30","1"]` や、FR大型では `["75","30","30","30","50","1"]`
  - `RevFlange=False`
  - DL02（Fore/Aft）に関してはこのフランジ付き仕様が再現されているため、同様の依存関係がある。

### 注意事項
- **Surfaces1 / Surfaces2 は“面ペア”で定義され、要素種別により指定方法が変わる**
  - Profile面: `element + ",FL"` / `",WF"` / `",WB"` など
  - 板/ソリッド面: 参考では `["PLS","False","False", nx, ny, nz, solid]` の形式で **PLS + 法線ベクトル**指定が混在
  - DL02のような例外ケースでは、Profileの直接`",FL"`指定ではなく、**PLS指定の方が安定**（参考 bracketPram2/5）
- **Sf1EndElements / Sf2EndElements を使うブラケットがある**
  - 参考スクリプトでは、1501でも `Sf1EndElements=[profile4[1]+",FR"]` のように端要素を指定し、ブラケットの取り付き位置を拘束している（bracketPram3/12）。
  - 1505では `Sf1EndElements=["PLS",..., profile1[0]]` を使うことで、デッキ面上の“端”を確実に決めている（bracketPram4/7/15/17/19/21/22）。
- **Mold（"+" / "-"）は相手部材と左右・表裏で変化**
  - Fore側（ReverseDir=Trueが多い）とAft側（ReverseDir=Falseが多い）で、ブラケットMold符号を合わせないと裏返りやすい。
  - 参考ではDL02はMold="-"、他はMold="+"が多いという傾向がある。
- **後処理（BodyDivideByCurves）**
  - 参考スクリプトは、生成したブラケットを `BodyDivideByCurves` で分割しており、ブラケット干渉や形状整理をしている。
  - 元スクリプトには該当処理がないため、ブラケットを追加する場合、**同じ分割スケッチ平面（例：`skt_pl4`）が存在するなら分割を検討**しないと、モデルが一体化・干渉しやすい。
