## ブラケット配置分析

### 対象部材とループ変数
- **対象となる“ブラケット母材（BaseElement）候補”**  
  元スクリプト内で `CreateProfile(...)` により生成される **profile系要素（profile1〜profile123、および MirrorCopy で複製された profile）** が主対象。  
  ただし、参考スクリプトの実装傾向から、ブラケットは次の2系統に大別できる。

- **系統A：Profile × Profile（主に FL/WF/WB 面どうし）**
  - 例：甲板側の梁/縦通材（ProfileType=1002/1003/1201）と、隔壁/壁側の同種材（ProfileType=1002/1003/1201）を **FL面ペア** で接続
  - 典型：`Surfaces1=[profileX[0]+",FL"]` と `Surfaces2=[profileY[0]+",FL"]`
  - 参考スクリプトでは BracketType=1501 が多い

- **系統B：PLS（平面定義）× Profile（FL面）**
  - 例：**Deck solid（solid1/solid2/solid3/solid4 等）** を `PLS ... solid` で平面化し、それと profile の FL を接続
  - 典型：  
    - `Surfaces1=["PLS","False","False","0","-0","-1", solidDeck]`（甲板面法線 -Z を想定）  
    - `Surfaces2=[profileX[0]+",FL"]`
  - 参考スクリプトでは BracketType=1505（BracketParams=["200"]）が多い  
  - さらに `Sf1EndElements` で **基準となる端要素（例：profile1[0]）** を入れて取り付け端側を拘束する実装がある

- **ループ化の単位（想定）**
  - 参考スクリプトは個別に bracketPram を都度作っているが、設計上は以下のグループでループ可能：
    - **Side 壁のFR系（FR07/08/10/11/12/14/15）**：`profile16, profile12, profile44, profile111, profile52, profile113, profile3` など（※名称に `HK.Casing.Wall.Side.FRxx.*` を含む）
    - **Fore/Aft 壁のDL系（DL01/02/03/04/05）**：`profile62/26/69/104/94/100/53/25/117/36/87/72/66/67/32...` など（※名称に `HK.Casing.Wall.(Fore|Aft).DLxx.*` を含む）
    - **Deck側のDL系 FP/PP**：`profile71/85/114/68/30/40/50/47/74/54/28/8/33/93...` など（※名称に `HK.Casing.Deck.(A|B|C|D).DLxx.*` を含む）

### ループ構造方針
- **方針1：命名規則ベースのグルーピング**
  - profile を「Wall.Fore」「Wall.Aft」「Wall.Side」「Deck.A〜D」等の用途別に辞書化し、  
    - `DL01/02/03/04/05`  
    - `FR07/08/09/10/11/12/13/14/15`  
    のキーで対応付けしてからブラケット生成を行う。
- **方針2：Side（FR系）は 1505 テンプレで一括**
  - `HK.Casing.Wall.Side.FRxx.CDP`（または同等のCDP側部材）を列挙し、
  - `Surfaces1` を **PLS + solidDeck**（-Z法線）に固定し、
  - `Surfaces2` を `[profileX[0]+",FL"]` にして BracketType=1505 を量産する。
- **方針3：Fore/Aft（DL系）は 1501 テンプレで一括**
  - `Wall.*.DLxx.CDP` と `Deck.*.DLxxP/FP/AP` の組を列挙し、
  - **FL×FL**で BracketType=1501 を量産する。
- **方針4：個別例外（DL02などフランジ付き）**
  - 参考スクリプトにあるように、DL02（ProfileType=1201）絡みは  
    - `FlangeType=262`  
    - `FlangeParams=[...]`  
    を伴うテンプレを別枠にする（PLSベクトル指定で面を作る場合がある）。

### パラメータ依存関係
参考スクリプトから読み取れる「ブラケットパラメータ決定ロジック（実質テンプレ）」は以下。

- **共通（DefinitionType=1）**
  - `bracketPram.DefinitionType = 1`
  - `bracketPram.MaterialName = "SS400"`
  - `bracketPram.UseSideSheetForPlane = False`
  - `bracketPram.Mold` は接続向きに依存（多くは `+`、DL02系で `-` が出る）

- **Thickness（板厚）**
  - profileサイズや型から“計算”している形跡は薄く、参考ではほぼ固定値テンプレ：
    - BracketType=1501 で `Thickness="7.9999..."`（=8相当）または `"12"`、`"8.9999..."`（=9相当）
    - BracketType=1505 で `Thickness="9.9999..."`（=10相当）が多い
  - よって仕様としては **接続先 profile の板厚系列に合わせて固定割当**が妥当
    - Side FRxx（150×90×t9系）→ Bracket t≈10
    - 小型アングル/FB（125×75×t7系）→ Bracket t≈8
    - T/FBフランジ付き（1201, t12/14系）→ Bracket t=12 など

- **BracketType**
  - **1501**：Profile×Profile（FL同士、またはWF/WB×FL）で多用
  - **1505**：Deck（PLS化）× Side FR材（FL）で多用  
    - 参考では `BracketParams=["200"]` をほぼセットで使用

- **寸法（DimensionType / Params）**
  - 1501で頻出：  
    - `Sf1DimensionType=1531, Sf1DimensonParams=["200","15"]`  
    - `Sf2DimensionType=1531, Sf2DimensonParams=["200","15"]`  
    - 大きいブラケットでは `["250","15"]` や `["300","15"]`、DL02では `["800","15"]`
  - 1505で頻出：  
    - `Sf1DimensionType=1541, Sf1DimensonParams=["0","100"]`  
    - `Sf2DimensionType=1531, Sf2DimensonParams=["200","15"]`  
    - 加えて `Sf1EndElements=...` を使い、端側要素で位置決めする

- **End1/End2 指定（Sf1EndElements / Sf2EndElements）**
  - Bracket自体には End1/End2 というより、**面の端基準**として `Sf1EndElements` が使われるケースがある（特に1505）。
  - 1501の一般形（FL×FL）では省略されることが多いが、WF/WB面を使う場合は参考で `Sf1EndElements=[...,FR]` が入る例がある。

### 注意事項
- **Surfaces は “面ペア” 定義であり、型が混在する**
  - Profile面指定：`"<element>,FL"` / `WF` / `WB` / `FR`
  - PLS指定：`["PLS","False","False", nx, ny, nz, <solid or element>]`  
    ここで nx,ny,nz は文字列で与えられており、参考では **デッキ面 -Z（0,0,-1）** を多用。
- **BracketType=1505 は BracketParams が必須級**
  - 参考では `BracketParams=["200"]` をセットしている。元スクリプトでも 1505 を採用する場合は同様に踏襲するのが安全。
- **DL02（ProfileType=1201）絡みはフランジパラメータが必要になる可能性**
  - 参考では `FlangeType=262` と `FlangeParams=[...]` を追加している。  
  - 元スクリプトの DL02 には 1201 が多く、単純な1501テンプレだけだと形状不整合が出るリスクがあるため、**DL02専用テンプレ**を別途定義する前提で仕様化する。
- **ミラーコピー部材にもブラケットが必要**
  - 参考スクリプトは左右（PL,Y）対称要素を多数作成しており、ブラケットも同様に左右発生する前提。  
  - 仕様としては「元要素側にブラケットを付けたら、MirrorCopy側にも同条件で付ける」方針を取る。
- **BodyDivideByCurves（分割）との順序**
  - 参考ではブラケット生成後に `BodyDivideByCurves` を実施している箇所がある。  
  - 元スクリプトでは分割処理が未投入なので、ブラケット配置後に同様の分割が必要な場合がある（ただし本タスクは“配置仕様”作成のため、順序の注意点として記載）。
