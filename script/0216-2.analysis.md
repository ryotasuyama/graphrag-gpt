## ブラケット配置分析

### 対象部材とループ変数
- **Plate（面材 / Sheet）系**
  - `extrude_sheet1` : HK.Casing.Deck.B（Deck B の板）
  - `extrude_sheet2` : HK.Casing.Deck.A（Deck A の板）
  - `extrude_sheet4` : HK.Casing.Deck.C（Deck C の板）
  - `extrude_sheet5` : HK.Casing.Deck.D（Deck D の板）
  - `extrude_sheet7` : HK.General.Deck.UpperDeck（上甲板）
  - `extrude_sheet3` : HK.Casing.Wall.Aft（ケーシング後壁）
  - `extrude_sheet8` : HK.Casing.Wall.Fore（ケーシング前壁）
  - `extrude_sheet6` : HK.Casing.Wall.SideP（ケーシング側壁P）
  - `extrude_sheet9` : HK.Casing.Wall.SideS（ケーシング側壁S）

- **Stiffener / Profile（補強材）系（代表）**
  - 多数の `profileN` が存在し、`ProfileType=1002/1003/1201/1007` などが混在。
    - **1002/1003**: 山形鋼系（Angle相当の断面パラメータ）
    - **1201**: T形/FB+Flange系（FlangeName を伴う）
    - **1007**: FB（フラットバー）系（`ProfileParams=["150","12"]` など）
  - 取付先の面として使いやすいのは、主に以下の「壁/甲板に取り付く補強」：
    - Aft壁取付：`ProfilePramX.AddAttachSurfaces(extrude_sheet3)` のもの（例：`profile1, profile2, profile4, profile7, profile9, profile11, profile14, profile19, profile75, profile83, profile87, profile115, profile119, profile120, profile121, profile123` 等）
    - Fore壁取付：`AddAttachSurfaces(extrude_sheet8)` のもの（例：`profile6, profile13, profile25, profile26, profile39, profile46, profile48, profile53, profile59, profile62, profile67, profile69, profile73, profile84, profile94, profile97, profile98, profile104, profile107, profile117` 等）
    - Side壁取付：`AddAttachSurfaces(extrude_sheet6)` のもの（例：`profile3, profile5, profile10, profile12, profile16, profile20, profile21, profile24, profile34, profile37, profile43, profile44, profile45, profile51, profile52, profile55, profile57, profile58, profile65, profile76, profile77, profile79, profile82, profile89, profile91, profile92, profile95, profile101, profile102, profile103, profile106, profile111, profile113, profile116, profile118, profile122` 等）
  - ミラーコピーされた部材（`mirror_copied*`）も多く、ブラケットは**P/S両舷で同条件**になりやすい。

> 重要：元スクリプトにはブラケット生成API（BracketParam/CreateBracket等）が一切登場しないため、どの部材端部にブラケットを付けるかは「既存の取合い（End1/End2で接続している要素）」を根拠に仕様化するのが妥当。

### ループ構造方針
- **方針A：End1/End2で参照している要素ペアを起点に列挙する（推奨）**
  - 元スクリプトは `ProfileParam.AddEnd1Elements(...)` / `AddEnd2Elements(...)` で接続先が明示されているため、ブラケット仕様もここから決めるのが自然。
  - ループ対象：
    - 「AttachSurfaces が壁（extrude_sheet3/6/8/9）で、End1/End2 が甲板（extrude_sheet1/2/4/5/7）である profile」
    - 「AttachSurfaces が甲板で、End1/End2 が壁である profile（例：profile17, profile18 等）」
  - ループ変数例（概念）：
    - `targets_wall = [profile1[0], profile2[0], profile4[0], ...]`
    - `targets_side = [profile3[0], profile5[0], ...]`
    - 各 `p` について `end_elems = (pのEnd1Elements, pのEnd2Elements)` を2回処理

- **方針B：主要壁×主要甲板の交線ごとに一括配置**
  - 例：Aft壁×DeckB、Aft壁×DeckA、Fore壁×DeckB、SideP壁×DeckC…などの「面ペア」を先に定義し、その交線付近にある補強材端部へ付ける。
  - ただし、このスクリプトだけでは「交線近傍判定（距離、投影、交点）」のロジックが無く、実装難易度が上がるため仕様としては補助的。

### パラメータ依存関係
- **寸法決定に使える情報（元スクリプトから確実に取れるもの）**
  - プロファイル断面寸法：
    - 1002/1003：`ProfileParams` に脚長や板厚相当が入っている（例：`["125","75","7","10","5"]`、`["200","90","9","14","14","7"]` 等）
    - 1201：`ProfileParams` に web/flange 相当（例：`["150","12","388","10"]`）
    - 1007：FB寸法（例：`["150","12"]`）
  - 板厚：
    - 多くの板は `CreateThicken(...,"10",...)` で **板厚10mm** が繰り返し使われている（Deck/Wall系 solid の thickening）。
- **ブラケット Height/Width/Thickness の考え方（仕様案）**
  - Thickness（ブラケット板厚）：
    - 基本：取付先板厚（多くが10mm）に合わせ **t=10**、または補強材板厚（7〜11mm）に合わせて **t=max(plate_t, stiff_t)**。
  - Height / Width：
    - 壁-甲板コーナー：補強材の脚長（例：L125×75 なら 75〜125）を基準に、余長を加えて **H=脚長側+α、W=脚長側+α**（α=25〜50mm程度）というルールが適用しやすい。
    - 1201/T形系：web高さ・flange幅（`ProfileParams` の大きい寸法側）を基準に設定。
- **End1/End2 の指定要否**
  - ブラケットは一般に「どの端部側に付くか」が重要。
  - 本モデルでは profile が End1/End2 で明確に deck/wall を参照しているため、ブラケットも
    - **End1側ブラケット（Surfaceペア：AttachSurface × End1Element）**
    - **End2側ブラケット（Surfaceペア：AttachSurface × End2Element）**
    の2つを基本単位として仕様化するのが整合的。

### 注意事項
- **元スクリプト内にブラケットAPI呼び出しが存在しない**
  - そのため、ブラケットの **BracketType番号、Orientation、参照要素（BaseElement）選定**は、EvoShip標準の「Bracket作成API仕様」または別途参考スクリプトが必要。
- **Surfacesは“面ペア”で定義する必要がある**
  - ブラケットは Surfaces1 / Surfaces2 の組で定義される前提だが、元スクリプトは面ID（PLS/FL/WF等）を直接扱っていない。
  - 仕様としては「Plate面（extrude_sheetX）× Profile面（profileY）」、または「Plate面×Plate面（壁×甲板）」のどちらでブラケットを切るかを先に統一する必要がある。
- **ミラーコピー部材の取り扱い**
  - `MirrorCopy(...,"PL,Y")` が非常に多い。ブラケットも同様にミラー配置するか、P側で作ってS側へミラーする運用が合理的。
- **EndTypeが1102/1103/1111/1113など多様**
  - 端部形状（スニップ、トリム、特殊カット）が異なる可能性が高い。
  - ブラケット干渉回避のため、EndType=1111/1113 を持つ部材周辺はブラケット形状（Type）を分ける、またはブラケット側に逃げ（スカラップ/カット）条件が必要になる可能性がある。
- **板同士をブーリアンでソリッド化している**
  - `solid*` と `extrude*` による積/削除が多く、最終形状は sheet のみではなく solid 側の加工結果に依存する。
  - ブラケット作成参照は「sheet（基準面）」か「solid（最終形状）」かを統一しないと、面参照の安定性が落ちる可能性がある。
