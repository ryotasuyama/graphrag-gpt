## ブラケット配置分析

### 対象部材とループ変数
- **対象モデル（元スクリプト）内でブラケットを付けるべき主要部材群（Bracketの接続候補）**
  - **Deck系 Sheet / Solid**
    - `extrude_sheet1`: HK.Casing.Deck.B（Deck.B 板）
    - `extrude_sheet2`: HK.Casing.Deck.A（Deck.A 板）
    - `extrude_sheet4`: HK.Casing.Deck.C（Deck.C 板）
    - `extrude_sheet5`: HK.Casing.Deck.D（Deck.D 板）
    - `solid1`: HK.Casing.Deck.C（Deck.C の厚み付けソリッド）
    - `solid2`: HK.Casing.Deck.D（Deck.D の厚み付けソリッド）
    - `solid3`: HK.Casing.Deck.B（Deck.B の厚み付けソリッド）
    - `solid4`: HK.Casing.Deck.A（Deck.A の厚み付けソリッド）
  - **Wall系 Sheet / Solid**
    - `extrude_sheet3`: HK.Casing.Wall.Aft（Aft 壁板）
    - `extrude_sheet8`: HK.Casing.Wall.Fore（Fore 壁板）
    - `extrude_sheet6`: HK.Casing.Wall.SideP（側壁P）
    - `extrude_sheet9`: HK.Casing.Wall.SideS（側壁S）
    - `solid5`: HK.Casing.Wall.Aft.CD（Aft壁の厚み＋巻き込み）
    - `solid10`: HK.Casing.Wall.Fore.CD（Fore壁の厚み＋巻き込み）
    - `solid8/solid12/solid15/solid16` 等：側壁の OAP/CDP/BCP/ABP 系ソリッド
  - **Profile（補強材）群（ブラケットのBaseElement/Surfacesとして直接使える）**
    - 代表例：
      - Fore側：`profile62`(Fore DL01.CDP), `profile100`(Fore DL04.CDP), `profile32`(Fore DL05.CDP), `profile117`(Fore DL03.CDP), `profile59`(Fore DL02.CDP) など
      - Aft側：`profile75`(Aft DL01.CDP), `profile87`(Aft DL04.CDP), `profile72`(Aft DL05.CDP), `profile36`(Aft DL03.CDP), `profile19`(Aft DL02.CDP) など
      - Side側（FR系）：`profile16`(FR07.CDP), `profile12`(FR08.CDP), `profile111`(FR11.CDP), `profile52`(FR12.CDP), `profile44`(FR10.CDP), `profile113`(FR14.CDP), `profile3`(FR15.CDP), `profile43`(FR13.CDP), `profile65`(FR09.CDP) など
      - Deck内縦通材：`profile17`(Deck.D DL05P), `profile18`(Deck.D DL02P), `profile23`(Deck.D DL01.AP), `profile35`(Deck.D DL03P), `profile86`(Deck.D DL04P), `profile33`(Deck.D DL00.A) など
- **ループ変数（量産配置の基本方針）**
  - 参考スクリプトは「FRxxごとに profile を作り、同じテンプレの BracketParam を繰り返し作る」構造が中心。
  - 元スクリプトも `profile16/profile12/profile111/profile52/profile44/profile113/profile3` 等、FR（フレーム位置）に相当する複数 profile が生成済みで、これらを**配列化して反復（ループ）**するのが最短経路。
  - さらに Fore/Aft の DL（デッキライン）相当 `profile62/profile100/profile32/profile117/profile59`、`profile75/profile87/profile72/profile36/profile19` も同様に反復候補。

### ループ構造方針
- **最重要：参考スクリプト同様に「一つのテンプレのBracketParamを、対象部材セットを変えて繰り返す」ことで漏れなく配置する。**
- 元スクリプトで「既に存在する複数FR部材」を配列化し、**同一ロジックで BracketType=1505 を量産**するのが確実。
  - 例：Side FR の CDP群（デッキに到達しているもの）
    - `profile16`(FR07.CDP), `profile12`(FR08.CDP), `profile44`(FR10.CDP), `profile111`(FR11.CDP), `profile52`(FR12.CDP), `profile113`(FR14.CDP), `profile3`(FR15.CDP)
  - これらに対し、参考の bracket4/7/15/17/21/22 と同形：
    - Surfaces1: `["PLS",... , solid(Deck)]`（デッキ側ソリッドを基準面）
    - Surfaces2: `[profileFR[0]+",FL"]`
    - Sf1EndElements: `["PLS",... , profileDeckRef]`（デッキ側の参照profile：参考では `profile1[0]` 等）
- 次に「1501（FL–FL）」系を、Fore/Aft/DL/Deck縦通材の組で増やす。
  - Deck側縦通材：`profile17(DL05P)`, `profile18(DL02P)`, `profile23(DL01.AP)`, `profile35(DL03P)`, `profile86(DL04P)`, `profile33(DL00.A)`
  - Wall側：Fore `profile62(DL01.CDP)`, `profile32(DL05.CDP)`, `profile117(DL03.CDP)`, `profile100(DL04.CDP)`, `profile59(DL02.CDP)`／Aft `profile75(DL01.CDP)`, `profile72(DL05.CDP)`, `profile36(DL03.CDP)`, `profile87(DL04.CDP)`, `profile19(DL02.CDP)`
  - これらを「同一DL番号同士」でペアリング（例：DL05: `profile32`–`profile17`, DL04: `profile100`–`profile86` 等）し、さらに左右ミラー側（`mirror_copied***`で生成済みの対称材）にも同様に付ければ容易に50超。
- **数量計画の目安（設計上のカウント）**
  - Side FR（7本）× Deck基準 1505（左右2舷に展開できる場合は×2）→ 14個
  - Fore DL（5本程度）× Deck DL（5本程度）の同番号ペア（左右展開含む）→ 10〜20個
  - Aft DL（5本程度）× Deck DL（5本程度）の同番号ペア（左右展開含む）→ 10〜20個
  - これだけで**合計50個以上が現実的**（実装では「存在するprofileのみ」を対象にするフィルタが必要）

### パラメータ依存関係
参考スクリプトから読み取れる、ブラケット作成パラメータの実運用ロジック（元スクリプトへ踏襲すべき仕様）：

- **共通基本**
  - `DefinitionType=1`
  - `UseSideSheetForPlane=False`
  - `MaterialName="SS400"`
  - `RevSf1/RevSf2/RevSf3` は基本 `False`
- **BracketType と主要パラメータ**
  - **BracketType=1501（汎用ブラケット）**
    - `Thickness` は接続先の板厚・部材厚に合わせて固定値で入っている（例：7.999..., 12, 8.999...）。
    - `Scallop1Type=1801` が多い。`Scallop1Params` は `"0"` または `"50"`。
    - 寸法指定は `Sf1DimensionType=1531` + `Sf1DimensonParams=["200","15"]` のような「長さ＋オフセット」指定が基本。
    - Flange（折り）を持たせる場合：`FlangeType=262` + `FlangeParams=[...]`（参考: bracket2/3/5/12）
  - **BracketType=1505（片側板基準のブラケット）**
    - `BracketParams=["200"]` が付与される（参考: bracket4/7/15/17/19/21/22）。
    - `Surfaces1` が `["PLS", ..., solidDeck]` の形式（板/ソリッド面を基準）になりやすい。
    - `Sf1DimensionType=1541` + `Sf1DimensonParams=["0","100"]` の「端基準距離」系が多い。
    - `Sf1EndElements` を **PLS形式で指定**し、デッキ側の終端参照を与える（例：`["PLS","False","False","-0","-1","-0",profile1[0]]`）
- **Surfaces指定の型（重要）**
  - 参考スクリプトでは Surfaces は大別して2形態：
    1) **要素名 + ",FL/WF/WB/FR"**（Profileの面記号）
    2) **"PLS"配列形式**（法線ベクトルや参照要素を含む）  
       例：`["PLS","False","False","0","-0","-1",solid1]`
- **End1/End2 指定**
  - BracketParam自体には End1/End2 よりも、`Sf1EndElements` のような「片側面の終端拘束」を入れている例がある（特に1501のWF絡み）。
  - 元スクリプト側では Profile生成で End1/End2 を多用しているが、ブラケットでは「必要なときだけ」踏襲すべき（＝1505での `Sf1EndElements` はほぼ必須、1501のFL–FLでは原則不要）。

### 注意事項
- **（最重要）ブラケットは Surfaces1 / Surfaces2 の“面ペア”で成立する**
  - 面記号（FL/WF/WB）を誤ると生成失敗しやすい。参考では、接続関係により面を厳密に選んでいる。
- **PLS形式の Surfaces 指定はベクトル方向に依存**
  - 参考の `["PLS","False","False","0","-0","-1",solid1]` のように、法線方向（`0,-0,-1` 等）が入っている。元スクリプトでも、デッキ面に対して同方向指定を踏襲する必要がある。
- **BracketType=1505 は `BracketParams` と `Sf1EndElements` の組がセット**
  - 参考では1505系はほぼ例外なく `BracketParams=["200"]` と `Sf1EndElements`（PLS形式）が付与され、寸法は `Sf1DimensionType=1541` を使う。
- **寸法指定（Sf1DimensionType/Sf2DimensionType）が未設定だと不安定**
  - 参考は必ず `Sf1DimensionType`/`Sf2DimensionType` とパラメータ配列を入れている。量産配置でも省略しない前提で仕様化する。
- **「50個以上」の実現は、単発追加ではなく“対象群×テンプレの反復”が必須**
  - 参考スクリプトは多数のブラケットを、同型の定義で繰り返し作る設計になっている。元スクリプトでも、FR群（Side）＋DL群（Fore/Aft）＋Deck縦通材群を組み合わせて数量を満たすのが自然。
- **後処理（BodyDivideByCurves）は参考では多数実行されているが、ブラケット配置仕様としては必須ではない**
  - 参考では `BodyDivideByCurves` によりブラケット形状のトリムや分割を行っている箇所がある（例：bracket1/bracket4/bracket6等）。  
  - 元スクリプトへ同等品質を求めるなら、同様の分割スケッチ平面（例：`skt_pl4`）を使った分割工程を「ブラケットごとに」適用する設計が必要。
