## ブラケット配置分析

### 対象部材とループ変数
- **Plate（面：PLS 指定で参照される板要素）**
  - `solid1`：HK.Casing.Deck.C（Deck C のソリッド。参考では Deck.D の `solid1` を PLS 面としてブラケットに使用）
  - `solid2`：HK.Casing.Deck.D（Deck D のソリッド）
  - `solid3`：HK.Casing.Deck.B（Deck B のソリッド）
  - `solid4`：HK.Casing.Deck.A（Deck A のソリッド）
  - `solid5`：HK.Casing.Wall.Aft.CD（Aft wall 系ソリッド）
  - `solid10`：HK.Casing.Wall.Fore.CD（Fore wall 系ソリッド）
  - `solid12`：HK.Casing.Wall.Side.CDP（Side wall 系ソリッド）
  - `solid8/solid15/solid16`：Side wall 系ソリッド（OAP/BCP/ABP）

- **Profile（主に FL 面を持つスティフナ/形鋼：ブラケットの相手面）**
  - Fore/Aft 壁の縦通材相当：`profile6,7,26,32,62,69,72,75,83,87,94,100,104,105,115,117,119,121,123` など（多くが `ProfileType=1002`）
  - Deck 側の縦通材：`profile8,17,23,33,35,47,50,54,61,68,71,74,85,93,108,109,114` など
  - Side 壁のフレーム材：`profile3,5,10,12,16,20,21,24,34,44,45,51,52,55,58,76,79,82,89,91,92,102,103,111,113,116,118,122` など
  - ガーダ/ブラケット基準になりうる大形材：`profile15,18,19,22,27,30,40,41,42,43,49,57,59,60,63,64,65,70,77,78,88,90,95,96,97,98,101,106,107,110,112,120` 等（`ProfileType=1003/1201` を含む）

- **ループ変数として扱える“配置候補の集合”**
  - 「壁×デッキ」：Fore/Aft/Side wall 側の *FL面 profile* と、Deck 側の *FL面 profile*（または Deck ソリッドの PLS）をペアリング
  - 「デッキ×デッキ」：Deck A/B/C/D の縦通材同士（例：DLxxP と DLxx.FP / FRxxP と FRxxC など）の交点でペアリング  
  ※元スクリプトにはブラケット作成が一切ないため、参考スクリプトのように「個別に bracketPram を量産」するか、「候補リストを作って for で生成」する設計が必要。

### ループ構造方針
- **方針A（推奨）：候補ペアの配列を定義して for で生成**
  - 例：  
    - `pairs_1501 = [(base, sf1_profile, sf2_profile, thickness, mold, sf1_dim, sf2_dim, scallop), ...]`
    - `pairs_1505 = [(base_profile, plate_solid, end_element_profile, thickness, bracket_param_R, ...), ...]`
  - for ループ内で `CreateBracketParam()` → 必須プロパティ設定 → `CreateBracket()` の順に統一。

- **方針B：部材群をグルーピングして二重ループ**
  - Side FR 系（`profile44/52/111/113/...`）を配列化し、同一 Plate（例：`solid2` Deck.D）へ 1505 を量産。
  - Fore/Aft DL 系は DL番号で対応する Deck 側 DL材（例：`profile17/23/35/86/...`）と突合して 1501 を量産。

### パラメータ依存関係
- **BracketType 選定**
  - `1501`：**Profile（FL等）× Profile（FL等）**の2面ブラケット（参考で最頻出）。  
    - Surfaces 指定例：`Surfaces1=[profileX+",FL"]`, `Surfaces2=[profileY+",FL"]`
    - 場合により `WF` / `WB` を使う例あり（参考：`profile4[0]+",WF"` や `",WB"`）。
  - `1505`：**Plate（PLS）× Profile（FL）**の2面ブラケット。  
    - Surfaces1 が PLS 配列指定、Surfaces2 が profile の `,FL`。
    - 参考では `BracketParams=["200"]` が付く（ナレッジグラフ上は R パラメータ）。

- **Thickness（板厚）**
  - 参考スクリプトの実績値：
    - `1501`：主に `Thickness="8"` または `"12"`、大形材近傍は `"9"`（例：WF絡み）
    - `1505`：`Thickness="10"` が多い
  - 依存関係としては「接続先 profile の板厚（例：7, 9, 12）に近い値」を採用しているように見えるが、参考では固定値運用が多い。
  - 本仕様では以下を標準ロジックとする：
    - ProfileParams に 7mm級（例：`["125","75","7",...]`）→ Bracket thickness 8mm
    - ProfileParams に 9mm級（例：`["150","90","9",...]`）→ Bracket thickness 10mm
    - Plate/FB 12mm級（例：`ProfileType=1201`）→ Bracket thickness 12mm

- **寸法（Sf1DimensionType / Sf2DimensionType）**
  - 参考の基本：
    - `Sf1DimensionType=1531`, `Sf1DimensonParams=["200","15"]` のように「長さ200、クリアランス15」系が多い
    - Plate PLS を使う 1505 では `Sf1DimensionType=1541`, `Sf1DimensonParams=["0","100"]` が頻出（端基準+オフセットのような使い方）
  - よって本仕様では：
    - 1501：両面 1531/["200(or 250/300/800)","15"]
    - 1505：Sf1（Plate側）1541/["0","100"]、Sf2（Profile側）1531/["200","15"]

- **Sf1EndElements / Sf2EndElements の必要性**
  - 参考では 1505 の Plate側寸法定義に **Sf1EndElements** を伴う例がある（`Sf1EndElements=["PLS",... , profile1[0]]`）。
  - 端位置を安定させるため、**PLS を使う 1505 は Sf1EndElements を付ける**方針が安全。
  - 1501 は通常 EndElements なしで成立している。

- **Scallop**
  - 参考は `Scallop1Type=1801`、パラメータ `["0"]` または `["50"]` が混在。
  - 端部干渉が起きやすい（フランジ付近/厚板付近）は `["50"]`、一般は `["0"]` を採用。

- **Mold（"+" / "-"）**
  - 参考では接続部材の向き・Mold と整合するように設定されているが、原則として「BaseElement 側の mold と同方向」を踏襲。
  - 仕様上は、Fore 側の多くは `"+"`、一部 FB/PL 系は `"-"` があり得る（参考：DL02 のブラケットは Mold="-" がある）。

### 注意事項
- **Surfaces 指定は“面ペア”が本体**
  - Bracket は `Surfaces1` と `Surfaces2` の組で決まり、`BaseElement` は参照・命名・向きの基準として重要。
  - Profile の面種別（`FL/WF/WB/FR`）の選択で結果が変わるため、基本は `,FL`、大形材のウェブに当てたい場合のみ `,WF` or `,WB` を使う（参考：FR13P/FR09P で WF/WB が使われる）。

- **PLS（Plate）参照の書式**
  - 参考にある `["PLS","False","False","0","-0","-1",solidX]` 形式を踏襲すること。  
  - ベクトル符号（例：`-1,0,0` / `0,0,-1`）を誤ると、ブラケット面が反転・生成失敗の原因になり得る。

- **ブラケット生成後の後処理**
  - 参考では `BodyDivideByCurves` によりブラケットや周辺部材を分割している箇所が多い。  
  - 元スクリプト側で同様の分割を行う場合、**ブラケット生成直後に分割スケッチ平面（例：`skt_pl4`）が存在すること**、および分割対象がソリッドであることを前提に順序を組む必要がある。

- **“20個以上”の満たし方**
  - 単に数を増やすのではなく、参考スクリプト同様に
    - Fore/Aft の DL01〜DL05
    - Side の FR07/08/10/11/12/14/15
    - Deck の FR09/FR13  
    といった**規則的な位置の繰り返し**で網羅するのが安定。上表はこの思想に沿っている。
