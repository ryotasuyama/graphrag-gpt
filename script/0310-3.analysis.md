### ループ構造方針
- **方針1: DL 系の Fore/Aft–Deck 接続を一括抽出**
  - `DL01, DL02, DL03, DL04, DL05, DL00` ごとに  
    - Fore 側 profile
    - Aft 側 profile
    - Deck 側 profile  
    を対応付けてブラケット生成対象にする。
- **方針2: Side FR 系の Deck.D 接続を一括抽出**
  - `FR07, FR08, FR10, FR11, FR12, FR14, FR15` は参考スクリプトどおり **Type 1505** を採用。
  - `Surfaces1` は deck solid の `PLS`、`Surfaces2` は side profile の `FL`。
  - `Sf1EndElements` は基準 deck longitudinal (`profile17[0]` 相当) を使って位置規制。
- **方針3: Side大型材 FR09/FR13 は Type 1501**
  - `FR09`, `FR13` は side profile と deck profile の双方が profile 系なので、`FL` 同士で 1501 を適用。
- **方針4: 20個以上を確実化**
  - 最低セット:
    - Fore/Aft × DL01〜DL05 = 10個
    - Side FR07/08/09/10/11/12/13/14/15 = 9個
    - DL00 の Fore/Aft = 2個
  - この時点で **21個** となり、編集指示の「20個以上」を満たす。
- **方針5: mirror 先は必要に応じて別ブラケット対象**
  - 元スクリプトでは `MirrorCopy` が大量にあるため、片舷生成後に対称コピーする方式も有効。
  - 参考スクリプトでは左右別 profile を直接使う例と、mirror 部材を端部要素に使う例が混在。

### パラメータ依存関係
- **BracketType の使い分け**
  - **1501**
    - profile–profile 接続に使用。
    - 典型例:
      - `Surfaces1=[profileX[0]+",FL"]`
      - `Surfaces2=[profileY[0]+",FL"]`
    - 一部で `WF`, `WB`, `PLS` の組み合わせあり。
  - **1505**
    - side stiffener を deck plate/solid へ取り付けるパターン。
    - 典型例:
      - `Surfaces1=["PLS","False","False","0","-0","-1",solid1]`
      - `Surfaces2=[profileX[0]+",FL"]`
      - `BracketParams=["200"]`
      - `Sf1EndElements=["PLS","False","False","-0","-1","-0",profile1[0]]`
- **Thickness**
  - 参考スクリプトから以下の傾向が明確。
    - profile 125x75 系どうしの小ブラケット: **約8 mm**
    - side 150x90 系 + deck solid の 1505: **約10 mm**
    - 1201 系フランジ付部材との接続: **12 mm**
    - 300x90 系 deck primary member 周辺: **約9 mm**
- **Sf1 / Sf2 寸法**
  - 小ブラケット（DL系 125x75）:
    - `Sf1DimensionType=1531`
    - `Sf1DimensonParams=["200","15"]`
    - `Sf2DimensionType=1531`
    - `Sf2DimensonParams=["200","15"]`
  - FR13 / FR09 など大型 profile 接続:
    - `["250","15"]` や `["300","15"]`
  - side to deck solid の 1505:
    - `Sf1DimensionType=1541`
    - `Sf1DimensonParams=["0","100"]`
    - `Sf2DimensionType=1531`
    - `Sf2DimensonParams=["200","15"]`
- **FlangeType / FlangeParams**
  - 1201 系との接続ブラケットではフランジ付加あり。
  - 典型:
    - `FlangeType=262`
    - `FlangeParams=["100","30","29.999999999999996","30","30","1"]`
  - FR09/FR13 の deck primary 接続では
    - `FlangeParams=["75","30","29.999999999999996","30","50","1"]`
- **Scallop**
  - 小ブラケット:
    - `Scallop1Type=1801`
    - `Scallop1Params=["0"]` または `["50"]`
    - `Scallop2Type=0`
  - 1201 付近では `["50"]` が多い。
- **BaseElement**
  - 基本はブラケットが取り付く側の profile を指定。
  - 参考スクリプトでも
    - Fore/Aft–Deck 系: 壁側 profile を `BaseElement`
    - Side–Deck 系: side profile を `BaseElement`
    - Deck primary 周辺: deck primary profile を `BaseElement`
- **End1/End2 指定について**
  - ブラケットそのものに `End1/End2` は不要。
  - ただし、ブラケットの位置確定に `Sf1EndElements` が必要なパターンがあり、**1505 では必須に近い**。
  - 特に deck solid を `PLS` で受ける場合、終端基準がないとブラケット位置が不安定化しやすい。

### 注意事項
- **面ペアの型を厳密に使い分ける必要がある**
  - 参考スクリプトでは `FL`, `WF`, `WB`, `PLS` が混在。
  - profile 同士なら `FL` が基本だが、deck primary の web 接続では `WF` / `WB` を使う。
- **1505 では `Sf1EndElements` を省略しない**
  - 参考スクリプトの side-to-deck ブラケットはすべて、deck 面の位置規制に `Sf1EndElements` を持つ。
  - これを落とすと始点の解釈がずれて、期待位置に出ない可能性が高い。
- **DL02 の 1201 系は通常の FL 指定だけでなく PLS ベースを優先**
  - 参考スクリプトでは `profile12[1]`, `profile11[1]`, `profile4[1]` などを使い、  
    `Surfaces1/2` に `["PLS", ... , profileX[1]]` の形式を採用している。
  - 元スクリプトでも `profile18[1]`, `profile19[1]`, `profile59[1]` があるため、同様に扱うべき。
- **20個以上にするなら DL系 + FR系の混在が最も自然**
  - 単純に同種ブラケットを乱立させるより、
    - Fore/Aft DL 接続
    - Side FR 接続
    - DL00 を追加
    の組み合わせが、参考スクリプトの設計思想に一致する。
- **MirrorCopy 活用は有効だが、仕様上は元部材と mirror 部材のどちらに付くか明示する必要がある**
  - 仕様書としては、P側生成＋Y面ミラーか、左右個別生成かを区別して記載するのが望ましい。
- **deck solid の選定を間違えない**
  - Deck.D ブラケットは `solid2`
  - Deck.C ブラケットは `solid1`
  - Deck.B ブラケットは `solid3`
  - Deck.A ブラケットは `solid4`
- **最も参考性が高い追加対象**
  - 今回の編集指示「20個以上ブラケット」への対応としては、参考スクリプトに実在した以下の並びを移植するのが安全:
    - Fore DL01〜05 / Aft DL01〜05
    - Side FR07/08/10/11/12/14/15
    - Side FR09/13
    - DL00系の Fore/Aft 追加

以上より、元スクリプトに対しては **Deck.D 周りを中心に少なくとも21個のブラケットを設計対象にするのが最適** です。
