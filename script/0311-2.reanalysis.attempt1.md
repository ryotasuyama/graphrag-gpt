### ループ構造方針

前回方針は基本的に妥当です。今回の修正版では、**失敗しやすい Side FR ↔ Deck FR の「近傍推定接続」を禁止**し、以下の3ループへ整理するのが安全です。

#### ループ1: Wall Fore/Aft の DL 系ブラケット
- Wall を `Fore` と `Aft` に分離
- DL番号ごとに対象を整理
- Deck候補は Deck A/B/C/D の DL プロファイルを抽出
- 必須チェック
  - `Wall.Fore.DLxx.*` → `Deck.*.DLxx.F / FP`
  - `Wall.Aft.DLxx.*` → `Deck.*.DLxx.A / AP`
- ここでは前回の成立候補を維持してよい

#### ループ2: Side Wall ↔ Deck 板ブラケット
- Side.CDP 系のうち Deck.D へ到達しているものを対象
  - `profile16,12,44,111,52,113,3`
- 生成形式は固定
  - `BracketType=1505`
  - `Surfaces1=["PLS",...,solid2]`
  - `Surfaces2=[side_profile+",FL"]`
  - `Sf1EndElements=["PLS",... ,profile17[0]]`
- これは参考スクリプトの成功実績が最も多い安全パターン

#### ループ3: Side Wall ↔ Deck FR プロファイルブラケット
- 直接 1501 接続は**成立実績のあるペアのみ**採用
- 採用可
  - `profile65 ↔ profile64`  （FR09）
  - `profile43 ↔ profile42`  （FR13）
  - `profile57 ↔ profile90`  （FR09）
  - `profile95 ↔ profile96`  （FR09）
  - `profile106 ↔ profile112`（FR09）
  - `profile77 ↔ profile41`  （FR13）
  - `profile101 ↔ profile110`（FR13）
  - `profile43 ↔ profile78`  （FR13）
- 採用禁止
  - FR番号不一致の近傍推定接続
  - 例: `FR07 ↔ FR09P`, `FR10 ↔ FR09P` のような、参考スクリプトに直接成立実績がないもの

### パラメータ依存関係

#### 1. BracketType の使い分け
- `1501`
  - Wall.DL ↔ Deck.DL
  - Side.FR ↔ Deck.FR
  - ただし Side↔Deck.FR は**成立実績のある組のみ**
- `1505`
  - Side.FR ↔ Deck 板面
  - 失敗回避の観点で最重要
  - `BracketParams=["200"]` を伴う
  - `Sf1EndElements` を必須継承

#### 2. Thickness
- `1002` 同士の DL 接続: `8`
- `1201` を含む DL02 系: `12`
- Side↔Deck板 `1505`: `10`
- `1003` を含む FR 接続: `8～9`

#### 3. Surface型
- 通常 profile 接続
  - `[profileX[0]+",FL"]`
- Deck FR 支持ブラケット
  - `[profile18[0]+",WF"]`
  - `Sf1EndElements=[profile18[1]+",FR"]`
- 板面基準
  - `["PLS","False","False","0","-0","-1",solid2]`
- `1201` 系は profile `[0] / [1]` の使い分けが必要

#### 4. DimensionType
- 一般 profile 接続
  - `1531 / ["200","15"]`
- FR大物
  - `1531 / ["250","15"]` または `["300","15"]`
- 板面基準 1505
  - `1541 / ["0","100"]`
  - `Sf1EndElements` 必須
- DL02 の大型接続
  - `1531 / ["800","15"]`

#### 5. 失敗原因として考えるべき依存
今回の COM 例外から、次の条件が失敗要因として強いです。
- `1501` なのに、**相手部材が幾何的に既存成功パターンと一致しない**
- FR番号不一致
- Side部材を Deck FR に直接つなぐが、参考スクリプトではその部材が本来 **Deck板 1505 型**で扱われている
- `PLS` を使うべき箇所を `FL-FL` で無理に組んでいる

### 注意事項

- **Fore/Aft 整合ルールは必須**
  - `Wall.Fore.DLxx.*` → `Deck.*.DLxx.F / FP` のみ
  - `Wall.Aft.DLxx.*` → `Deck.*.DLxx.A / AP` のみ
- 片側方向の Deck しか存在しない場合は、反対側 Wall 候補を除外する
- Side Wall は方向制約なしだが、**Deck側部材の存在確認は必須**

- **成功済み bracket 1〜12 は維持**
  - 今回の修正対象は失敗した候補のみ
  - 既存成功候補には手を入れない

- **前回 #13 は除外**
  - `Side.FR07 ↔ Deck.D.FR09P` の直接 1501 接続は不採用
  - 代替は `Side.FR07 ↔ Deck.D板` の 1505 型

- **Side FR の直接接続は“番号一致または明確な成立実績”がある場合のみ**
  - 可: FR09↔FR09P、FR13↔FR13P
  - 不可: FR07↔FR09P のような近傍推定

- **1505 型では Sf1EndElements を省略しない**
  - 参考スクリプト準拠で必須
  - これがないと位置決め不安定や COM 例外の原因になりやすい

- **候補数要件は維持**
  - 前回 46候補から失敗1件を除外しても、代替候補を入れたため **43件** の安全候補を確保
  - よって「30個以上」の要件は十分満たす

- **実装時の優先順位**
  1. 成功実績のある 1〜12 をそのまま配置
  2. Side系はまず 1505 板ブラケットを配置
  3. その後に FR09 / FR13 の成立実績ある 1501 を追加
  4. 最後に Fore 側の Deck A/B/C への拡張DLブラケットを追加

以上より、修正版では **前回の成功済み候補を維持しつつ、失敗した Side.FR07 ↔ Deck.D.FR09P 直接接続を除外し、Deck.D 板面を使う 1505 型へ置換**するのが最も安全です。
