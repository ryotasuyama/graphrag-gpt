## ブラケット配置分析

### 対象部材とループ変数
本件「元のスクリプト」は、甲板（Deck A/B/C/D：`extrude_sheet1/2/4/5`）とケーシング外板（Wall Aft/Fore/SideP/SideS：`extrude_sheet3/8/6/9`）、およびそれらに取り付く多数のプロファイル（`profile1`〜`profile123` と多数の `mirror_copied*`）を生成しています。

ブラケット（`CreateBracketParam` → `CreateBracket`）が必要となる接合は、参考スクリプトの実装パターンから以下の2系統に分類できます。

- **系統A：FL面どうしの“面ペア”で定義するブラケット（BracketType=1501）**  
  - 主に「同一DL/FRで交差する *2本のプロファイル*（デッキ側FP/DP系 + 壁側CDP/ABP系など）」の取り合い  
  - `Surfaces1=[profileX[0]+",FL"]` と `Surfaces2=[profileY[0]+",FL"]` のように **Profile面指定（FL/WF/WB 等）** で定義
  - 寸法は `Sf1DimensionType=1531`, `Sf2DimensionType=1531` の組み合わせが基本

- **系統B：PLS（平面）× Profile の“面ペア”で定義するブラケット（BracketType=1505）**  
  - 主に「Side Wall（SideP/SideS）に付くFR材（縦通材）と Deck Solid（Deck板ソリッド）との取り合い」  
  - `Surfaces1=["PLS", ... , solidDeck]` と `Surfaces2=[profileFR[0]+",FL"]` のように **PLS定義（法線ベクトル + 参照要素）** を使う
  - `Sf1DimensionType=1541` + `Sf1EndElements` を併用して端部基準を固定するのが参考スクリプトの必須パターン

このため、ループ変数（ブラケット付与対象の列）は次のように整理するのが合理的です。

- **SideP/SideSに付くFR系プロファイル（BracketType=1505対象）**  
  - 例：`profile16/profile12/profile44/profile52/profile111/profile113/profile5/profile10/profile21/profile51/profile79/profile92/profile89/profile103/profile91/profile102/profile76/profile82/profile122/profile94/...` および対応する `mirror_copied*`  
  - “デッキ面（solid）× FR材FL面”の取り合いとして扱う

- **Aft/Fore壁とDeck間でDLごとに成立するプロファイル対（BracketType=1501対象）**  
  - 例：DL01〜DL05、DL03/04などに相当する「壁側CDP/BCP」プロファイルと「デッキ側FP/AP」プロファイルの対  
  - 代表例（元スクリプト中に同一DL名で出現）：  
    - DL01：`profile75`（Aft DL01.CDP）× `profile23`（Deck.D DL01.AP）系、または `profile62`（Fore DL01.CDP）× `profile61`（Deck.D DL01.FP）系  
    - DL03：`profile36`（Aft DL03.CDP）× `profile35`（Deck.D DL03P）系、`profile117`（Fore DL03.CDP）× `profile35`（Deck.D DL03P）系  
    - DL04：`profile87`（Aft DL04.CDP）× `profile86`（Deck.D DL04P）系、`profile100`（Fore DL04.CDP）× `profile86`（Deck.D DL04P）系  
    - DL05：`profile72`（Aft DL05.CDP）× `profile17`（Deck.D DL05P）系、`profile32`（Fore DL05.CDP）× `profile17`（Deck.D DL05P）系  
  - これらは “profile FL × profile FL” の1501パターンで統一できる

補足（参考スクリプトの重要点）  
- 1505型では `Surfaces1` は **PLS + 法線 + 参照要素（solid）** を必ず含める。  
- 1501型では `Surfaces1/2` は **Profileの面（FL/WF/WB）** で与えるのが基本。  
- `BaseElement` は原則として “Surfaces2側のFR材” あるいは “壁側CDP材” など、ブラケット名の主語側部材に合わせる（参考スクリプトはその傾向が強い）。

### ループ構造方針
- **方針1（推奨）：接合タイプごとに対象プロファイル配列を用意してループ**
  - `brackets_1505 = [(solidDeck, profileFR), ...]` を作り、要素ごとに `CreateBracketParam()` → `CreateBracket()` を生成
  - `brackets_1501 = [(profileWall, profileDeck), ...]` を作り、同様に生成
- **方針2：ミラーコピーの自動展開**
  - 元スクリプトでは多くの `mirror_copied*` が既に生成されているため、  
    - 片舷（P）用のリストだけ定義し、**存在する場合は対応ミラー材にも同条件で作成**  
    - もしくは「同名/同プロパティの要素」を検索して両舷で作る（APIに検索がある場合）

### パラメータ依存関係
参考スクリプトから読み取れる、ブラケットパラメータの“固定”パターンは次の通りです（元スクリプトへ適用する際も踏襲）。

- **BracketType**
  - FL×FL（プロファイル同士）：`BracketType = 1501`
  - PLS×Profile（Deck solid面基準の片ブラケット）：`BracketType = 1505` + `BracketParams=["200"]`（参考例）

- **Thickness（板厚）**
  - 1501型：概ね `7.999...` または `12` や `8.999...` が使われる  
    - 元スクリプトで接続する相手が `ProfileType=1002`（t=7相当）なら 8mm級、`ProfileType=1201`（t=12）なら 12mm級、という“近い板厚”に寄せるのが参考スクリプトの傾向
  - 1505型：Side FR材（`ProfileParams` に t=9相当が多い）に対して `9.999...`（≒10）を使う例が多い

- **寸法（Sf1/Sf2 Dimension）**
  - 1501型（FL×FL）：  
    - `Sf1DimensionType = 1531`, `Sf1DimensonParams=["200","15"]`（または部材により 250/300/800 など）  
    - `Sf2DimensionType = 1531`, `Sf2DimensonParams` 同様
  - 1505型（PLS×Profile）：  
    - `Sf1DimensionType = 1541`, `Sf1DimensonParams=["0","100"]` のように **PLS側をオフセット/長さで規定**  
    - `Sf1EndElements` を併用して端部の基準要素を指定（参考では `["PLS", ..., profile1[0]]` のように PLS形式で再指定する例あり）  
    - `Sf2DimensionType = 1531`, `Sf2DimensonParams=["200","15"]` をProfile側に設定

- **Scallop**
  - `Scallop1Type=1801` を基本に、`Scallop1Params=["0"]` または `["50"]`
  - 1501型で強めの取り合い（フランジ付など）では `["50"]` が頻出

- **End指定（Sf1EndElements / Sf2EndElements）**
  - 参考スクリプトで `Sf1EndElements` / `Sf2EndElements` が入っているブラケット（特に1505）では、**これを省略しない**のが重要  
  - 1501型の単純なFL×FLでは省略されることも多い（参考の bracket1/6/9/10/11/13/18/20 等）

### 注意事項
- **Surfaces1 / Surfaces2 は“面ペア”で定義する**  
  - Profileの場合：`profileX[0]+",FL"` や `",WF"`, `",WB"` の指定が必要  
  - PLSの場合：`["PLS", <bool>, <bool>, nx, ny, nz, <refElement>]` 形式（参考スクリプトの `Surfaces1=["PLS",... , solid1]` を踏襲）
- **BracketType=1505 を使う場合、BracketParams と Sf1EndElements をセットで踏襲**  
  - 参考スクリプトでは 1505 の多くが `BracketParams=["200"]` を持ち、さらに `Sf1EndElements` を指定して取り付け基準を固定している
- **BaseElement は必ず実体要素（多くは profile の [0] 側）を指定する**  
  - 参考では `BaseElement=profileXX[0]` が原則
- **ミラー要素（`mirror_copied*`）へのブラケット“漏れ”が最も起きやすい**  
  - 元スクリプトは左右対称材を大量に生成しているため、ブラケットも同等に両舷へ配置する設計が必須
- **“Deck sheet”ではなく“Deck solid”をPLS参照に使うケースがある**  
  - 参考の1505型は `Surfaces1` の参照に `solid1`（デッキの厚み付きソリッド）を使っている。元スクリプトも `solid1〜solid4`（Deck C/D/B/A）を作っているため、同様に **solid参照**で揃えるのが安全
