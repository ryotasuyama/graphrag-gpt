### ループ構造方針
- **方針1: Fore/Aft の縦通材 DL 系を Deck 側 DL 系へ順番に接続**
  - DL00, DL01, DL03, DL04, DL05 について、
    - Fore壁側 `profileXX`
    - Aft壁側 `profileYY`
    - Deck側 `profileZZ`
  - を1組として順番にブラケット化する。
- **方針2: Side壁 FR 系を Deck.D に対して順番に接続**
  - `FR07, FR08, FR10, FR11, FR12, FR14, FR15` は参考スクリプト通り **Plate PLS × Profile FL** の `BracketType=1505` を優先。
  - `FR09, FR13` は参考スクリプト通り **Profile FL × Profile FL** の `BracketType=1501` を優先。
- **方針3: MirrorCopy 済み部材は必要に応じて左右対称にも展開**
  - 元スクリプトには大量の `MirrorCopy` があるため、片舷で定義したブラケット仕様を反対舷へ横展開しやすい。
  - ただしブラケット自体を MirrorCopy するか、左右別に再定義するかは、命名規則と面向き整合で決める。
- **方針4: まず参考スクリプトの既知成功パターンを踏襲**
  - `1501`: `Profile FL × Profile FL`
  - `1505`: `PLS × Profile FL`
  - `Sf1EndElements` を使うケースは必ず継承する。

### パラメータ依存関係
- **BracketType の選定**
  - **1501**
    - ナレッジ上の標準用途: **Profile FL × Profile FL**
    - 参考スクリプトでも、壁縦材と甲板縦材の交点に多数採用。
    - 例:
      - `profile7[0]+",FL"` × `profile6[0]+",FL"`
      - `profile25[0]+",FL"` × `profile24[0]+",FL"`
  - **1505**
    - ナレッジ上の標準用途: **Plate PLS × Profile FL**
    - 参考スクリプトでも、Deck plate ソリッド + Side profile の取り合いに採用。
    - `BracketParams=["200"]` を伴う。
    - `Sf1EndElements` が必要。

- **Thickness の依存関係**
  - 参考スクリプトからの実績値:
    - **125x75x7 系の Profile FL × Profile FL** → `Thickness ≒ 8`
    - **150x90x9 系の Plate × Profile** → `Thickness ≒ 10`
    - **1201系（板厚12）接続** → `Thickness = 12`
    - **200/300級 profile 接続** → `Thickness ≒ 8〜9`
  - よって本件では次のように整理できる:
    - DL00/DL01/DL03/DL04/DL05 の 1002 同士 → **8**
    - FR07/08/10/11/12/14/15 の 1002 + plate → **10**
    - DL02 の 1201 接続 → **12**
    - FR09/FR13 の 1003 接続 → **8**

- **寸法ロジック**
  - `Sf1DimensionType=1531`, `Sf2DimensionType=1531`
    - Profile–Profile の標準。
    - 参考値:
      - DL系小型ブラケット → `["200","15"]`
      - FR09/FR13系や大型端部 → `["250","15"]` または `["300","15"]`
      - DL02の1201接続 → `["800","15"]`
  - `Sf1DimensionType=1541`
    - Plate側から端部要素基準で寸法を取る場合に使用。
    - 参考値:
      - `Sf1DimensonParams=["0","100"]`
      - `Sf1EndElements=["PLS","False","False","-0","-1","-0",profile1[0]]`
  - `BracketParams`
    - `1505` では `["200"]` を踏襲するのが安全。

- **FlangeType / FlangeParams**
  - DL02やFR09/FR13の一部ブラケットで使用。
  - 参考スクリプトで確認できた代表値:
    - `FlangeType=262`
    - `FlangeParams=["100","30","29.999999999999996","30","30","1"]`
    - または `["75","30","29.999999999999996","30","50","1"]`
  - 本件で20個以上を安全に増やす目的なら、まずは**フランジなしの1501/1505**を主に採用し、DL02接続のみ参考通りフランジ付きで維持するのが妥当。

- **Scallop の依存関係**
  - 参考スクリプトではブラケットの標準として:
    - `Scallop1Type=1801`
    - `Scallop1Params=["0"]` または `["50"]`
    - `Scallop2Type=0`
  - 小型標準ブラケットは `["0"]`
  - DL02や大型端部は `["50"]`
  - この使い分けを踏襲するのがよい。

- **BaseElement の決め方**
  - 原則として **壁側 profile** を `BaseElement` にしている例が多い。
  - 例:
    - `HK.Casing.Wall.Fore.DL01.Deck.DP`
    - `HK.Casing.Wall.Side.FR10.Deck.DP`
    - `HK.Casing.Wall.Aft.DL04.Deck.DP`
  - よって仕様としても、**壁側部材を BaseElement に統一**すると整合しやすい。

### 注意事項
- **Surfaces1 / Surfaces2 は単なる要素名ではなく「面指定」**
  - `profileX[0]+",FL"` のような **Profile のフランジ面**
  - `profile4[0]+",WF"` / `",WB"` のような **Web面**
  - `["PLS", ..., solid1]` のような **Plate面参照**
  - を厳密に使い分ける必要がある。

- **1505 を使う場合は Sf1EndElements を省略しない**
  - 参考スクリプトで `Plate PLS × Profile FL` の成功例は、すべて `Sf1EndElements` を持つ。
  - これがないと Plate 側寸法基準が曖昧になりやすい。

- **DL02接続は通常の 1501 よりも扱いが特殊**
  - `ProfileType=1201` を含み、`PLS` 参照や `FlangeType=262` を伴う。
  - ここは参考スクリプトの組み方をそのまま踏襲すべき。

- **Fore/Aft の符号差に注意**
  - Fore側とAft側で `Mold`, `ReverseDir`, `Surfaces1` のベクトル符号が変わる。
  - 特に `PLS` 参照では `["-1","-0","0", ...]` と `["1","0","-0", ...]` の違いが重要。

- **MirrorCopy 済み部材を使う場合の面向き**
  - ミラー後は見た目が同じでも `FL/WF/WB` の解釈が変わる場合がある。
  - 参考スクリプトではミラーした部材に対しブラケットを直接張るより、元側と別定義しているケースが多い。

- **20個以上を確実に満たす最小安全セット**
  - 実務上は以下を優先するとよい:
    - Fore/Aft × Deck.D の DL系: 10件
    - Side × Deck.D の FR系: 9件
    - Fore/Aft × Deck.C の DL系: 10件
  - この時点で **29件** となり、編集指示を十分満たす。

- **参考スクリプトから読み取れる最重要パターン**
  1. **Profile FL × Profile FL** は `1501`
  2. **Plate PLS × Profile FL** は `1505`
  3. `Sf1EndElements` があるケースは必ず継承
  4. `BaseElement` は壁側 profile を採用
  5. 小型DL系は `200/15`, Side系 plate接続は `1541 + 100`, DL02は `800/15`

以上より、元スクリプトに対して「20個以上ブラケットをつける」仕様としては、**Deck.D・Deck.C 周辺の Fore/Aft/Side 取り合いを中心に、1501 と 1505 を使い分けて最低 29 件程度を配置する方針**が最も自然です。
