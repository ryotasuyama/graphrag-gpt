### ループ構造方針
- **Wall.Fore × Deck(F/FP)** を DL ごとにループ
  - 対象DL: `DL00, DL01, DL03, DL04, DL05`
  - 許可ペア:
    - `Wall.Fore.DLxx.*` → `Deck.X.DLxx.F` / `Deck.X.DLxx.FP`
  - 実際の候補:
    - Deck.A/B/C の `*.F` / `*.FP`
    - Deck.D は Fore 側で `FP` があるものを優先
- **Wall.Aft × Deck(A/AP)** を DL ごとにループ
  - 許可ペア:
    - `Wall.Aft.DLxx.*` → `Deck.X.DLxx.A` / `Deck.X.DLxx.AP`
  - 実際の候補:
    - Deck.D に `DL00.A`, `DL01.AP` が存在
    - Aft 側については Deck.A/B/C に明示的 A/AP が少ないため、候補除外判定が必要
- **Wall.Side(FR) × Deck面 or Deck FRプロファイル** を FR ごとにループ
  - `FR07, 08, 10, 11, 12, 14, 15` は参考スクリプト同様 **BracketType=1505**
  - `FR09, FR13` は Deckの梁材/縦通材プロファイルに対する **BracketType=1501**
- **FB/PL系のDL02接続** は通常ループから分離
  - `profile59/profile19` と `profile18` を組み合わせる
  - `Surfaces1/2` は `PLS` 配列形式を使用
  - Flange指定ありのブラケットを採用
- **ミラー体は補助的利用**
  - ミラーコピー後の要素は、反対舷・反対側候補の位置合わせに利用可能
  - ただしブラケット自体をミラーで複製するより、対象要素から個別生成する方が名前・向き整合が安全

### パラメータ依存関係
- **BracketType の選択**
  - **1501**
    - 壁縦通材 `CDP` と Deck縦通材 `FP/P/AP` の **FL–FL 接続**
    - Deck FR材と Wall.Side FR材の **FL–FL 接続**
    - Deck梁の `WF/WB` 面と FR材 `FL` の接続
    - DL02 のような **PLS–PLS 接続**
  - **1505**
    - Wall.Side FR材と Deck板ソリッド面の接続
    - 参考スクリプトでは `Surfaces1=["PLS",...,solid1]`、`Surfaces2=[profileX[0]+",FL"]`

- **Thickness**
  - 参考スクリプトから以下のパターンが読み取れる
    - 一般DLブラケット（125x75系）: `7.9999999999999964` → 実質 **8t**
    - FB/PL系 DL02: **12t**
    - Side FR のDeck板直付け（1505）: `9.9999999999999982` → 実質 **10t**
    - FR13/FR09 など1003形状との接続: `8.9999999999999982` → 実質 **9t**
  - したがって仕様上は以下で整理できる
    - DL00/01/03/04/05 のCDP系: **8t**
    - DL02 FB/PL接続: **12t**
    - Side FR07/08/10/11/12/14/15 の1505: **10t**
    - FR09/FR13 の梁接続: **8〜9t**

- **Sf1 / Sf2 寸法**
  - FL–FL の標準:
    - `Sf1DimensionType=1531`
    - `Sf2DimensionType=1531`
    - `Sf1DimensonParams=["200","15"]`
    - `Sf2DimensonParams=["200","15"]`
  - FR13 / FR09 の大きめ材:
    - `["250","15"]` または `["300","15"]`
  - 1505 板面接続:
    - `Sf1DimensionType=1541`
    - `Sf1DimensonParams=["0","100"]`
    - `Sf1EndElements` が必須
    - `Sf2DimensionType=1531`
    - `Sf2DimensonParams=["200","15"]`

- **FlangeType / FlangeParams**
  - DL02 のPLS型、および一部 FR13/FR09 のWF接続で必要
  - 参考値:
    - `FlangeType=262`
    - DL02: `["100","30","29.999999999999996","30","30","1"]`
    - FR09/FR13: `["75","30","29.999999999999996","30","50","1"]`

- **Scallop**
  - 標準:
    - `Scallop1Type=1801`
    - `Scallop1Params=["0"]` または `["50"]`
    - `Scallop2Type=0`
  - DL02 のFB/PL系、FR09/FR13 のWF接続では `["50"]` が多い
  - 単純FL–FL接続は `["0"]` ベースでも成立

- **BaseElement**
  - 原則として **壁側またはSide側の部材** を `BaseElement` にする
  - 例外:
    - Deck FR材起点のブラケットでは Deck側を `BaseElement` にしている例あり
  - 編集方針としては統一のため
    - Wall.Fore / Wall.Aft 接続 → 壁側 `CDP`
    - Wall.Side × Deck面接続 → Side側 `CDP`
    - Deck FR × Wall.Side FR 接続 → Deck FR材を `BaseElement`

### 注意事項
- **Fore/Aft 整合ルールは必須**
  - `Wall.Fore.DLxx.*` と `Deck.*.DLxx.A / AP` の組み合わせは禁止
  - `Wall.Aft.DLxx.*` と `Deck.*.DLxx.F / FP` の組み合わせは禁止
  - 元スクリプトには `DL00/01/03/04/05` で Fore/Aft 両壁材と、Deck A/B/C/D の F/FP/A/AP 系が混在しているため、実装時は **ProfileName の末尾記号で厳密判定**が必要
- **Side Wall（FR系）は方向制約なし**
  - ただし接続先のDeckプロファイルまたはDeck面の存在確認は必要
- **PLS形式を使うケースを落とさない**
  - DL02 は `profile18`, `profile19`, `profile59` のような FB/PL 系で、`FL` 指定ではなく `PLS` 指定を使う必要がある
- **Sf1EndElements / Sf2EndElements の継承**
  - 1505 の Deck板直付けブラケットでは、参考スクリプト同様
    - `Sf1EndElements=["PLS","False","False","-0","-1","-0",profile1[0]]`
    - のような終端拘束が必要
- **WF / WB / FL の面種別を正しく使い分ける**
  - `1003` など梁形プロファイル相手では `WF` または `WB`
  - 一般アングル/フラット相手は `FL`
- **30個以上の配置は可能**
  - 参考スクリプトでは実際に20件超のブラケットが生成されており、
  - 元スクリプトはそれ以上に多いプロファイル群を持つため、
  - **Fore/Aftを守った候補抽出 + Side FR全数適用 + FR09/13梁接続** を組み合わせれば 30件超は十分達成可能
- **推奨の実装優先順位**
  1. Side FR の 1505 ブラケットを先に全件生成
  2. Fore/Aft の DL接続ブラケットを生成
  3. FR09/FR13 の梁接続ブラケットを生成
  4. DL02 の PLS型ブラケットを生成
- **名前付け規則**
  - 参考スクリプト準拠なら
    - `HK.Casing.Wall.Fore.DL01.Deck.DP`
    - `HK.Casing.Wall.Side.FR10.Deck.DP`
    - `HK.Casing.Deck.D.FR13P`
  - のように、接続元主材を先頭にした命名が一貫している

以上より、編集指示「**30個以上ブラケットをつけてください**」に対しては、  
**Wall.Fore/Aft のDL接続 + Wall.Side のFR接続 + Deck FR接続** の3系統で構成し、  
**BracketType 1501 / 1505 を使い分ける仕様**が最適です。
