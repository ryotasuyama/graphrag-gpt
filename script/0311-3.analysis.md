### ループ構造方針
- **Fore/Aft 壁 × Deck縦材接続ループ**
  - DLごとに `Wall.Fore` / `Wall.Aft` の `CDP` を抽出し、対応する Deck の同一DLプロファイルとペアリングする。
  - ループ変数例:
    - `wall_fore_profiles = [profile62, profile59, profile117, profile100, profile32]`
    - `wall_aft_profiles = [profile75, profile19, profile36, profile87, profile72]`
    - `deck_d_profiles = [profile61, profile18, profile35, profile86, profile17]`
- **Side Wall × Deckソリッド接続ループ**
  - FRごとの Side Wall `CDP` を順番に処理し、Deck.D ソリッド面に対して 1505 型ブラケットを配置する。
  - 参考スクリプト準拠で `Surfaces1 = PLS + solid`, `Surfaces2 = profile + ",FL"` の組合せを採用。
- **Side Wall × Deck FRプロファイル接続ループ**
  - FR09, FR13 のように Deck 側にも明示的な横材 `FRxxP/C` がある位置では、1501 型で profile-to-profile 接続を行う。
- **候補生成 → 方向整合チェック → ブラケット化**
  - 先に候補表を作る
  - 各候補で ProfileName から Fore/Aft サフィックスを確認
  - 条件に一致したもののみブラケット作成対象にする

### パラメータ依存関係
- **BracketType**
  - `1501`: 主に **profile と profile の直接接続**
    - 例: `Wall.Fore.DL01.CDP` ↔ `Deck.D.DL01.FP`
    - 例: `Wall.Side.FR13.CDP` ↔ `Deck.D.FR13P`
  - `1505`: 主に **Deck plate / solid 面 と Side Wall profile の接続**
    - 例: `Surfaces1 = ["PLS", ..., solid2]`
    - 例: `Surfaces2 = [profile13[0]+",FL"]`
- **Thickness**
  - 参考スクリプトから以下の傾向を継承する。
  - Wall.Fore / Wall.Aft の `1002` 同士接続:
    - ほぼ `7.9999999999999964` 付近
  - `1201` を含む DL02 接続:
    - `12`
  - Side Wall `1002` と plate/solid 接続:
    - `9.9999999999999982`
  - Side Wall `1003` や Deck `1003` 接続:
    - `7.9999999999999964` または `8.9999999999999982`
- **Sf1 / Sf2 Dimension**
  - `1501` の標準:
    - `Sf1DimensionType = 1531`
    - `Sf1DimensonParams = ["200","15"]` または `["250","15"]` / `["300","15"]`
    - `Sf2DimensionType = 1531`
    - `Sf2DimensonParams = 同系統`
  - `1505` の標準:
    - `Sf1DimensionType = 1541`
    - `Sf1DimensonParams = ["0","100"]` または `["0","80"]`
    - `Sf2DimensionType = 1531`
    - `Sf2DimensonParams = ["200","15"]`
- **Sf1EndElements**
  - 参考スクリプトで **1505型には必須級**
  - Deck plate 基準の終端位置を与えるため、
    - 例: `["PLS","False","False","-0","-1","-0",profile17[0]]`
    - 例: `profile18[1]+",FR"` のような profile端指定
- **FlangeType / FlangeParams**
  - `1201` や `WF/WB` を使う一部の 1501 ブラケットでは設定あり
  - 例:
    - `FlangeType = 262`
    - `FlangeParams = ["75","30","29.999999999999996","30","50","1"]`
    - もしくは `["100","30","29.999999999999996","30","30","1"]`
  - 今回の大量配置では、まず参考スクリプトに存在する同型組合せに限定して採用する方針が安全

### 注意事項
- **Fore/Aft 方向整合は必須**
  - `Wall.Fore.DLxx.*` は `Deck.*.DLxx.F` / `Deck.*.DLxx.FP` のみ接続対象
  - `Wall.Aft.DLxx.*` は `Deck.*.DLxx.A` / `Deck.*.DLxx.AP` のみ接続対象
  - したがって、候補表作成時に **DL一致だけでなく F/A も一致確認** が必要
- **Side Wall（FR系）は方向制約なし**
  - ただし、接続先 Deck プロファイルまたは solid の存在確認は必要
- **Surfaces は面ペアで定義**
  - `profile + ",FL"` のような profile面指定
  - `profile + ",WF"` / `",WB"` のような web面指定
  - `["PLS", ..., solid]` のような plate/solid の擬似面指定
  - 接続部材のタイプに応じて形式を変える必要がある
- **BaseElement の選び方**
  - 基本は壁側部材を BaseElement にしている
  - Side Wall × Deck profile でも Side Wall 側を BaseElement に統一すると管理しやすい
- **DL02 の 1201 断面は特別扱い**
  - `profile18`, `profile19`, `profile59`, `profile120` のように flange付きで戻り値が複数要素になっている
  - `Surfaces1/2` や `BaseElement` に使う要素番号を誤ると失敗しやすい
- **1505 はソリッド準備が前提**
  - `solid2` など Deck ソリッドが存在し、その面に対してブラケット平面を取る構成
  - ソリッド未生成のDeckに同方式を適用しない
- **参考スクリプトでは BodyDivideByCurves が多用される**
  - ブラケット作成後の分割処理自体は後工程だが、ブラケット位置の見え方や後続処理に影響する
- **30個以上のブラケットを安定して配置するには**
  - まず参考スクリプト実績のある組合せを優先
  - 次に同じパターンを DL・FR の全候補へ横展開
  - 最後に Fore/Aft ルール違反候補を除外するのが安全

### まとめ
- 参考スクリプトから読み取れるブラケット配置の核は以下です。
  1. **Wall.Fore / Wall.Aft と Deck縦材の 1501 接続**
  2. **Wall.Side と Deck面の 1505 接続**
  3. **Side FR位置での Deck横材との 1501 接続**
- 元スクリプトには、ブラケット生成コード自体は未追加ですが、**接続対象となる profile / solid は十分に揃っている**ため、参考スクリプトのブラケットパターンを移植すれば **35件規模** のブラケット配置が可能です。
