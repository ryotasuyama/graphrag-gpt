### ループ構造方針
- **方針1: Wall.Fore / Wall.Aft × Deck.DL の自動ペアリング**
  - `Wall.Fore.DLxx.*` は `Deck.*.DLxx.F / FP` のみを接続候補とする
  - `Wall.Aft.DLxx.*` は `Deck.*.DLxx.A / AP` のみを接続候補とする
  - `CDP` 同士の主接続は参考スクリプトどおり `Wall.CDP` を `BaseElement` にし、Deck 側は `Surfaces2` に `FL` を指定
  - `DL02` のT材のみ `PLS` ベース面参照を使う

- **方針2: Side Wall FR 系の一括ループ**
  - 対象 FR: `07,08,09,10,11,12,13,14,15`
  - FRごとに Deck 側の存在を確認して候補化
  - 2種類に分ける:
    - `1505`: `PLS + solid(deck)` 対 `SideWall.FL`
    - `1501`: `Deck.FR profile FL` 対 `SideWall.FL`

- **方針3: Deck レベルごとのループ**
  - Deck D, C, B, A ごとにテーブルを持ち、
    - `DL00/01/03/04/05`
    - `FR09/13`
  - 参照可能な deck profile 名 suffix を見て Fore/Aft 一致を判定

- **方針4: 参考スクリプトの実装継承**
  - `BracketType=1501`:
    - `Surfaces1=[wall_or_deck_profile+",FL"]`
    - `Surfaces2=[paired_profile+",FL"]`
  - `BracketType=1505`:
    - `Surfaces1=["PLS","False","False","0","-0","-1",solidX]`
    - `Surfaces2=[side_profile+",FL"]`
    - `Sf1EndElements=["PLS","False","False","-0","-1","-0",deck_dl_profile]`

- **方針5: 今回のエラー再発防止ルール**
  - **ミラー生成された Deck 側プロファイルを、Aft/Fore 判定の代用品として使わない**
  - `mirror_copied***` は幾何学上対称位置を作るだけで、命名由来の Fore/Aft 整合を保証しない
  - とくに `Wall.Aft.DL01.*` に対して `mirror_copied106[0]` のようなミラー体を `Surfaces2` に与える構成は除外する
  - BCP / ABP / OAP などの帯区分プロファイルについては、**対応する Deck 側に A / AP 実体名がある場合のみ** 1501 候補に昇格させる

### パラメータ依存関係
- **BracketType の使い分け**
  - `1501`
    - 壁縦材と deck 縦材の直接 FL-FL 接続
    - Side FR でも deck 横桁/フレームの FL と直接結ぶ場合に使用
  - `1505`
    - Side Wall と deck plate/solid 面を基準にしたブラケット
    - `BracketParams=["200"]` を伴うケースを参考スクリプトどおり踏襲

- **Thickness**
  - 参考スクリプトの傾向:
    - 小型L形/FB相当接続: `7.9999999999999964` ≒ **8 mm**
    - Side Wall CDP 系: `9.9999999999999982` ≒ **10 mm**
    - T材接続（DL02）: `12`
    - Deck FR P/C 系の一部: `8.9999999999999982` ≒ **9 mm**
  - したがって仕様上の割当は以下が妥当:
    - DL01/03/04/05・DL00 の一般接続: **8 mm**
    - Side FR の 1505: **10 mm**
    - DL02 T材: **12 mm**
    - FR09/FR13 の deck framed bracket: **8〜9 mm**

- **Sf1 / Sf2 寸法**
  - `1501` 一般:
    - `Sf1DimensionType=1531`
    - `Sf2DimensionType=1531`
    - 代表値:
      - DLブラケット: `["200","15"]`
      - FRブラケット: `["250","15"]` または deck 側が大きい場合 `["300","15"]`
  - `1505` Side Wall:
    - `Sf1DimensionType=1541`
    - `Sf1DimensonParams=["0","100"]`
    - `Sf2DimensionType=1531`
    - `Sf2DimensonParams=["200","15"]`

- **Flange 指定**
  - T材接続や deck girder 系ではフランジ付きブラケットを使用
  - 参考スクリプトの DL02 / FR09 / FR13 で確認される典型:
    - `FlangeType=262`
    - `FlangeParams=["100","30","29.999999999999996","30","30","1"]`
    - または `["75","30","29.999999999999996","30","50","1"]`
  - したがって `DL02` と一部 FR 交差部ブラケットに限定して採用

- **Scallop**
  - 標準:
    - `Scallop1Type=1801`
    - `Scallop1Params=["0"]` または `["50"]`
    - `Scallop2Type=0`
  - T材や大断面側では `50`
  - 一般小型ブラケットは `0`

- **End1/End2 の必要性**
  - ブラケット自体に `End1/End2` は不要
  - ただし `Sf1EndElements` / `Sf2EndElements` は 1505 や 1541 寸法使用時に必要
  - 特に参考スクリプトでは `1505 + Sf1DimensionType=1541` のとき **`Sf1EndElements` が必須**

- **今回失敗した 27番候補に関する依存関係の見直し**
  - 失敗原因は厚みや寸法よりも、**接続相手の面ペア成立性**の問題である可能性が高い
  - `profile105[0]+",FL"` 自体は wall profile 面として妥当だが、`mirror_copied106[0]+",FL"` が bracket plane を安定定義できなかったと判断する
  - 同じ DL01 でも成功実績は `profile75[0] ↔ profile23[0]` であり、**Deck 側実体プロファイルを直接使う構成**が必要

### 注意事項
- **Fore/Aft 整合ルールは必須**
  - `Wall.Fore.DLxx.*` ↔ `Deck.*.DLxx.F / FP`
  - `Wall.Aft.DLxx.*` ↔ `Deck.*.DLxx.A / AP`
  - 例:
    - `profile62 (Wall.Fore.DL01.CDP)` は `profile61 (Deck.D.DL01.FP)` と接続可
    - `profile75 (Wall.Aft.DL01.CDP)` は `profile23 (Deck.D.DL01.AP)` と接続可
    - `Wall.Fore` と `Deck.*.AP` の接続は不可
    - `Wall.Aft` と `Deck.*.FP` の接続は不可

- **今回の除外判断**
  - `profile105 = HK.Casing.Wall.Aft.DL01.BCP` と `mirror_copied106[0]` の組は除外
  - 理由:
    1. COM 例外の実績あり
    2. `mirror_copied106[0]` は元が `Deck.D.DL01.AP` のミラーであり、Fore/Aft 命名整合を満たす保証がない
    3. 参考スクリプトに同型の成功パターンが存在しない
  - よって **失敗したブラケット候補のみ修正・除外**の要件に従い、27番は除外確定とする

- **代替候補の扱い**
  - 候補数維持のため、27番の代替としては以下の順で採用するのが安全
    1. 既存の成功済み 1〜26 を維持
    2. Fore 側 Deck C/B/A の DL 系候補 32〜41 を優先採用
    3. Side FR × Deck C/B/A の候補 42〜47 を採用
    4. Aft 側 B/C 帯候補 28〜30 は、Deck 側に **A / AP 実体プロファイル**が確認できた場合のみ追加
  - この順序なら、27番を除外しても **30個以上**を安定維持できる

- **DL00 の扱い**
  - `DL00` は `F/A` または `CD/OA/AB/BC` など命名揺れがあるため、suffix を文字列で厳密判定する必要がある
  - `Deck.D.DL00.A` は Aft 側、`Deck.*.DL00.F` は Fore 側として扱う

- **Side Wall（FR系）は方向制約なし**
  - ただし接続先 Deck プロファイルの存在確認は必要
  - deck 側の `FR09P`, `FR13P`, `FR09C`, `FR13C` 等がない FR では 1501 型ブラケットを作らない

- **Surface 指定の型差**
  - 参考スクリプトでは 2種類存在:
    - `profile+",FL"` / `profile+",WF"` / `profile+",WB"`
    - `["PLS","False","False",nx,ny,nz,element]`
  - `1501` は profile 面同士
  - `1505` と T材特殊部は `PLS` ベース面を使う
  - T材系 (`ProfileType=1201`) のブラケットでは profile 本体 `[0]` と flange/body `[1]` のどちらを使うかに注意

- **Deck.FR ブラケットの参照面**
  - 参考スクリプトでは deck 大断面材に対して
    - `WF`
    - `WB`
  - を使い分ける例がある
  - したがって `FR09P/FR13P` 系の deck frame にブラケットを付ける場合、単純な `FL` ではなく部位により `WF/WB` が適切

- **ミラー体の使用制限**
  - `mirror_copied***` は補助候補として扱い、**方向整合チェックを通過した場合のみ使用**
  - 壁材の Fore/Aft を満たす Deck 側実体が存在するなら、そちらを優先する
  - ミラー体しか存在しない場合、その候補は原則 **除外または保留** とする

- **30個以上の配置保証**
  - 修正後も、確定候補だけで 40件超を維持できる
  - よって 27番除外の影響で要件未達になることはない
  - 優先度順は以下:
    1. Deck.D × Fore/Aft DL01/02/03/04/05
    2. Side FR × Deck.D
    3. Deck.C/B/A × Fore 側 DL
    4. Side FR × Deck.C/B/A の FR09/13
    5. DL00 系
    6. Aft 側 B/C 帯候補は実体 Deck 側存在確認後のみ追加

- **参考スクリプトで踏襲すべき重要点**
  - `BracketType=1501/1505` の使い分け
  - `Sf1EndElements` を伴う 1541 寸法の side bracket
  - `PLS` 面参照の構造
  - `Thickness` の部材依存
  - T材部 (`DL02`) での flange 付き bracket 仕様

--- 

以上より、修正版の配置方針としては、  
**成功済み bracket 1〜26 を維持し、COM 例外を起こした `HK.Casing.Wall.Aft.DL01.BCP` × ミラー Deck 側の 27番候補のみ除外する。代わりに Fore 側 Deck.C/B/A 展開および Side FR 展開を優先することで、Fore/Aft 整合ルールを守りながら 30個以上のブラケット候補を安定維持する** 方針が最適です。
