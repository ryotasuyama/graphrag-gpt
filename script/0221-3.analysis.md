## ブラケット配置分析

### 対象部材とループ変数
本スクリプト内でブラケット必須対象となる **ProfileType=1002/1003（山形鋼）・1201（T形鋼）** を抽出し、**End1/End2 接続先ごと**にブラケット候補とします。  
（ProfileType=1007 は必須対象ではないため本分析のブラケット対象外）

- **1003（山形）**: `profile1, profile13, profile14, profile15, profile22, profile29, profile30, profile37, profile38, profile39, profile40, profile41, profile42, profile43, profile49, profile57, profile60, profile64, profile65, profile70, profile77, profile78, profile88, profile90, profile95, profile96, profile97, profile98, profile101, profile106, profile110, profile112, profile120` ほか（ミラー複製は原則オリジナル側のみ作成）
- **1002（山形）**: `profile3,4,5,6,7,8,9,10,12,16,17,19,20,21,23,24,25,26,28,32,33,34,35,36,44,45,46,47,48,50,51,52,53,54,55,58,61,62,63,66,67,68,69,71,72,73,74,75,76,79,80,82,83,84,85,86,87,89,91,92,93,94,99,100,102,103,104,105,107,108,109,111,113,114,115,116,117,118,119,121,122,123` ほか
- **1201（T形）**: `profile2,11,18,19,59,88,97,98,120`（各End接続にブラケット必須）
- **ミラーコピー部材**（`mirror_copiedX`）は「注意事項」に従い原則ブラケット作成対象外（必要なら後でミラー運用）

ループ変数方針（実装時）:
- `targets = [(profile1, ProfilePram1), (profile2, ProfilePram2), ...]` のように **profile変数を配列化**し、各profileについて **End1/End2 を展開して bracket を2個（またはEnd未設定なら0個）**生成する。
- ただし **profile同士接続（End要素が profileX[0]）** の場合は、同一点に対して重複ブラケットを生みやすいので「備考」に従って片側のみ生成するルールを別途持つ。

### ループ構造方針
- **基本方針**:  
  1) 「ブラケット必須プロファイル（type in {1002,1003,1201}）」の profile 変数を列挙  
  2) 各 profile について **End1Elements と End2Elements を評価**し、接続先が  
     - `extrude_sheet*`（板）なら **BracketType=1505（Plate×Profile）**  
     - `profileX[0]`（別プロファイル）なら **BracketType=1501（Profile×Profile）**
  3) 1505 の場合は **Sf1EndElements を必ず設定**（End1/End2側要素を入れる）
- **重複回避（推奨）**:  
  Profile×Profile(1501) は A側とB側の両方で生成すると二重化するため、  
  - 例: `(min(profileA_ID, profileB_ID) 側だけ作成)` のようなキーで一意化  
  - あるいは「End要素が `profileX[0]` の場合は、呼び出し元profile番号 < 接続先profile番号 のときだけ作成」等

### パラメータ依存関係
- **BracketType 判定**
  - `EndElement` が `extrude_sheetN`（または thicken 済の対応 solid）: **1505**
  - `EndElement` が `profileM[0]`: **1501**
- **Surfaces1（1505）**
  - 形式: `["PLS","False","False", nx,ny,nz, solidX_or_sheetX]`
  - `solidX_or_sheetX` は、当該板が `CreateThicken` 済なら **solidを優先**  
    - `extrude_sheet1 → solid3`  
    - `extrude_sheet2 → solid4`  
    - `extrude_sheet4 → solid1`  
    - `extrude_sheet5 → solid2`  
    - `extrude_sheet3 → solid5/solid9/solid11/solid13`（壁用途に応じて代表を割当）  
    - `extrude_sheet6 → solid8/solid12/solid15/solid16`（Side用途に応じて割当）  
- **Surfaces2（共通）**
  - 形式: `[profileN[0] + ",FL"]`
- **BaseElement**
  - 必ず `profileN[0]`
- **Sf1EndElements（1505で必須）**
  - 形式: `["PLS","False","False", ex,ey,ez, endElementVar]`
  - `endElementVar` は End1/End2 で指定されている要素（例: `extrude_sheet1` や `solid3` 等、推奨はsolid）
- **寸法パラメータ（デフォルト）**
  - 1505:  
    - BracketParams `["200"]`  
    - Sf1DimensionType=1541, Sf1DimensonParams=`["0","100"]`  
    - Sf2DimensionType=1531, Sf2DimensonParams=`["200","15"]`  
    - Scallop1Type=1801, Scallop1Params=`["0"]`, Scallop2Type=-1
  - 1501:  
    - Sf1DimensionType=1531, Sf1DimensonParams=`["200","15"]`  
    - Sf2DimensionType=1531, Sf2DimensonParams=`["200","15"]`  
    - Scallop1Type=1801, Scallop1Params=`["0"]`, Scallop2Type=0
- **Thickness / Mold / Material**
  - Thickness: profile板厚相当（多くが 7, 9, 11, 12, 14mm 系）→標準 `"10"` を基準にし、必要なら profile param から選択  
  - Mold: 原則 profile の `Mold` を踏襲  
  - MaterialName: `"SS400"`

### 注意事項
- **Sf1EndElements は BracketType=1505 の場合に必須**。省略すると COM 例外になりやすい。
- **ミラーコピー部材（mirror_copiedX）はオリジナル側のみブラケット作成**し、必要に応じて後工程でミラーする（仕様ガイド準拠）。
- ブラケットは **全solid/profile生成後にまとめて作成**する（依存要素が揃ってから）。
- `part.CreateBracket(bracketParam, False)` の第2引数は **必ず False**。
- `part.BlankElement(bracketX, True)` は使用しない。
- Profile×Profile（1501）は **重複生成**しやすいので、一意化ルール（片側のみ生成）を必ず設計に入れる。
- Surfaces の法線 `nx,ny,nz` は、本来は `SheetAlignNormal` の符号反転値を厳密に追従すべき。今回の元スクリプトでは板の向きが概ね以下に整理できるため、実装では板種別ごとに法線を固定マップ化するのが安全：  
  - Deck系（extrude_sheet1/2/4/5/7）: 代表 `0,0,-1`  
  - Aft/Fore壁（extrude_sheet3/8）: 代表 `-1,0,0`  
  - Side壁（extrude_sheet6/9）: 代表 `0,1,0`
