### ループ構造方針
- **Wall.Fore × Deck の DL 系**
  - `DL00〜DL05` をキーに抽出
  - `Wall.Fore.DLxx.*` は **Deck.*.DLxx.F / FP** にのみ接続
  - 例
    - `profile62 = HK.Casing.Wall.Fore.DL01.CDP`
    - `profile61 = HK.Casing.Deck.D.DL01.FP`
  - Fore 側に `.F/.FP` が存在しない DL は候補除外

- **Wall.Aft × Deck の DL 系**
  - `DL00〜DL05` をキーに抽出
  - `Wall.Aft.DLxx.*` は **Deck.*.DLxx.A / AP** にのみ接続
  - 例
    - `profile75 = HK.Casing.Wall.Aft.DL01.CDP`
    - `profile23 = HK.Casing.Deck.D.DL01.AP`
  - Aft 側に `.A/.AP` が存在しない DL は候補除外

- **Wall.Side × Deck の FR 系**
  - Side Wall は Fore/Aft 制約なし
  - ただし **Deck 側実在確認**は必須
  - 優先順は次の通り
    1. `Side.CDP ↔ Deck.FRxxP`
    2. `Side.* ↔ Deck.DLxx.F / FP`
    3. `Deck plate(PLS) × Side profile(FL)` の `1505`
  - `FRxxF***` のような特殊補助材は、参考スクリプトの成功型と一致しない限り**直接接続候補から除外**

- **失敗候補修正ルール**
  - 今回失敗した `Side.FR11.BCP ↔ Deck.A.FR14F415` は
    - FR一致なし
    - 特殊 FR 補助材
    - `1501 / FL×FL` の成立実績なし
    のため除外
  - 差替えは
    - `Side.FR11.* ↔ Deck.A.DL01.FP / DL03.FP`
    - `Deck.A plate(P﻿LS) ↔ Side.FR11.*`
    のような**既存成功型に近い候補**に限定

- **BracketType のループ選択**
  - `Profile FL × Profile FL` は `1501`
  - `Plate PLS × Profile FL` は `1505`
  - Side 用 `1505` では `Sf1EndElements` を必ず設定する

- **BaseElement の固定方針**
  - 原則として **Wall 側 profile**
  - Fore/Aft/Side のいずれでも同様
  - plate を使う場合も `BaseElement` は side/wall 側 profile のままとする

### パラメータ依存関係
- **BracketType**
  - `1501`
    - 標準 2面ブラケット
    - `FL × FL` の profile-profile 接続
    - DL 系 wall-deck、FR 系 side-deck の通常構成
  - `1505`
    - `PLS × FL` の plate-profile 接続
    - Side Wall 系で特に安定
    - `BracketParams=["200"]` を付与する参考スクリプト型を踏襲すべき構成

- **今回の失敗からの追加判断**
  - `profile49 = HK.Casing.Deck.A.FR14F415` は profile ではあるが、通常の `FR14P` ではなく**特殊 FR 補助材**
  - `FL×FL / 1501` に見えても、取付平面・参照面・交差条件が成立しない可能性が高い
  - よって `FR14F415` への直接ブラケットは、少なくとも今回の分析対象では**除外対象**

- **Thickness**
  - `1002` 同士の標準 DL 接続 → **8**
  - Side と deck plate の `1505` → **10**
  - `1201` を含む DL02 系 → **12**
  - Deck FR 主部材との接続 → **8 または 9**
  - 失敗原因は厚さではなく、**接続面と部材組合せ不適合**と判断

- **Sf1 / Sf2 Dimension**
  - `1501` の標準:
    - `Sf1DimensionType=1531`
    - `Sf2DimensionType=1531`
    - `Sf1DimensonParams=["200","15"]`
    - `Sf2DimensonParams=["200","15"]`
  - FR 主部材周りでは
    - `["250","15"]`
    - `["300","15"]`
    の拡大型もありうる
  - `1505` の plate-profile:
    - `Sf1DimensionType=1541`
    - `Sf1DimensonParams=["0","100"]` または `["0","80"]`
    - `Sf2DimensionType=1531`
    - `Sf2DimensonParams=["200","15"]` または `["300","15"]`

- **FlangeType / FlangeParams**
  - DL02 の 1201 系、または `WF/WB` を使う FR 主部材ブラケットで適用
  - 今回の追加代替候補では、失敗回避を優先して**標準 1501 または 1505 に寄せる**
  - `FR14F415` にはこの特殊 flange パターンを安易に流用しない

- **Sf1EndElements / Sf2EndElements**
  - Side の `1505` では重要
  - 参考スクリプトで成功している plate-profile 構成は、`Sf1EndElements` 付き
  - 今回の代替候補 #24, #29, #30 もこの方針に合わせる

- **BaseElement**
  - 常に Wall 側 profile を優先
  - 失敗候補でも `BaseElement=profile10[0]` 自体は妥当
  - 問題は接続先 `profile49[0]` の選定

### 注意事項
- **Fore/Aft 方向整合ルールは必須**
  - `Wall.Fore.DLxx.*` → `Deck.*.DLxx.F / FP` のみ
  - `Wall.Aft.DLxx.*` → `Deck.*.DLxx.A / AP` のみ
  - 今回の修正版でもこのルールを再確認済み

- **片側 Deck のみ存在する DL は反対側壁から除外**
  - `.F/.FP` のみなら Aft から除外
  - `.A/.AP` のみなら Fore から除外

- **Side Wall は方向制約なし**
  - ただし Deck 側の実在は必須
  - さらに **FR一致優先**
  - FR不一致かつ特殊補助材の組合せは原則除外

- **今回の失敗候補の扱い**
  - `profile10[0]+",FL"` × `profile49[0]+",FL"` は**除外確定**
  - 理由
    1. COM 例外発生済み
    2. FR11 ↔ FR14 の不一致
    3. `FR14F415` が特殊補助材
    4. 参考スクリプト成功型から外れる

- **成功済み候補の維持**
  - bracket 1〜19 はそのまま維持
  - 再分析では**失敗候補のみ修正・除外**

- **代替候補の優先順位**
  1. 既成功の `CDP ↔ Deck.FRxxP`
  2. `Side.* ↔ Deck.DLxx.F/FP`
  3. `Deck plate(PLS) ↔ Side profile(FL)` の `1505`
  4. 特殊 `FRxxF***` は除外寄り

- **面の型を混同しない**
  - `1501` は基本 `FL × FL`
  - `1505` は `PLS × FL`
  - `WF/WB` は参考スクリプトに成功実績がある特定ケースのみ

- **DL02 の 1201 断面は特殊**
  - `profile18`, `profile19`, `profile59`, `profile97`, `profile120` などは `[0] / [1]` の使い分けが必要
  - PLS 参照は flange/body 側の一致が必要

- **候補数要件**
  - 成功済み 1〜19 を維持
  - 失敗した旧 #20 を除外
  - 代替候補を追加した結果、**30個以上**を維持し、最終候補数は 42 件

以上より、この元スクリプトに対する修正版分析としては、**成功済み 1〜19 を維持し、失敗した `Side.FR11.BCP ↔ Deck.A.FR14F415` を除外し、FR一致または Plate 基準の安全な Side-Deck 候補へ置換する方針**が最適です。
