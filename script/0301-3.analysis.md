## ブラケット配置分析

### 対象部材とループ変数

#### 1) ブラケット接続に使える「面」を持つ代表要素（候補母材）
元のスクリプト内で、ブラケット定義に必要となる **面参照（例: `,FL` / `,WF` / `,WB` / `PLS`）** を持ち、接続対象として使いやすい要素は以下です。

- **Deck/Wallの板（Sheet/Solid）系**
  - `extrude_sheet1` : HK.Casing.Deck.B（板）
  - `extrude_sheet2` : HK.Casing.Deck.A（板）
  - `extrude_sheet4` : HK.Casing.Deck.C（板）
  - `extrude_sheet5` : HK.Casing.Deck.D（板）
  - `extrude_sheet3` : HK.Casing.Wall.Aft（板）
  - `extrude_sheet8` : HK.Casing.Wall.Fore（板）
  - `extrude_sheet6` : HK.Casing.Wall.SideP（板）
  - `extrude_sheet9` : HK.Casing.Wall.SideS（板）
  - `solid1`～`solid16` : 各Deck/Wallの厚み付け後ソリッド（板の代替参照に使える）

- **Profile（梁/ロンジ/ガーダ相当、FL/WF/WB面を持つ）系**
  - `profile*`（例: `profile6[0]`, `profile17[0]`, `profile18[0]` など多数）
  - `mirror_copied*`（ミラー複製された profile/solid）

> 参考スクリプトを見る限り、ブラケットは「板（PLS指定またはソリッド参照）× profile（FL/WF/WB）」または「profile×profile」の組合せで定義されることが多い。

#### 2) ループ変数化に適する集合
「50個以上ブラケット」を漏れなく作るには、元スクリプトで大量に生成されている profile 群をリスト化して回すのが最も自然です。

- **Side系の縦通材候補（FR系）**
  - FR07/08/10/11/12/14/15/13/09 などが存在  
  - 例: `profile16, profile12, profile44, profile111, profile52, profile113, profile3, profile43, profile65, profile21, profile64, profile57, profile37, profile77, profile101` 等
- **Fore/Aft系のDL系（水平材）**
  - DL00/01/02/03/04/05 が多数存在  
  - 例: `profile6, profile7, profile17, profile18, profile19, profile25, profile26, profile32, profile35, profile36, profile72, profile86, profile87, profile94, profile99, profile100, profile115, profile117` 等
- **Deck上の梁（FRxxP/FRxxC、DLxxP/FP）**
  - `profile22, profile23, profile42, profile60, profile64, profile90, profile96, profile110, profile112` 等

### ループ構造方針

- **基本方針**: 「同型の接続」をテンプレート化し、接続対象プロファイル群をリストで回す。
  - ループA（Type1505）: **Deck(PLS/solid) × Side縦通材(profile, FL)**  
    - `base_profiles_side = [profile16[0], profile12[0], profile44[0], profile111[0], profile52[0], profile113[0], profile3[0], ...]`  
    - `deck_solids = [solid2, solid1, solid3, solid4]`（D/C/B/A）
    - 各 `deck_solid` × 各 `base_profile` に対し bracket を生成（左右ミラーも含める）
  - ループB（Type1501）: **Wall DL系(profile, FL) × Deck DL系(profile, FL)**  
    - `pairs = [(profile62[0], profile61[0]), (profile72[0], profile17[0]), ...]` のように「同DL番号」優先で作る
    - Aft/Fore両方で作る（DL01〜DL05等）

- **ミラー展開**  
  - 参考スクリプト同様、左右対称にしたい部材は `MirrorCopy` 済み要素（`mirror_copied*`）を同じリストに追加し、同一ロジックでブラケット数を倍化する。

### パラメータ依存関係

参考スクリプトから読み取れる、ブラケットパラメータ決定の実務ロジックは以下。

#### 1) BracketTypeの使い分け
- **1501**: profile同士（FL×FL、またはWF/WB×FL）など、比較的標準的な三角ブラケット
  - 例: `Surfaces1=[profileX+",FL"]`, `Surfaces2=[profileY+",FL"]`
  - Deck梁（FRxxP）で `Surfaces1=[profile4[0]+",WF"]` のような **WF/WB** 面を使うケースもある
- **1505**: 板（PLS/solid面）に対して profile を当てるタイプ（寸法取りにSf1EndElementsを使うことが多い）
  - 例: `Surfaces1=["PLS", ..., solid1]`, `Surfaces2=[profileX+",FL"]`
  - `BracketParams=["200"]` のように追加パラメータが必要（参考スクリプト踏襲）

#### 2) Thickness（板厚）
参考スクリプトの傾向（元モデルでの部材属性連動）：
- Side FR系（1505）: `Thickness ≒ 10`（例: `"9.9999999999999982"`）
- 梁/Deck FRxxP（1501）: `Thickness ≒ 9`（例: `"8.9999999999999982"`）
- DL02フランジ付（1201絡み）: `Thickness = 12` が多い
- 一般の125x75系（1002）絡み: `Thickness = 8` が多い

元スクリプト側に明示計算式は無いので、仕様としては以下で統一すると破綻しにくい。
- **1505**: 10mm固定  
- **1501（梁系/WF・WB参照）**: 9mm固定  
- **1501（一般FL×FL）**: 8mm固定  
- **DL02（1201が絡む箇所）**: 12mm固定

#### 3) 寸法指定（Sf1DimensionType/Sf2DimensionType）
参考スクリプトで頻出の組合せ：
- `Sf1DimensionType=1541`（オフセット/距離指定系）＋ `Sf1EndElements` を併用することがある（1505で多い）
  - `Sf1DimensonParams=["0","100"]` のような「オフセット0、長さ100」パターン
  - `Sf1EndElements=["PLS", ..., profile1[0]]` のように“止め”となる要素を指定
- `Sf2DimensionType=1531`（長さ/余長指定系）
  - `Sf2DimensonParams=["200","15"]`（長さ200、余白15）など

仕様としては、以下の固定値が参考スクリプト互換で無難です。
- **1505（板×profile）**
  - `Sf1DimensionType=1541`, `Sf1DimensonParams=["0","100"]`
  - `Sf2DimensionType=1531`, `Sf2DimensonParams=["200","15"]`
  - `Sf1EndElements` は「Deck上の基準材（例: 同一DLのprofile）」または「代表梁」を1つ指定
- **1501（profile×profile）**
  - `Sf1DimensionType=1531`, `Sf1DimensonParams=["200","15"]`
  - `Sf2DimensionType=1531`, `Sf2DimensonParams=["200","15"]`

#### 4) スカラップ
参考スクリプトではほぼ共通で、
- `Scallop1Type=1801`
- `Scallop1Params=["0"]` または `["50"]`
を使用。  
DL02など大物・フランジ付では `["50"]` が多く、一般では `["0"]` が多い。

### 注意事項

- **Surfacesの指定形式を誤ると作成失敗しやすい**
  - profile同士: `profileX[0]+",FL"` のような簡易指定が中心
  - 板相手（PL面）: 参考スクリプトでは `["PLS","False","False","nx","ny","nz", element]` というベクトル付き指定が使われる  
    - 元スクリプトは板を多用しているため、**1505はこのPLS形式を踏襲**する必要が高い
- **BaseElementは必ず保存し、BracketParam.BaseElementに設定**
  - 参考スクリプトでは常に `BaseElement=profileXX[0]` のように、ブラケット生成の基準部材を明示している
- **BracketType=1505には `BracketParams` が必要**
  - 参考スクリプト: `BracketParams=["200"]` が付く。これを省略すると失敗する可能性がある
- **Sf1EndElements/Sf2EndElementsの有無**
  - 1505では `Sf1EndElements` を要求されるケースがある（参考スクリプト踏襲推奨）
  - 1501では多くの場合不要だが、WF側を基準に取る場合に `Sf1EndElements=[...+",FR"]` のような指定が入ることがある（例: FR09P/FR13P）
- **数量「50個以上」を確実に満たすには**
  - 片舷だけでなく **mirror_copied要素にも同条件でブラケット作成**する
  - Deck面（A/B/C/D）ごとに同じFR群へ付ける（1FR×4Deck×左右＝8個/FR）など、明確な増殖規則を採用することが確実
