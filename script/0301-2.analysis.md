## ブラケット配置分析

### 対象部材とループ変数
- **Deck 系 Sheet（面材）**
  - `extrude_sheet1`: HK.Casing.Deck.B（Deck.B の板）
  - `extrude_sheet2`: HK.Casing.Deck.A
  - `extrude_sheet4`: HK.Casing.Deck.C
  - `extrude_sheet5`: HK.Casing.Deck.D
  - `extrude_sheet7`: HK.General.Deck.UpperDeck（上甲板）
- **Wall 系 Sheet（面材）**
  - `extrude_sheet3`: HK.Casing.Wall.Aft
  - `extrude_sheet8`: HK.Casing.Wall.Fore
  - `extrude_sheet6`: HK.Casing.Wall.SideP（Side.P）
  - `extrude_sheet9`: HK.Casing.Wall.SideS（Side.S）
- **Solid（主に PLS 指定に使われる“板の実体”としての参照）**
  - `solid1`: HK.Casing.Deck.C（Deck.C の厚み付きソリッド）
  - `solid2`: HK.Casing.Deck.D
  - `solid3`: HK.Casing.Deck.B
  - `solid4`: HK.Casing.Deck.A
  - `solid5`: HK.Casing.Wall.Aft.CD（Aft 側の厚み付きソリッド群の一つ）
  - `solid10`: HK.Casing.Wall.Fore.CD
  - `solid12`: HK.Casing.Wall.Side.CDP
  - `solid8/solid15/solid16`: Side の OAP/BCP/ABP 系厚み付きソリッド（面参照用）
- **Profile（補強材・縦材・フランジ等。ブラケットの取り付け相手・BaseElement になりうる）**
  - **Deck 取り合い用（DLxx/FRxx の profile）**  
    - Deck.D: `profile17(DL05P)`, `profile18(DL02P)`, `profile23(DL01.AP)`, `profile35(DL03P)`, `profile86(DL04P)` など  
    - Deck.C: `profile8(DL05P)`, `profile15(DL04P)`, `profile28(DL01.FP)`, `profile54(DL03.FP)`, `profile74(DL00.F)` など  
    - Deck.B: `profile68(DL05P)`, `profile30(DL04P)`, `profile71(DL01.FP)`, `profile114(DL03.FP)`, `profile85(DL00.F)` など  
    - Deck.A: `profile47(DL05P)`, `profile40(DL04P)`, `profile108(DL03.FP)`, `profile109(DL01.FP)`, `profile50(DL00.F)` など
  - **Wall 取り合い用**
    - Fore: `profile6(DL05.BCP)`, `profile26(DL01.BCP)`, `profile53(DL03.BCP)`, `profile100(DL04.CDP)`, `profile32(DL05.CDP)` など
    - Aft: `profile7(DL05.BCP)`, `profile75(DL01.CDP)`, `profile36(DL03.CDP)`, `profile87(DL04.CDP)`, `profile72(DL05.CDP)` など
  - **Side（FRxx）**
    - `profile44(FR10.CDP)`, `profile10(FR11.BCP)`, `profile5(FR08.BCP)`, `profile16(FR07.CDP)`, `profile3/20/89/118(FR15系)`, `profile21/45/76/113(FR14系)`, `profile37/43/77/101(FR13系)`, `profile57/65/95/106(FR09系)` など  
- **ループ変数方針（仕様として）**
  - 参考スクリプトは「ループ」よりも**個別定義（bracketPram1…）を直列で大量生成**しているが、編集指示（20個以上）を満たすには、元スクリプトの profile 群を以下のように**リスト化して反復適用**するのが自然：
    - Side FRxx の CDP/OAP/BCP/ABP をまとめた `side_profiles`
    - Fore/Aft の DLxx 系をまとめた `wall_profiles`
    - Deck.A/B/C/D の DLxx/FRxx 交点候補をまとめた `deck_profiles`
  - ただし EvoShip のブラケットは **Surfaces1/Surfaces2 が “面ペア”**であり、相手が Plate/Solid/PLS 指定か、Profile の FL/WF/WB/FR 等のどの面を使うかでパラメータセットが変わるため、**ブラケット種別ごとにテンプレート化してループ**する。

### ループ構造方針
- **方針1: Side FRxx 系（1505）の量産で“20個以上”を確実に満たす**
  - 対象: `profile44, profile10, profile5, profile16, profile52, profile113, profile3` およびそれらの `MirrorCopy` 結果
  - ループ単位:
    - `base = 各FR profile`
    - `sf1 = PLS(Deck.D solid2 など、対象甲板の厚み付きソリッド)`
    - `sf2 = base + ",FL"`
- **方針2: Fore/Aft DLxx 系（1501）を“壁×甲板のProfile同士”で付与**
  - 対象: `(Fore wall profile, Deck profile)` のペア  
    - 例: (profile62, profile61), (profile100, profile86), (profile32, profile17) など
  - ループ単位:
    - `base = wall側profile`
    - `sf1 = wall側 + ",FL"`
    - `sf2 = deck側 + ",FL"`
- **方針3: Mirror 部材は MirrorCopy 済み要素を“別要素”として同一ループに含める**
  - 元スクリプトは大量に `mirror_copiedXX` を生成しているため、**左右舷で同数ブラケット**を付けて不足が出ないようにする。

### パラメータ依存関係
- **BracketType**
  - **1501**: 典型的な三角ブラケット（Profile面同士の取り合いで多用）
    - 参考スクリプトでは `Thickness=7.999...` または `12` や `8.999...` 等、相手部材に合わせるが、元スクリプト側では板厚10のソリッドが多く、**Profileが125x75x7等なら t=8、フランジ/FB系なら t=12** といった選択が整合しやすい。
  - **1505**: PLS面（板/ソリッド）に対して Profile を受けるタイプ（Side FRxx と Deck の取り合いで多用）
    - `BracketParams=["200"]` が参考スクリプトに存在（1505の代表パラメータ）
- **Surfaces1/Surfaces2 の型**
  - Profile面指定: `profileX[0]+",FL"`（ほか WF/WB/FR もあり得る）
  - PLS指定: `["PLS","False","False", nx, ny, nz, solid_or_profile]`
    - 例: 甲板面なら概ね `0,0,-1` 系、壁なら `±1,0,0` 系、など参照スクリプトに準拠
- **寸法（Sf1DimensionType / Sf2DimensionType）**
  - 参考スクリプトの頻出:
    - `Sf?DimensionType=1531`, `Params=["200","15"]`（一般的な長さ/オフセット指定）
    - `Sf?DimensionType=1541`, `Params=["0","100"]`（端要素基準の距離指定に使う）
  - **1505では `Sf1EndElements` が重要**
    - 参考例: `Sf1EndElements=["PLS",... , profile1[0]]` のように、PLS面側の基準端（端点・交点）を固定する
- **Scallop（切欠き）**
  - 多くが `Scallop1Type=1801`
  - `Scallop1Params=["0"]` または `["50"]` が混在  
    - “大きめブラケット/フランジ付き”では 50 が使われやすい（参考 bracketPram2/3/5/12）
- **FlangeType / FlangeParams**
  - 参考スクリプトでは 1501 でも一部に `FlangeType=262` を付ける（例: bracketPram2/3/5/12）
  - 元スクリプトにおいても、FB/アングル/チャンネル等の取り合い（`ProfileType=1201` 等）でブラケットを作る場合は、同様にフランジ付与が必要になり得る。

### 注意事項
- **Surfaces1/Surfaces2 は「面ペア」であり、要素（Profile/Solid）そのものではない**
  - Profileの場合も `",FL"` 等の面識別子が必須。
  - PLSの場合は **配列形式**（"PLS", 反転フラグ, ベクトル, 対象要素）を維持すること。
- **BaseElement は必ず“取り付け側の主部材（多くは Profile）”を指定**
  - 参考では Wall側や Side FR側 profile を BaseElement にすることが多い。
- **1505（PLS×Profile）の場合、Sf1EndElements を設定しないと位置が不安定になりやすい**
  - 参考スクリプトで 1505 に `Sf1EndElements` が高頻度で入っている点は踏襲すべき。
- **MirrorCopy した部材に対してもブラケットが必要**
  - 元スクリプトは左右対称要素を大量生成しているため、片舷のみのブラケット配置だと仕様不足（20個以上）になりやすく、またモデルとしても不完全になりやすい。
- **板ソリッド（solid1/2/3/4）を PLS 参照に使う場合、法線方向（nx,ny,nz）の符号を誤るとブラケットが裏返る**
  - 参考では Deck 面は `0,0,-1` 系、Wall は `±1,0,0` 系が多い。元スクリプトの座標系（PL,X/PL,Zの使い方）に合わせて統一する必要がある。
