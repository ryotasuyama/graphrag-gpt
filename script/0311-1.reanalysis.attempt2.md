### ループ構造方針
- **方針1: Wall.Fore / Wall.Aft × Deck.DL の自動ペアリング**
  - `Wall.Fore.DLxx.*` は `Deck.*.DLxx.F / FP` のみを接続候補とする
  - `Wall.Aft.DLxx.*` は `Deck.*.DLxx.A / AP` のみを接続候補とする
  - `CDP` 同士の主接続は参考スクリプトどおり `Wall.CDP` を `BaseElement` にし、Deck 側は `Surfaces2` に `FL` を指定
  - `DL02` のT材のみ `PLS` ベース面参照を使う

- **方針2: Side Wall FR 系の一括ループ**
  - 対象 FR: `07,08,09,10,11,12,13,14,15`
  - FRごとに Deck 側の存在を確認して候補化
  - 2種類に分ける:
    - `1505`: `PLS + solid(deck)` 対 `SideWall.FL`
    - `1501`: `Deck.FR profile FL` 対 `SideWall.FL`

- **方針3: Deck レベルごとのループ**
  - Deck D, C, B, A ごとにテーブルを持ち、
    - `DL00/01/03/04/05`
    - `FR09/13`
  - 参照可能な deck profile 名 suffix を見て Fore/Aft 一致を判定する

- **方針4: 参考スクリプトの実装継承**
  - `BracketType=1501`
    - `Surfaces1=[wall_or_deck_profile+",FL"]`
    - `Surfaces2=[paired_profile+",FL"]`
  - `BracketType=1505`
    - `Surfaces1=["PLS","False","False","0","-0","-1",solidX]`
    - `Surfaces2=[side_profile+",FL"]`
    - `Sf1EndElements=["PLS","False","False","-0","-1","-0",deck_dl_profile]`

- **方針5: 今回のエラー再発防止ルール**
  - `Wall.Aft.DL00.BC` のような Aft 壁プロファイルでも、**Deck 側との名称整合だけで採用しない**
  - `1501 / FL-FL` は、実際に参考スクリプトまたは成功済み候補と同系統の面ペア成立実績がある組のみ確定採用する
  - とくに `profile80 (Wall.Aft.DL00.BC) × profile33 (Deck.D.DL00.A)` は **名称上は Aft 整合でも COM 例外実績あり**のため除外する
  - `mirror_copied***` は Fore/Aft 判定の代用品にせず、Aft 実体 deck profile が無い場合は **保留または除外**とする

### パラメータ依存関係
- **BracketType の使い分け**
  - `1501`
    - 壁縦材と deck 縦材の直接 FL-FL 接続
    - Side FR でも deck 横桁/フレームの FL と直接結ぶ場合に使用
  - `1505`
    - Side Wall と deck plate/solid 面を基準にしたブラケット
    - `BracketParams=["200"]` を伴うケースを参考スクリプトどおり踏襲

- **Thickness**
  - 参考スクリプトの傾向:
    - 小型L形/FB相当接続: `7.9999999999999964` ≒ **8 mm**
    - Side Wall CDP 系: `9.9999999999999982` ≒ **10 mm**
    - T材接続（DL02）: `12`
    - Deck FR P/C 系の一部: `8.9999999999999982` ≒ **9 mm**
  - したがって仕様上の割当は以下が妥当:
    - DL01/03/04/05・DL00 の一般接続: **8 mm**
    - Side FR の 1505: **10 mm**
    - DL02 T材: **12 mm**
    - FR09/FR13 の deck framed bracket: **8〜9 mm**

- **Sf1 / Sf2 寸法**
  - `1501` 一般:
    - `Sf1DimensionType=1531`
    - `Sf2DimensionType=1531`
    - 代表値:
      - DLブラケット: `["200","15"]`
      - FRブラケット: `["250","15"]` または `["300","15"]`
  - `1505` Side Wall:
    - `Sf1DimensionType=1541`
    - `Sf1DimensonParams=["0","100"]`
    - `Sf2DimensionType=1531`
    - `Sf2DimensonParams=["200","15"]`

- **Flange 指定**
  - T材接続や deck girder 系ではフランジ付きブラケットを使用
  - 参考スクリプトの DL02 / FR09 / FR13 で確認される典型:
    - `FlangeType=262`
    - `FlangeParams=["100","30","29.999999999999996","30","30","1"]`
    - または `["75","30","29.999999999999996","30","50","1"]`
  - よって `DL02` と一部 FR 交差部ブラケットに限定する

- **Scallop**
  - 標準:
    - `Scallop1Type=1801`
    - `Scallop1Params=["0"]` または `["50"]`
    - `Scallop2Type=0`
  - T材や大断面側では `50`
  - 一般小型ブラケットは `0`

- **End1/End2 の必要性**
  - ブラケット自体に `End1/End2` は不要
  - ただし `Sf1EndElements` / `Sf2EndElements` は 1505 や 1541 寸法使用時に必要
  - 特に参考スクリプトでは `1505 + Sf1DimensionType=1541` のとき **`Sf1EndElements` が必須**

- **今回失敗した候補に関する依存関係の見直し**
  - 失敗原因は厚みや寸法不足ではなく、**面ペア成立性・ブラケット平面定義の不成立**とみなすのが妥当
  - `profile121 (Wall.Aft.DL00.CD) × profile33 (Deck.D.DL00.A)` は成功系として維持できる一方、
    `profile80 (Wall.Aft.DL00.BC) × profile33 (Deck.D.DL00.A)` は同じ DL00-Aft でも不成立だった
  - よって、**DL00 は Aft/Fore suffix だけでなく帯区分（CD / BC / AB / OA）ごとの成功実績差も見る必要がある**

### 注意事項
- **Fore/Aft 整合ルールは必須**
  - `Wall.Fore.DLxx.*` ↔ `Deck.*.DLxx.F / FP`
  - `Wall.Aft.DLxx.*` ↔ `Deck.*.DLxx.A / AP`
  - 例:
    - `profile62 (Wall.Fore.DL01.CDP)` は `profile61 (Deck.D.DL01.FP)` と接続可
    - `profile75 (Wall.Aft.DL01.CDP)` は `profile23 (Deck.D.DL01.AP)` と接続可
    - `profile121 (Wall.Aft.DL00.CD)` は `profile33 (Deck.D.DL00.A)` と接続可
    - `profile80 (Wall.Aft.DL00.BC)` は名称上は Aft でも、実行失敗のため採用不可

- **今回の除外判断**
  - `profile80 = HK.Casing.Wall.Aft.DL00.BC` と `profile33 = HK.Casing.Deck.D.DL00.A` の組は除外
  - 理由:
    1. 実際に `bracketPram27` で COM 例外が発生している
    2. 同じ Aft-DL00 でも、成功しているのは `CD` 帯の `profile121 × profile33`
    3. `BC` 帯では同じ `1501 / FL-FL` が安定しないため、帯区分差を考慮して除外すべき

- **成功済み候補の維持**
  - 指示どおり、成功済み bracket 1〜26 相当の候補は維持する

    6. Aft 側 BCP/ABP/OAP は実体 A/AP deck 側が確認できる場合のみ追加

- **DL00 の扱い**
  - `DL00` は `F/A` と `OA/AB/BC/CD` が混在するため、単純な suffix 判定だけでなく帯区分も確認する
  - `Deck.D.DL00.A` は Aft 側 Deck 実体
  - `Deck.*.DL00.F` は Fore 側 Deck 実体
  - ただし Wall 側の `CD/BC/AB/OA` のどれが安定に接続できるかは成功実績ベースで判定する

- **Side Wall（FR系）は方向制約なし**
  - ただし接続先 Deck プロファイルの存在確認は必要
  - deck 側の `FR09P`, `FR13P` 等が無い FR では 1501 型ブラケットを作らない
  - 1505 型では `solid(deck)` と `Sf1EndElements` の存在を必ず確認する

- **Surface 指定の型差**
  - 参考スクリプトでは 2種類存在:
    - `profile+",FL"` / `profile+",WF"` / `profile+",WB"`
    - `["PLS","False","False",nx,ny,nz,element]`
  - `1501` は profile 面同士
  - `1505` と T材特殊部は `PLS` ベース面を使う
  - T材系 (`ProfileType=1201`) のブラケットでは profile 本体 `[0]` と flange/body `[1]` の使い分けに注意する

- **ミラー体の使用制限**
  - `mirror_copied***` は補助形状であり、Fore/Aft 整合の証明には使わない
  - とくに Aft 側 Wall に対して mirror した Fore 系 Deck を当てる構成は原則禁止
  - 実体プロファイル名に `A / AP` が無いものは Aft 側候補に昇格させない

- **30個以上の配置保証**
  - 修正後も、**確定候補だけで 43件**を維持できる
  - したがって、失敗候補を 1 件除外しても要件未達にはならない
  - むしろ、安全性を優先して不安定候補を除外する方が妥当である
