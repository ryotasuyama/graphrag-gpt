### ループ構造方針
- 元スクリプトはループ未使用の**完全展開型**ですが、実質的には以下のグループで整理できます。
- **グループ1: Fore/Aft の DL 系ブラケット**
  - `DL01, DL03, DL04, DL05` を対象に、
    - Wall.Fore 系 `profile62, profile117, profile100, profile32`
    - Wall.Aft 系 `profile75, profile36, profile87, profile72`
    - Deck 系 `profile61, profile35, profile86, profile17`
  - 各 DL 番号ごとに `1501` を1対1で適用する方針。
- **グループ2: DL02 のフランジ付ブラケット**
  - `profile59` / `profile19` を対象に、`profile18[1]` を含む `PLS` 面参照で `1501`
  - 参考スクリプト通り `FlangeType=262` を使うのが望ましい。
- **グループ3: Side FR 系の plate 対 stiffener ブラケット**
  - `FR07, FR08, FR10, FR11, FR12, FR14, FR15`
  - Side 壁縦材 `profile16, 12, 44, 111, 52, 113, 3`
  - これらは `solid2` を `Surfaces1` に使う `1505` パターンで統一可能。
- **グループ4: Side FR09 / FR13 の大断面材接合**
  - `profile65 ↔ profile64`
  - `profile43 ↔ profile42`
  - さらに deck girder の `WF/WB` を使う補助ブラケットも追加可能。
- **グループ5: Deck C 側の同種配置**
  - Deck D と同じ構図を Deck C にも適用し、必要個数を満たす。
- **グループ6: 必要ならミラー側も個別に追加**
  - 既存 `mirror_copied*` を基準に、P/S 両舷へ同型ブラケットを展開できる。

### パラメータ依存関係
- **BracketType の選定**
  - `1501`  
    - 部材同士の交差、特に `FL × FL`、`WF/WB × FL` に使用
    - 例: Fore/Aft の縦材と Deck 縦材の接合
  - `1505`  
    - `PLS` 面と profile フランジ面の接合
    - 側壁材が deck plate/solid に取り付くケースに適合
- **Thickness**
  - 参考スクリプトから以下の依存が読み取れる
    - 小型アングル/FB系接合: `7.9999999999999964` ≒ **8t**
    - Side FR 系一般: `9.9999999999999982` ≒ **10t**
    - 1201 系フランジ材接合: **12t**
    - Deck girder 系 `1003` 接合: `8.9999999999999982` ≒ **9t**
- **寸法指定**
  - `Sf1DimensionType=1531`, `Sf2DimensionType=1531`
    - 一般に `["200","15"]` または `["250","15"]`, `["300","15"]`, `["800","15"]`
  - `Sf1DimensionType=1541`
    - `PLS` 基準のときに多用
    - 例: `["0","100"]`, `["0","80"]`
- **FlangeType / FlangeParams**
  - DL02 系や Deck girder 接合では追加フランジが必要
  - 代表値
    - `FlangeType=262`
    - `FlangeParams=["100","30","29.999999999999996","30","30","1"]`
    - または `["75","30","29.999999999999996","30","50","1"]`
- **Sf1EndElements / Sf2EndElements**
  - 特に `1505` では必須に近い
  - `PLS` 側寸法起点の安定化に使用
  - 参考スクリプトでは `profile1[0]` や `profile4[1]+",FR"` を指定している
- **Scallop**
  - `Scallop1Type=1801`
  - `Scallop1Params=["0"]` または `["50"]`
  - `Scallop2Type=0`
  - 参考スクリプトのまま踏襲するのが安全

### 注意事項
- **Surfaces1 / Surfaces2 は単なる部材名ではなく「面指定」**
  - 例:
    - `profileX[0]+",FL"`
    - `profileY[0]+",WF"`
    - `profileZ[0]+",WB"`
    - `["PLS","False","False","0","-0","-1",solid2]`
- **PLS 参照のブラケットは EndElements を付ける**
  - `1505` の場合、`Sf1EndElements` を省くと位置決定が不安定になりやすい。
- **BaseElement は通常 profile 本体**
  - Deck plate や solid ではなく、ブラケットを従属させたい縦材・梁材を `BaseElement` にする。
- **DL02 系の 1201 プロファイルは flange body を返す**
  - `profile18[0]` と `profile18[1]`、`profile19[0]` と `profile19[1]` のように返り値が複数あるため、どちらを面参照に使うかを参考スクリプト準拠で区別する必要がある。
- **Side FR13 / FR09 は 2パターンある**
  - `1501` の `FL × FL`
  - `1501` の `WF/WB × FL + FlangeType=262`
  - どちらも参考スクリプトに実例があるため、両方使えば配置数を増やしつつ整合性を保てる。
- **20個以上にするなら、Deck D を主軸に構成するのが最も自然**
  - 参考スクリプトでも Deck D まわりのブラケット実績が最も多い。
- **左右対称が必要な場合**
  - 既存の `mirror_copied*` プロファイルに対して同型ブラケットを追加するか、
  - 元材側と mirrored 側を別々に `BaseElement` 指定して作る。

以上より、元スクリプトに対しては、**Deck D 周辺を中心に 20〜30個のブラケットを個別追加する仕様**が最も参考スクリプトに整合し、API上も安全です。
