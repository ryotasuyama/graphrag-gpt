### 対象部材とループ変数
- **Deck（板）系 Solid/Sheet**
  - `extrude_sheet1`: HK.Casing.Deck.B（Sheet）
  - `extrude_sheet2`: HK.Casing.Deck.A（Sheet）
  - `extrude_sheet4`: HK.Casing.Deck.C（Sheet）
  - `extrude_sheet5`: HK.Casing.Deck.D（Sheet）
  - `solid1`: HK.Casing.Deck.C（ThickenされたSolid）
  - `solid2`: HK.Casing.Deck.D（ThickenされたSolid）
  - `solid3`: HK.Casing.Deck.B（ThickenされたSolid）
  - `solid4`: HK.Casing.Deck.A（ThickenされたSolid）

- **Wall（板）系 Sheet / Solid**
  - `extrude_sheet3`: HK.Casing.Wall.Aft（Sheet）
  - `extrude_sheet8`: HK.Casing.Wall.Fore（Sheet）
  - `extrude_sheet6`: HK.Casing.Wall.SideP（Sheet）
  - `extrude_sheet9`: HK.Casing.Wall.SideS（Sheet）
  - `solid5`: HK.Casing.Wall.Aft.CD（Solid）
  - `solid9`: HK.Casing.Wall.Aft.OA（Solid）
  - `solid10`: HK.Casing.Wall.Fore.CD（Solid）
  - `solid6/solid7/solid14`: Fore側の派生Solid（AB/OA/BC 等）
  - `solid8/solid12/solid15/solid16` とそれらの `MirrorCopy`: Side系の派生Solid

- **Profile（主に Stiffener/FB/Angle/Channel 相当）**
  - Fore/Aft の縦通材（DLxx）  
    - Fore: `profile26`(DL01.CDP), `profile94`(DL00.CD), `profile32`(DL05.CDP), `profile100`(DL04.CDP), `profile59`(DL02.CDP), `profile117`(DL03.CDP) など  
    - Aft: `profile75`(DL01.CDP), `profile121`(DL00.CD), `profile72`(DL05.CDP), `profile87`(DL04.CDP), `profile19`(DL02.CDP), `profile36`(DL03.CDP) など
  - Side のフレーム（FRxx）  
    - `profile16`(FR07.CDP), `profile12`(FR08.CDP), `profile44`(FR10.CDP), `profile10`/`profile111`(FR11), `profile52`(FR12), `profile43`/`profile101`(FR13), `profile21`/`profile113`(FR14), `profile3`(FR15) 等（各種 BCP/CDP/OAP/ABP が混在）

- **ループ変数候補（ブラケット配置を漏れなくするための分類キー）**
  - **DL系（Fore/Aft）**: `DL00, DL01, DL02, DL03, DL04, DL05`
  - **FR系（Side）**: `FR07, FR08, FR10, FR11, FR12, FR14, FR15`（加えてFR09/FR13は「接続ブラケット」タイプが必要）
  - **基準Deck**: “Deck.C / Deck.D” がブラケット取り合いの中心（参考スクリプトでは Deck.D が中心、元スクリプトは Deck.C/D/A/B すべて生成）

### ループ構造方針
- **DL（Fore/Aft）→Deck ブラケット（BracketType=1501）**
  - 対象：`DL00, DL01, DL03, DL04, DL05` は基本的に **Surfaces1=[DL側 profile,FL] / Surfaces2=[Deck側 profile,FL]**
  - `DL02` のみ例外で、参考スクリプト同様に **PLS×PLS + FlangeType=262** を使う。
  - 実装上は次の2つの辞書（または配列）で整理してループ化するのが安全：
    - `dl_simple_pairs = [(fore_dl01, deck_dl01), (fore_dl03, deck_dl03), ... , (aft_dl05, deck_dl05)]`
    - `dl02_pairs = [(fore_dl02, deck_dl02), (aft_dl02, deck_dl02)]`

- **Side FRxx（CDP）→Deck Solid（BracketType=1505）**
  - 対象：FR07/08/10/11/12/14/15 等、**ProfileType=1002のSide縦材**は共通で `BracketType=1505`。
  - `Surfaces1` は参考通り **PLSでDeck側Solidを指定**（元スクリプトでは Deck.D の `solid2` を基準にするのが整合的）。
  - `Surfaces2=[side_profile,FL]`
  - これも `side_fr_profiles = [profile16[0], profile12[0], profile44[0], profile52[0], profile113[0], profile3[0], ...]` のように配列化してループ。

- **Side FR09/FR13 等（1003系）→Deck梁（1003系）ブラケット（BracketType=1501）**
  - 参考スクリプトでは FR09P/FR13P と Side FR09/FR13 に **1501（FL-FL）** が設定されているため踏襲。
  - `fr_bridge_pairs = [(side_fr09_cdp, deck_fr09p), (side_fr13_cdp, deck_fr13p)]` を作ってループ。

### パラメータ依存関係
- **共通（参考スクリプト準拠の代表ロジック）**
  - `DefinitionType=1`, `MaterialName="SS400"`, `UseSideSheetForPlane=False` を基本とする。
  - `BracketType`
    - **1501**：一般的な三角ブラケット（FL-FL、WF-FL、WB-FL、またはPLS×PLS）
    - **1505**：Deck板（Solid）×Side縦材（FL）の“板基準ブラケット”。`BracketParams=["200"]` が必須。
  - `Thickness`
    - DL01/DL03/DL04/DL05/Fore-Aft系の小ブラケット：**約8mm（"7.9999..."）**
    - Side FRxx（1505）：**約10mm**
    - DL02（フランジ付き、PLS×PLS）：**12mm**
    - Deck梁（FR09P/FR13Pのような300x90系）関連：**約9mm**を採用するケースあり（参考では 8.999...）
  - `Mold`
    - DL02（フランジ付）では参考通り `"Mold"="-"` を優先
    - その他は接続方向により `"+"` が基本（元スクリプトのプロファイル向きに合わせて最終調整）
  - `Scallop1Type=1801`
    - `Scallop1Params=["0"]` または `["50"]`
    - DL02や梁周りは `["50"]` が多い（参考準拠）
  - `Sf1DimensionType / Sf2DimensionType`
    - **1531**：一般寸法（例 `"200","15"` や `"250","15"`、DL02は `"800","15"`）
    - **1541**：端部要素で止める（例 `"0","80"` や `"0","100"`）  
      - 参考では **Sf1EndElements** を併用（FRxx→Deckの1505、梁→縦通の1501(1541) など）

- **End要素（Sf1EndElements / Sf2EndElements）の要否**
  - **必須になりやすいケース**
    - `Sf1DimensionType=1541` を使う場合（端部規制）：参考スクリプトでは `Sf1EndElements` が必ずセットされている。
    - Side FRxx→Deck（BracketType=1505）では `Sf1EndElements=["PLS",... , profile1[0]]` のように、Deck側の“止まり”要素を入れている。
  - **不要（省略可）になりやすいケース**
    - 両側とも `1531` で単純寸法指定の場合（DL01等の小ブラケット）。

### 注意事項
- **Surfaces1 / Surfaces2 の指定形式が2系統ある**
  1. `["element,FL"]` / `["element,WF"]` / `["element,WB"]` のような **部材面ラベル指定**
  2. `["PLS","False","False","nx","ny","nz", element]` のような **PLS（基準平面/ベクトル）指定**  
     - 参考では Deck板（Solid）基準にする場合に多用される（BracketType=1505、DL02の一部など）。
- **BracketType=1505 は BracketParams が必須**
  - 参考では `BracketParams=["200"]` が必ず入り、これが無いと形状定義が不完全になる可能性が高い。
- **DL02は他DLと同じ作り方にしない**
  - 参考スクリプトでDL02のみ「PLS×PLS + FlangeType=262」を採用しており、単純な FL-FL ブラケット（1501, 200/15）に置換すると形状・干渉・意図（フランジ付き補強）が崩れる。
- **BodyDivideByCurves による分割を行う場合**
  - 参考スクリプトでは一部ブラケット作成後に `BodyDivideByCurves` を実施して端部形状を整えている（特定SketchPlane上のラインで切断）。
  - 元スクリプト側で同等の仕上げが要求される場合、ブラケット生成直後に同様の分割工程を追加する設計を検討する（ただし編集指示は「ブラケットをつけてください」なので必須工程ではない）。
- **左右対称（MirrorCopy）部材へのブラケット**
  - 元スクリプトは多数の `MirrorCopy` が存在するため、ブラケットも **P側で作ったものをMirrorする**か、**S側のベース要素（mirror_copiedXX）にも同条件で別作成**する必要がある。  
  - 参考スクリプトは左右を個別に作るというより、必要箇所を都度生成しているため、漏れ防止には「P側生成→MirrorCopyでS側」という方針が堅牢。
