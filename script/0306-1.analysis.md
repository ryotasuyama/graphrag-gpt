### ループ構造方針
- **方針A（Side系・BracketType=1505 を量産して50超え）**
  - 対象：`profile16(FR07)`, `profile12(FR08)`, `profile44(FR10)`, `profile111(FR11)`, `profile52(FR12)`, `profile113(FR14)`, `profile3(FR15)` と、それぞれの `MirrorCopy` 結果
  - ループ変数例：`side_profiles = [profile16[0], profile12[0], profile44[0], profile111[0], profile52[0], profile113[0], profile3[0], mirror_of_each...]`
  - 各要素に対し、以下を共通設定としてブラケット生成する
    - `BracketType=1505`, `BracketParams=["200"]`
    - `Surfaces1 = ["PLS",... , solid2]`（Deck.D の solid を基準面にする）
    - `Surfaces2 = [prof + ",FL"]`
    - `Sf1DimensionType=1541`, `Sf1DimensonParams=["0","100"]`
    - `Sf1EndElements = ["PLS",... , profile17[0]]`（Deck.D の DL05P 等を端部拘束の基準に）
    - `Sf2DimensionType=1531`, `Sf2DimensonParams=["200","15"]`
- **方針B（DL系・BracketType=1501 を Fore/Aftに展開）**
  - 対象：`profile62(DL01 Fore CDP)`, `profile75(DL01 Aft CDP)`, `profile100(DL04 Fore CDP)`, `profile87(DL04 Aft CDP)`, `profile117(DL03 Fore CDP)`, `profile36(DL03 Aft CDP)`, `profile32/72(DL05)` 等
  - ループ変数例：`dl_pairs = [(fore_profile, deck_profile), (aft_profile, deck_profile), ...]`
  - 各ペアに対し、以下を共通設定としてブラケット生成する
    - `BracketType=1501`
    - `Surfaces1=[p1+",FL"]`, `Surfaces2=[p2+",FL"]`
    - `Sf1DimensionType=1531`, `Sf1DimensonParams=["200","15"]`
    - `Sf2DimensionType=1531`, `Sf2DimensonParams=["200","15"]`
- **50個超えの達成方法（最短）**
  - 方針Aだけでも、Side FRxx の **BCP/CDP/OAP/ABP など複数バリエーション**と Mirror を含めて対象を広げると 50 を超えやすい。
  - 方針A + 方針B を併用すると確実。

### パラメータ依存関係
- **Thickness（板厚）**
  - 参考スクリプトでは接続相手により厚みが分岐している。
    - BracketType=1501（一般三角ブラケット系）  
      - 例：`Thickness="7.9999..."`（おおむね 8mm）  
      - 例：DL02 等のフランジ付Profileに絡む場合 `Thickness="12"`
    - BracketType=1505（スロープ/パラメトリック系）  
      - 例：`Thickness="9.9999..."`（おおむね 10mm）
  - 元スクリプトでは ProfileParams から板厚を直接は引けないため、仕様としては
    - **DL02系（ProfileType=1201, plate厚12相当）→ bracket厚12**
    - **それ以外の一般アングル/チャンネル（ProfileType=1002/1003）→ bracket厚8〜10**
- **BracketType の使い分け**
  - **1501**：`Surfaces1` と `Surfaces2` を **Profile面（FL/WF/WB 等）同士**で与えるパターンが多い。必要に応じて `FlangeType=262` を併用。
  - **1505**：`Surfaces1` を **PLS（平面/ソリッド参照）**で与え、`Surfaces2` を Profile の `,FL` にするパターンが多い。`BracketParams=["200"]` が付与される。
- **Sf1EndElements / Sf2EndElements**
  - 参考スクリプトで **1505** の場合に `Sf1EndElements` を使い、Deck側の端部拘束（基準となる Profile や PLS）を与えている。
  - 参考スクリプトで **1501 + WF拘束** の場合に `Sf1EndElements=[profile4[1]+",FR"]` のような端部制限が出てくる（梁端・フレーム等）。

### 注意事項
- **Surfaces1 / Surfaces2 は“面ペア”でブラケット形状が決まる**
  - 参考スクリプトでは以下の2系統がある：
    1) `SurfacesX=[profile+",FL/WF/WB"]`（Profileの面種別を明示）  
    2) `SurfacesX=["PLS","False","False", vx,vy,vz, element]`（平面法線ベクトル+参照要素で面を定義）
- **Mold（符号）と Reverse（RevSf1/RevSf2/RevFlange）**
  - 参考スクリプトでは Fore/Aft や Profileの向きにより `Mold="+"` と `Mold="-"` が混在する。
  - 元スクリプトは左右ミラーが大量にあるため、Mirror側では**RevSf1/RevSf2 または Mold の整合**が崩れる可能性がある（左右でブラケットが裏返る）。
- **DL02 系（ProfileType=1201）に絡むブラケットは FlangeType を伴う場合がある**
  - 参考の bracketPram2/5 のように `FlangeType=262` と `FlangeParams` を要求するパターンがあり、単純な 1501 より設定項目が増える。
- **実体分割（BodyDivideByCurves）の存在**
  - 参考スクリプトはブラケット作成後に `BodyDivideByCurves` でトリム/分割している。  
  - 元スクリプト側で同等の見た目・干渉回避を目指すなら、ブラケット作成後の分割工程も仕様に含める必要がある（今回の編集指示は“50個以上”のみだが、干渉回避には重要）。
