#### 失敗候補の修正結果
- **除外対象**
  - 旧候補 `bracketPram26`  
    `profile117[0]+",FL"` × `profile54[0]+",FL"`  
    `BracketName="HK.Casing.Wall.Fore.DL03.Deck.CP"`

- **除外理由**
  - 実行時に `CreateBracket` で COM 例外が発生しており、理論上の Fore/Aft 整合だけでは成立しないことが確認された
  - `profile117` は `HK.Casing.Wall.Fore.DL03.CDP`、`profile54` は `HK.Casing.Deck.C.DL03.FP` で、名称上は Fore 側整合がある
  - しかし Deck.C 側では、同じ `1002` 系でも **端部拘束、交差位置、または有効面の取り方が Deck.D / Deck.B / Deck.A の成功例と完全一致していない可能性** が高い
  - 参考スクリプトの直接成功根拠は Deck.D 系に集中しており、Deck.C 側 DL03.FP への単純横展開は安全候補としては弱い
  - よってこの候補は **修正採用ではなく除外** が妥当

- **代替方針**
  - 候補数要件を満たすため、除外した 1 件の代わりに、**既に前回候補に入っていて失敗報告のない Fore整合済み Deck.A/B 側候補を維持**
  - 具体的には `Deck.B/A` の `DL03.FP`, `DL01.FP`, `DL00.F`, `DL05P` を優先採用し、総数 **35件** を確保する
  - `Deck.C DL03.FP` は今回のみ安全候補から外し、再検証対象とする

### ループ構造方針
- **Wall.Fore × Deck(F/FP/P) を DL ごとにループ**
  - 対象DL: `DL00, DL01, DL03, DL04, DL05`
  - 許可ペア:
    - `Wall.Fore.DLxx.*` → `Deck.X.DLxx.F` / `Deck.X.DLxx.FP`
    - 実スクリプト上の成功実績により、`Deck.D.DL03P`, `Deck.D.DL04P`, `Deck.X.DL05P` も個別採用可
  - 安全候補:
    - Deck.A/B の `DL00.F`, `DL01.FP`, `DL03.FP`, `DL05P`
    - Deck.C の `DL00.F`, `DL01.FP`, `DL05P`
    - Deck.D の `DL00.F`, `DL01.FP`, `DL03P`, `DL04P`, `DL05P`
  - **除外**
    - `Deck.C.DL03.FP` への `Wall.Fore.DL03.CDP` 接続は今回の失敗実績により除外
    - `Deck.A/B/C.DL04P` への Fore 横展開は引き続き除外

- **Wall.Aft × Deck(A/AP) を DL ごとにループ**
  - 許可ペア:
    - `Wall.Aft.DLxx.*` → `Deck.X.DLxx.A` / `Deck.X.DLxx.AP`
  - 現スクリプト上で明確に成立している安全候補:
    - `Deck.D.DL00.A`
    - `Deck.D.DL01.AP`
  - `Deck.A/B/C` 側には A/AP 明示プロファイルが不足しているため、**それらとの Aft ブラケットは候補除外**
  - `DL03/DL04/DL05` の Aft 側は、成功済み 1〜25 に含まれる Deck.D 接続のみ維持

- **Wall.Side(FR) × Deck面 or Deck FRプロファイル**
  - `FR07, 08, 10, 11, 12, 14, 15` は **BracketType=1505**
  - `FR09, FR13` は Deck 側 FR 材との **BracketType=1501**
  - Side Wall には Fore/Aft 制約を適用しない
  - ただし接続先として Deck 側ソリッド面または FR プロファイルの存在確認は必須

- **FB/PL 系の DL02 接続は独立処理**

  - `Surfaces1/2` は `PLS` 配列形式
  - Flange 付きブラケットを使用
  - ここは通常の `FL–FL` ループに混ぜない

### パラメータ依存関係
- **BracketType の選択**
  - **1501**
    - 壁縦通材 `CDP` と Deck 縦通材の **FL–FL 接続**
    - Deck FR 材と Wall.Side FR 材の **FL–FL 接続**
    - Deck 梁の `WF/WB` 面と FR 材 `FL` の接続
    - DL02 の **PLS–PLS 接続**
  - **1505**
    - Wall.Side FR 材と Deck 板ソリッド面の接続
    - 参考スクリプト準拠で
      - `Surfaces1=["PLS",...,solid2]`
      - `Surfaces2=[profileX[0]+",FL"]`
      - `Sf1EndElements=["PLS",...,profile17[0]]`

- **Thickness**
  - 一般 DL ブラケット（125x75 系）: **8t**
  - DL02 FB/PL 系: **12t**
  - Side FR の Deck 板直付け（1505）: **10t**
  - FR09 / FR13 系との接続: **9t 前後**
  - 今回失敗した `Fore DL03 × Deck.C DL03.FP` も厚みは標準 8t であり、**失敗要因は厚みではなく接続面成立性**と判断

- **Sf1 / Sf2 寸法**
  - FL–FL の標準:
    - `Sf1DimensionType=1531`
    - `Sf2DimensionType=1531`
    - `Sf1DimensonParams=["200","15"]`
    - `Sf2DimensonParams=["200","15"]`
  - FR13 / FR09 の大きめ材:
    - `["250","15"]` または `["300","15"]`
  - 1505 板面接続:
    - `Sf1DimensionType=1541`
    - `Sf1DimensonParams=["0","100"]`
    - `Sf1EndElements` 必須
    - `Sf2DimensionType=1531`
    - `Sf2DimensonParams=["200","15"]`

- **FlangeType / FlangeParams**
  - DL02 の PLS 型で必要
  - 参考値:
    - `FlangeType=262`
    - `FlangeParams=["100","30","29.999999999999996","30","30","1"]`
  - Deck.D の FR09 / FR13 の WF 接続では
    - `FlangeType=262`
    - `FlangeParams=["75","30","29.999999999999996","30","50","1"]`
    - `Sf1EndElements` を併用

- **BaseElement**
  - Wall.Fore / Wall.Aft 接続 → 壁側 `CDP`
  - Wall.Side × Deck 面接続 → Side 側 `CDP`
  - Deck FR × Wall.Side FR 接続 → 成功例では Deck 側 `FR` を `BaseElement`
  - 1003 梁系 Deck 材へ横展開する場合は、単純な `FL–FL` ではなく `WF/WB` や端部拘束の必要性を個別確認する
