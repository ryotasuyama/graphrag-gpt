   - `Surfaces2=[deck_profile+",FL"]`

   - 主に `BracketType=1505` または `1501`
   - `1505` は Deck 板ソリッド面 (`PLS ... solidX`) と Side profile の FL を結ぶパターン
   - `1501` は Deck FR profile と Side FR profile の FL 同士を結ぶパターン

### ループ構造方針

ブラケット追加を漏れなく行うには、参考スクリプトのような**個別生成の列挙**ではなく、仕様上は以下の3ループに分けるのが最も安全です。

- **ループ1: Fore/Aft の DL 系ブラケット**
  - Wall プロファイル群を DL ごとに分類する
    - Fore: `profile62,59,117,100,32,94`
    - Aft: `profile75,19,36,87,72,121`
  - Deck プロファイル群を Deck A/B/C/D ごとに DL・方向サフィックスで分類する
  - 各 Wall プロファイルについて、同一 DL の Deck 候補を検索
  - Fore は `.F/.FP` のみ、Aft は `.A/.AP` のみ採用
  - 生成形式は原則 `BracketType=1501`

- **ループ2: Side Wall ↔ Deck 板ブラケット**
  - Side CDP プロファイル群:
    - `profile16,12,44,111,52,113,3`
  - Deck 板基準:
    - 基本は `solid2`（Deck.D）を優先
  - 生成形式:
    - `BracketType=1505`
    - `Surfaces1=["PLS",... , solid2]`
    - `Surfaces2=[side_profile+",FL"]`
    - `Sf1EndElements=["PLS",... , profile17[0]]`
  - 参考スクリプトの side bracket パターンを踏襲

- **ループ3: Side Wall ↔ Deck FR プロファイルブラケット**
  - 代表ペア:
    - FR09 系: `profile65 ↔ profile64`, `profile95 ↔ profile96`, `profile106 ↔ profile112`, `profile57 ↔ profile90`
    - FR13 系: `profile43 ↔ profile42`, `profile101 ↔ profile110`, `profile77 ↔ profile78`, `profile37 ↔ profile41`
  - 生成形式:
    - `BracketType=1501`
    - `Surfaces1=[side_or_support_profile+",FL"]`
    - `Surfaces2=[deck_fr_profile+",FL"]`

この構造にすると、**Wall-Deck DL接続** と **Side-Deck FR接続** を分離でき、Fore/Aft整合チェックも実装しやすくなります。

### パラメータ依存関係

#### 1. BracketType の使い分け
参考スクリプトから読み取れる基本ルールは以下です。

- **1501**
  - プロファイル同士の接続
  - 例:
    - Wall.DL ↔ Deck.DL
    - Side.FR ↔ Deck.FR
  - `Surfaces1/Surfaces2` は `profile + ",FL"` や `",WF"`, `",WB"` が中心

- **1505**
  - 板面（PLS）とプロファイルをつなぐサイドブラケット
  - 例:
    - Side.FR ↔ Deck 板
  - `BracketParams=["200"]` を持つ
  - `Sf1EndElements` を併用するのが重要

#### 2. Thickness
参考スクリプトでは概ね以下の対応です。

- Wall/Deck の一般 `1002` 系 DL材同士
  - `Thickness ≒ 8`
- `1201` 系や比較的大型部材
  - `Thickness = 12`
- Side Wall `1002` 系と Deck 板の 1505
  - `Thickness ≒ 10`
- FR の `1003` 系と Deck FR 系
  - `Thickness ≒ 8` または `9`

したがって仕様上の推奨は:

- `BracketType=1501` かつ `1002` 同士: **8**
- `1201` を含む: **12**
- `BracketType=1505`: **10**
- `1003` を含む FR 接続: **8～9**

#### 3. Sf1/Sf2 DimensionType
参考スクリプト準拠なら次が基本です。

- 一般的な profile-FL 接続:
  - `Sf1DimensionType=1531`
  - `Sf1DimensonParams=["200","15"]`
  - `Sf2DimensionType=1531`
  - `Sf2DimensonParams=["200","15"]`

- FR や大きめ profile:
  - `["250","15"]` または `["300","15"]`

- 板面（PLS）を基準にする 1505:
  - `Sf1DimensionType=1541`
  - `Sf1DimensonParams=["0","100"]`
  - `Sf1EndElements` 必須
  - `Sf2DimensionType=1531`
  - `Sf2DimensonParams=["200","15"]`

- `1201` 系のDL02接続のような大型ブラケット:
  - `["800","15"]`

#### 4. FlangeType / FlangeParams
参考スクリプトで flange が使われているのは主に `1201` や Deck FR の一部です。

- 代表値:
  - `FlangeType=262`
  - `FlangeParams=["100","30","29.999999999999996","30","30","1"]`
  - または
  - `FlangeParams=["75","30","29.999999999999996","30","50","1"]`

今回の 30個以上の追加では、**まずは flange なしの 1501 / 1505 を優先**し、  
DL02 のような既存 1201 同士の接続だけ flange 付き仕様を流用するのが安全です。

#### 5. End1/End2 の指定要否
- ブラケット自体には `End1/End2` は不要
- ただし `Sf1EndElements` が必要になるケースがある
  - 特に `BracketType=1505` で `PLS` 面を基準にする場合
  - 参考スクリプトではこれが未指定だと位置決め不安定になる傾向が読み取れる

### 注意事項

- **Fore/Aft 整合ルールは必須**
  - `Wall.Fore.DLxx.*` → `Deck.*.DLxx.F / FP` のみ
  - `Wall.Aft.DLxx.*` → `Deck.*.DLxx.A / AP` のみ
  - これを破るペアは候補から除外する必要があります。

- **DL00 も方向解釈が必要**
  - 元スクリプトでは
    - Fore 側: `Deck.*.DL00.F`
    - Aft 側: `Deck.D.DL00.A`
  - よって DL00 も通常の Fore/Aft ルールで扱うのが安全です。

- **Side Wall は Fore/Aft 制約なし**
  - ただし接続先 Deck profile または Deck solid が実在していることが必要です。

- **Surfaces1 / Surfaces2 の型選択に注意**
  - 単純 profile 接続: `profileX[0]+",FL"`
  - T材/FB/板面参照: `["PLS", ... , element]`
  - FR profile の web 側参照が必要なら `",WF"` や `",WB"` を使う

- **Sf1EndElements を参考スクリプト通り継承すること**
  - `1505` 系では特に重要
  - 例: `["PLS","False","False","-0","-1","-0",profile17[0]]`

- **BaseElement は接続の主対象に合わせる**
  - Wall↔Deck DL なら通常 Wall 側プロファイル
  - Deck FR 支持ブラケットなら Deck 側 FR profile を Base にする構成も参考スクリプトに存在

- **DL02 の 1201 系は例外扱い**
  - `profile18 / profile19 / profile59 / profile120 / profile88 / profile97` などは flange付き・複数ボディのため、
    profile index `[0]` と `[1]` の使い分けを誤ると面参照がずれます。

- **30個以上の配置は十分可能**
  - Wall Fore/Aft の DL × Deck A/B/C/D の適合法だけで多数確保できる
  - さらに Side FR 系を加えることで 40件超も可能

- **参考スクリプトの実績パターンを優先**
  - `Wall.DL ↔ Deck.DL` は 1501
  - `Side.FR ↔ Deck板` は 1505 + `Sf1EndElements`
  - `Side.FR ↔ Deck.FR` は 1501

以上より、元スクリプトに対して「30個以上ブラケットをつける」編集を行う場合、  
**Wall Fore/Aft のDL系ブラケット群 + Side FR 系ブラケット群** を組み合わせる方針が最も整合的です。
