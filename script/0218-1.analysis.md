## ブラケット配置分析

### 対象部材とループ変数
本【元のスクリプト】は、ケーシング周りの **Deck（A/B/C/D）シート**、**Wall（Fore/Aft/SideP/SideS）シート**、それらに取り付く **Profiles（FB/Angles/形鋼相当：ProfileType 1002/1003/1201/1007）**、および一部シート厚み付けした **Solids（Deck・Wallの実体化）** を生成しています。

参考スクリプトのブラケットは「**Profile端部（End要素）や交差部**」ではなく、**Surfaces1/Surfaces2 の“面ペア”**でブラケットを定義しており、主に以下の3系統の接続に分類できます（参考スクリプトから抽出）:

- **(A) Profile（壁側Casing Wallの縦材/横材）× Profile（Deck側の材）**  
  - 例: `profile7[0]+",FL"` と `profile6[0]+",FL"` のように **FL面同士**  
  - BracketType=1501 が多い
- **(B) PLS（板面）× Profile（形材FL面）**（Side側で顕著）  
  - 例: `Surfaces1=["PLS", ... , solid1]` と `Surfaces2=[profile13[0]+",FL"]`  
  - BracketType=1505（BracketParams=["200"]）が多い
- **(C) ProfileのWF/WB（Web/面）× ProfileのFL**（大きい形材同士）  
  - 例: `Surfaces1=[profile4[0]+",WF"]` と `Surfaces2=[profile14[0]+",FL"]`  
  - BracketType=1501 + FlangeType=262 を伴うケースあり

したがって、【元のスクリプト】においてブラケット対象となるのは、概ね次の「接続ペア」を作るための Profile 群です（ループ化候補の塊）:

- **Wall.Fore / Wall.Aft の DLxx 系（ProfileType=1002/1003/1201）**  
  - 例: `profile62, profile94, profile100, profile117, profile59/97/88, profile69` 等  
- **Wall.Side（SideP/SideS） の FRxx 系（ProfileType=1002/1003）**  
  - 例: `profile12,16,44,52,111,113,24,34,45,55,58,76,82,91,102,103,116,118,122` 等  
- **Deck側で“壁材を端部要素に持つ”梁/縦材（ProfileType=1002/1003/1201/1007）**  
  - 例: `profile8,15,27,28,54,74`（Deck.C系）  
  - `profile17,18,22,23,33,35,60,61,64`（Deck.D系）  
  - `profile30,31,70,71,85,112,114`（Deck.B系）  
  - `profile40,41,49,50,56,90,108,109`（Deck.A系）

### ループ構造方針
参考スクリプトは明示ループではなく逐次生成ですが、仕様としては以下の2ループに整理可能です（漏れ防止）:

1. **Deck板（solid）× Side縦材（profile）の 1505 ブラケット群**
   - 対象例: `profile16, profile12, profile44, profile111, profile52, profile113, profile3, profile5, profile51, profile79, profile89, profile91, profile92, profile102, profile103, profile122 ...`
   - 方針:  
     - `for p in side_profiles:`  
       - `Surfaces1 = ["PLS", ..., solid2(or solid1/solid3/solid4), normal]`  
       - `Surfaces2 = [p[0]+",FL"]`  
       - `BracketType=1505`, `BracketParams=["200"]`, `Sf1DimensionType=1541`, `Sf1DimensonParams=["0","100"]`, `Sf1EndElements` を Deck側の基準要素（参考では profile1[0] 等）に合わせて指定

2. **Wall（Fore/Aft）DLxx材 × Deck側DLxx材（主にFL×FL）の 1501 ブラケット群**
   - 対象例:  
     - DL01: `profile62` × `profile61`、`profile75` × `profile23`  
     - DL03: `profile36` × `profile35`、`profile117` × `profile35`  
     - DL04: `profile87` × `profile86`、`profile100` × `profile86`  
     - DL05: `profile72` × `profile17`、`profile32` × `profile17`
   - 方針:  
     - `for (wall_prof, deck_prof) in pairs:`  
       - `Surfaces1=[wall_prof[0]+",FL"]`  
       - `Surfaces2=[deck_prof[0]+",FL"]`  
       - `BracketType=1501`, `Sf1DimensionType=1531`, `Sf1DimensonParams=["200","15"]`（または参考に倣いサイズ別に 200/250/300/800 へ可変）

### パラメータ依存関係
参考スクリプトから読み取れる依存関係（本件の仕様として踏襲すべき決定規則）:

- **BracketType**
  - `1501`: 形材同士（FL×FL、WF/WB×FL）や“PLSを介した板扱い”での一般ブラケット
  - `1505`: **Deck板面（PLS）× Side形材（FL）**で多用。`BracketParams=["200"]` がセットされる

- **Thickness**
  - 参考では多くが **接続している形材板厚に追従**している  
    - 例: Sideの 1002(9mm系) → Thickness≈10  
    - 1002(7mm系) → Thickness≈8  
    - 1201(12mm) → Thickness=12
  - 従って仕様としては、基準を「BaseElement（多くは壁側Profile）の板厚」へ合わせる（7→8、9→10、12→12のように端数丸めが入っている）

- **寸法（Sf1DimensionType/Sf2DimensionType）**
  - FL×FL の 1501: `1531` + `["200","15"]` が標準（大きいものは 250/300/800 等）
  - PLS×FL の 1505:  
    - Sf1（PLS側）: `1541` + `["0","100"]`（板側基準からのオフセット/長さ指定の形）  
    - さらに `Sf1EndElements` が必須級（参考では PLSの終端を決めるために指定）

- **FlangeType/FlangeParams**
  - 参考では **大形材（ProfileType 1003/1201）で WF×FL の場合に付く**ことがある（例: `FlangeType=262`）
  - よって本スクリプトでも、`profile18`（1201）×`profile64`（1003）など “WF/WB を使う”接続には Flange を付与するのが整合的

- **End1/End2指定の要否**
  - ブラケット自体は End1/End2 を持たず、**Surfaces1/Surfaces2 と Sf1EndElements/Sf2EndElements** で“どこまで”を決めている
  - 特に `1505` では **Sf1EndElements を必ず与える**（参考: bracketPram4/7/15/17/22）

### 注意事項
- **Surfaces1/Surfaces2 は「面ペア」定義が必須**  
  - `profileX[0]+",FL"` のような **面種（FL/WF/WB）込みの指定**を必ず行う
  - 板側は `PLS` 指定（`["PLS", ... , solid]`）を使うのが参考の流儀

- **Mold（+/-）の整合**
  - 参考では Wall.Fore と Wall.Aft、また 1201（Mold="-") の場合に Mold が反転するケースがある  
  - ブラケットも `Mold="+"` / `Mold="-"` を接続部材の向きに合わせる（特に Fore 側は ReverseDir=True のProfileが多く、Mold整合が重要）

- **Sf1EndElements/Sf2EndElements の未設定は失敗要因**
  - 1505（PLS×FL）では、板側のブラケット範囲が曖昧になりやすく、参考スクリプトは必ず EndElements を与えている  
  - 仕様として「1505は必ずSf1EndElementsを指定」とする

- **BodyDivideByCurves は後処理であり、ブラケット生成仕様とは独立**
  - 参考スクリプトではブラケット作成後に `BodyDivideByCurves` で切断している例が多数  
  - 本件「ブラケットをつける」要求の範囲では、まずブラケット生成を完了させ、その後必要なら同様の切断工程を追加する、という工程分離が安全
