## ブラケット配置分析

### 対象部材とループ変数
- **Deck（板/ソリッド化対象）**
  - `extrude_sheet1` : HK.Casing.Deck.B（シート）
  - `extrude_sheet2` : HK.Casing.Deck.A（シート）
  - `extrude_sheet4` : HK.Casing.Deck.C（シート）→ `solid1`（HK.Casing.Deck.C：厚み付け+押し出し/くり抜き後のソリッド）
  - `extrude_sheet5` : HK.Casing.Deck.D（シート）→ `solid2`（HK.Casing.Deck.D：ソリッド）
  - `solid3` : HK.Casing.Deck.B（ソリッド）
  - `solid4` : HK.Casing.Deck.A（ソリッド）

- **Wall（板）**
  - `extrude_sheet3` : HK.Casing.Wall.Aft（シート）
  - `extrude_sheet6` : HK.Casing.Wall.SideP（シート）
  - `extrude_sheet8` : HK.Casing.Wall.Fore（シート）
  - `extrude_sheet9` : HK.Casing.Wall.SideS（シート）

- **Profiles（防撓材/桁/アングル/FB等：ブラケットの基準要素・相手面）**
  - Side系（主に`extrude_sheet6`へAttach）  
    `profile3,5,10,12,16,20,21,24,34,44,45,51,52,55,58,76,79,82,89,91,92,102,103,111,113,116,118,122` など
  - Aft/Fore系（`extrude_sheet3`/`extrude_sheet8`へAttach）

- **左右対称配置（ミラー）**
  - `part.MirrorCopy(..., "PL,Y", "")` が多数使用されており、**P/Sで同仕様のブラケットを片舷作成→Y平面でミラー**が基本方針。

- **ループ変数の扱い**
  - 参考スクリプトは明示的なforループは少ないが、実質的に
    - 「FR番号（`var_elm*`）× 取り付け対象Deck（A/B/C/D）」
    - 「DL番号（`Casing.DL**`）× 取り付け対象Deck（A/B/C/D）」
    の組合せを逐次生成しており、生成規模的には **テーブル駆動＋反復（ループ）**が適する。

### ループ構造方針
- **基本方針：片舷（P側）だけを生成し、必要なものは `MirrorCopy(..., "PL,Y","")` で反対舷を作る。**
- 実装上の反復単位（推奨）  
  1) **Side系（FR）× Deck（A/B/C/D）**  
     - 対象Profile：`FR07/08/09/10/11/12/13/14/15` に相当する `profile**`  
     - 対象Deckソリッド：`solid1(C) / solid2(D) / solid3(B) / solid4(A)`
     - ブラケット：多くが **Type=1505（板-形鋼：PLS×FL）**
  2) **Aft/Fore系（DL）× Deck（A/B/C/D）**  
     - 対象Profile：`Casing.DL00〜DL05` の `profile**`
     - ブラケット：**Type=1501（形鋼FL×形鋼FL）** が中心、一部Type=1505もあり。
- 反復生成に必要な入力は、参考スクリプトからみると実質「部材参照（solid/profile）」の組と、寸法タイプ（1531/1541）とそのパラメータ（例：`["200","15"]`）なので、**ブラケット定義テーブル（辞書/リスト）を回す**のが最も安全。

### パラメータ依存関係
- **BracketType**
  - **1505**：Deck等の板ソリッド面（PLS）に対して、Profileの FL 面へ取り付ける典型（Side FR→Deck）。
  - **1501**：Profile同士（多くは `FL`×`FL`）で取り合う典型（Fore/Aft DL→Deck の一部、Side FR13など）。

- **Thickness（板厚）**
  - Side FR→Deck（Type1505）：概ね **t=10**（`"9.9999999999999982"`）。
  - DL系/一部（Type1501）：概ね **t=8**（`"7.9999999999999964"`）。
  - 例外として DL02周りなどで **t=12** のケースあり（ブラケットがフランジ付・大寸法）。

- **BracketParams（主に1505で出現）**
  - `BracketParams=["200"]` が多く、ブラケット寸法の代表値（高さ/脚長相当）として固定入力。

- **Mold（+/-）**
  - 1505は多くが `"+"`（板の表側基準）。
  - Fore/Aft DL04→Deck の 1501などで `Mold="-"` が使われる（向きが反転しやすい箇所）。

- **Scallop**
  - `Scallop1Type=1801` が基本。
  - `Scallop1Params=["0"]` が多いが、WF/WB参照を伴う箇所では `["50"]` などが出る。

- **Surfaces 指定（面ペア）**
  - **PLS面（板）**：`["PLS","False","False", nx, ny, nz, solidX]` 形式。Deckは概ね法線が `0,0,-1` 系。
  - **形鋼面**：`[profileX[0]+",FL"]` が基本。例外で `WF/WB/FR` を参照するケースもある（例：`profile18[0]+",WF"`）。

- **DimensionType**
  - **1531**：距離指定の標準。多くが `["200","15"]` または `["250","15"]`、大物で `["800","15"]` など。
  - **1541**：板側の位置決め（端基準）で使用されることが多い。`Sf1EndElements` とセットで出現し、`["0","100"]` や `["0","80"]` のようなオフセット指定。

- **EndElements（端基準）**
  - 1505で板側を確実に切り詰め・位置決めするために頻出。
  - 例：Deck.C では `Sf1EndElements=... profile8[0]`（Deck.C.DL05P）  
    Deck.A では `Sf1EndElements=... profile47[0]`（Deck.A.DL05P）  
    Deck.B では `Sf1EndElements=... profile68[0]`（Deck.B.DL05P）  
    Deck.D では `Sf1EndElements=... profile17[0]`（Deck.D.DL05P）

### 注意事項
- **元のスクリプトはブラケットが一切生成されていない**ため、追加時は必ず `CreateBracketParam()` → 必須フィールド（DefinitionType/Name/Material/BaseElement/Thickness/BracketType/Surfaces1/Surfaces2/DimensionType 等）→ `CreateBracket()` の順で行う必要がある。
- **Surfaces1/Surfaces2 は「面ペア」定義であり、単に要素を渡すだけでは成立しない**（特にPLS指定はベクトルと対象ソリッドが必要）。
- **1505（PLS×FL）では Sf1EndElements を省略すると形状生成が不安定になりやすい**（参考では多用）。Deck毎に「端基準プロファイル」を用意し、それを `Sf1EndElements` に渡す設計が重要。
- **左右対称（ミラー）前提の配置は、BaseElementが片舷側のものを指していることに依存**する。ミラー後の要素参照で追加加工する場合は、ミラー戻り値を必ず変数に保持する。
- **Type1501でも、形鋼の向き（Mold/Reverse）によっては Surfaces の向きが逆転し形状が裏返る**ため、参考スクリプトにある Mold 設定を踏襲するのが安全。
- **WF/WB/FR面を参照するブラケット（例：profile18のWF/WB絡み）は例外処理扱い**とし、一般の FR→Deck の1505テンプレートとは分けて管理するのが望ましい。
