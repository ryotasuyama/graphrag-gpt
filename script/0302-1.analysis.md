## ブラケット配置分析

### 対象部材とループ変数
元スクリプトには **ブラケット作成（CreateBracket / CreateBracketParam）が一切登場しない**ため、現状は「ブラケット0個」のモデル生成スクリプトです。  
一方、参考スクリプトでは多数の `CreateBracketParam()` が使われ、**同一の部材カテゴリ（Deck/Wall/Side の各 Profile）に対し、同型式ブラケットをフレーム位置（FRxx）やデッキライン（DLxx）ごとに反復配置**して 50個以上を実現しています。

本件（「50個以上ブラケットをつけてください」）に対して、元スクリプトの構造から、ブラケット対象になり得る部材群（= Surfaces1/2 と BaseElement 候補）は以下です。

- **Deck 系（押し出しSheet）**
  - `extrude_sheet1` : HK.Casing.Deck.B（Sheet）
  - `extrude_sheet2` : HK.Casing.Deck.A（Sheet）
  - `extrude_sheet4` : HK.Casing.Deck.C（Sheet）
  - `extrude_sheet5` : HK.Casing.Deck.D（Sheet）
  - `extrude_sheet7` : HK.General.Deck.UpperDeck（Sheet）

- **Wall 系（押し出しSheet）**
  - `extrude_sheet3` : HK.Casing.Wall.Aft（Sheet）
  - `extrude_sheet8` : HK.Casing.Wall.Fore（Sheet）
  - `extrude_sheet6` : HK.Casing.Wall.SideP（Sheet）
  - `extrude_sheet9` : HK.Casing.Wall.SideS（Sheet）

- **Profile（ブラケットの主な BaseElement / 取付相手）**
  - **DL系（Y基準）**：`profile4,6,7,8,9,17,18,19,23,25,26,28,33,35,36,40,46,47,48,50,53,54,59,61,62,63,66,67,68,69,71,72,73,74,75,80,83,84,85,86,87,93,94,99,100,104,105,107,108,109,114,115,117,119,121,123...`

  - **MirrorCopy生成物**も多数あり、左右舷対称に同数のブラケットを追加可能

#### ループ変数（実装方針上の整理）
元スクリプトにループはありませんが、50個以上を確実に満たすには、参考スクリプト同様に「対象Profileの配列」を作り、以下のような **繰り返し対象セット**を構成するのが合理的です。

- `side_cdp_profiles = [profile16[0], profile12[0], profile44[0], profile52[0], profile113[0], profile111[0], profile3[0], profile89[0], ...]`
- `fore_dl_profiles = [profile26[0], profile32[0], profile53[0], profile100[0], profile62[0], profile94[0], ...]`
- `aft_dl_profiles  = [profile75[0], profile72[0], profile63[0], profile87[0], profile121[0], ...]`
- `deck_dl_profiles = [profile17[0], profile18[0], profile23[0], profile33[0], profile35[0], profile86[0], ...]`

※実際の配置数は、左右ミラー分も含めると容易に 50 を超えます。

#### 50個以上にするための「配置数の作り方」（設計）
- 上表 #3/#4/#16/#17 のような **1505（Deck面PLS ⇔ Side FR材FL）**を、Side FR材（FR07/08/10/11/12/14/15 + ミラー側）に一括で付けるだけで 10〜20個は確保可能。
- DL01〜DL05、DL00 それぞれで **1501（FL⇔FL）**を Fore/Aft/Deck の組合せで作ると、さらに 20〜40 個増やせる。
- 参考スクリプトが実施しているように、必要なら **BodyDivideByCurves**（分割）をブラケット作成後に適用し、干渉部の分割・整理も可能。

### ループ構造方針
元スクリプトは逐次生成で、部材が非常に多いため、ブラケット追加仕様としては次のループ方針が最適です（参考スクリプトの「同型ブラケットを複数FRに配置」の思想を踏襲）。

- **方針A（Side FR系：BracketType=1505 を量産）**
  - ループ対象: `side_fr_profiles = [profile16[0], profile12[0], profile44[0], profile111[0], profile52[0], profile113[0], profile3[0], profile89[0], ...]` と、それらの MirrorCopy 側
  - 各要素について:
    - `BaseElement = 対象profile`
    - `Surfaces2 = [対象profile + ",FL"]`
    - `Surfaces1 = ["PLS", ..., solid2]`（Deck.Dソリッドの下面/上面相当をPLSで与える：参考 bracketPram4 等）
    - `Sf1EndElements = ["PLS", ..., profile17[0]]`（Deck側DL05P等を端要素扱いにして安定させる：参考 bracketPram4/7/15/17 等）
  - これを左右で繰り返し、まず **最低30個程度**を確保しやすい

- **方針B（DL系：BracketType=1501 を量産）**
  - ループ対象: `fore_pairs = [(profile62[0], profile61[0]), (profile100[0], profile86[0]), (profile117[0], profile35[0]), ...]`  
    `aft_pairs  = [(profile75[0], profile23[0]), (profile87[0], profile86[0]), (profile36[0], profile35[0]), ...]`
  - 各ペアについて:
    - `BaseElement = wall側/縦材側（基本はFore/AftのWall側Profile）`
    - `Surfaces1 = [BaseElement + ",FL"]`
    - `Surfaces2 = [相手Profile + ",FL"]`
  - これで **20個以上**を追加可能（DL00〜DL05、Fore/Aft、Deck接続などを組み合わせ）

> 実運用上の安定性は、参考スクリプトのように「同一BracketType・同一DimensionType」を守って量産する方が高い（パラメータのばらつきが減る）。

### パラメータ依存関係
参考スクリプトから読み取れる、ブラケットパラメータの「決め方」は以下の通りです（元スクリプトへ適用する際の仕様）。

#### BracketTypeの使い分け
- **1501**: 基本ブラケット（FL面同士、あるいはWF/WB⇔FLなど）
  - `Surfaces1=[profileX+",FL"]` / `Surfaces2=[profileY+",FL"]` が典型
  - `Sf1DimensionType=1531`, `Sf2DimensionType=1531` が多い
- **1505**: 片側が **PLS（平面/ソリッド参照）**で、片側がProfile FLのタイプ（Side FR材⇔Deck面など）
  - `Surfaces1=["PLS",... , solidX]`
  - `Sf1DimensionType=1541` が多く、`Sf1EndElements` を併用する例が多い
  - `BracketParams=["200"]` のような追加パラメータあり

#### Thickness（板厚）
参考では部材系統によりほぼ固定値で運用されているため、元スクリプトでも以下のように寄せるのが妥当です。
- Side FR系（ProfileParamsが["150","90","9",...]) → ブラケット板厚 `9.999...` が頻出（1505）
- 一般DL系（ProfileParamsが["125","75","7",...]) → ブラケット板厚 `7.999...` が頻出（1501）
- フランジ付き/厚板（ProfileType=1201 で t=12 を含む） → ブラケット板厚 `12`（1501 + FlangeType=262）

#### 寸法定義（Sf1/Sf2 Dimension）
- `Sf?DimensionType=1531` + `Sf?DimensonParams=["200","15"]` が基本形（1501）
- `Sf1DimensionType=1541` + `Sf1DimensonParams=["0","100"]` が PLS面側で多用（1505）
- FR縦材（WF）に当てる場合は `Sf1EndElements=[profile?+",FR"]` を付けて端部基準を固定（参考 bracketPram3/12）

#### End1/End2（ブラケット自体のEnd要素）
- 参考スクリプトのブラケットでは End1/End2 を直接指定していない例が多く、代わりに
  - BaseElement の End処理（Profile側 End1/End2）
  - ブラケットの `Sf1EndElements` / `Sf2EndElements`
  で整合を取っている。

### 注意事項
- **元スクリプトにはブラケット生成が無い**ため、編集指示（50個以上）を満たすには `CreateBracketParam()` と `CreateBracket()` を新規追加する必要がある（=「元コードの一部編集」ではなく「機能追加」になる）。
- 参考スクリプトでは、ブラケット作成後に `BodyDivideByCurves()` を多数実施しており、これは
  - ブラケット（および関連Profile/solid）を基準線で分割し、形状を整える
  - 干渉回避・製作分割を意図している
  可能性が高い。元スクリプト側でも、ブラケットを追加しただけで干渉が出る場合は、同様に分割工程が必要。
- **Surfaces の指定形式が2系統ある**
  1) `profileX[0]+",FL"` のような **部材面指定**
  2) `["PLS","False","False", nx,ny,nz, element]` のような **平面（法線ベクトル）+ 参照要素指定**
  
  1505 を量産する場合は後者が中心になるため、法線（例: `"0","-0","-1"`）の整合が重要。
- **Mold（"+" / "-"）の扱い**
  - 参考では接続方向や部材の反転により Mold が変わる（例: DL02厚板のブラケットで Mold="-" が出る）
  - 元スクリプトで Fore/Aft をまたぐ場合、同一ロジックで Mold を決めないと板が裏返る恐れがある。
- `Sf1EndElements` / `Sf2EndElements` を使うパターン（特に 1505 と 1541）は、**端部基準が無いと寸法決定が不安定**になりがちなので、参考に合わせて付与するのが安全。
