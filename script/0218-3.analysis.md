## ブラケット配置分析

### 対象部材とループ変数
- **Deck（板）**
  - `extrude_sheet1`: HK.Casing.Deck.B（Sheet）
  - `extrude_sheet2`: HK.Casing.Deck.A（Sheet）
  - `extrude_sheet4`: HK.Casing.Deck.C（Sheet）
  - `extrude_sheet5`: HK.Casing.Deck.D（Sheet）

  - `solid1`: HK.Casing.Deck.C（厚み付け対象：`extrude_sheet4`）
  - `solid3`: HK.Casing.Deck.B（厚み付け対象：`extrude_sheet1`）
  - `solid4`: HK.Casing.Deck.A（厚み付け対象：`extrude_sheet2`）
  - `solid5`: HK.Casing.Wall.Aft.CD（厚み付け対象：`extrude_sheet3`）
  - `solid10`: HK.Casing.Wall.Fore.CD（厚み付け対象：`extrude_sheet8`）
  - `solid12`: HK.Casing.Wall.Side.CDP（厚み付け対象：`extrude_sheet6`）
  - （他にも OA/AB/BC 系の `solid6/7/9/11/13/14/15/16` が存在するが、参考スクリプトのブラケットは主に **Deck(D) のソリッド（solid1相当）** を“PLS面”で参照している）
- **Profile（スティフナ/形鋼：ブラケットの BaseElement になり得る）**
  - Fore/Aft の縦通材（主に `ProfileType=1002` の L形相当）  
    - 例：`profile62`(Fore DL01 CDP), `profile26`(Fore DL01 BCP), `profile94`(Fore DL00 CD), `profile75`(Aft DL01 CDP), `profile72`(Aft DL05 CDP), `profile87`(Aft DL04 CDP), `profile36`(Aft DL03 CDP) など
  - Side のフレーム位置材（`FR07/08/10/11/12/14/15` 等）
    - 例：`profile16`(FR07 CDP), `profile12`(FR08 CDP), `profile44`(FR10 CDP), `profile111`(FR11 CDP), `profile52`(FR12 CDP), `profile113`(FR14 CDP), `profile3`(FR15 CDP) 等
  - 大型 Profile（`ProfileType=1003`）  
    - 例：`profile42`(Deck.D FR13P), `profile64`(Deck.D FR09P), `profile22`(Deck.D FR09C), `profile60`(Deck.D FR13C) 等
  - フランジ付き（`ProfileType=1201`）  
    - 例：`profile19`(Aft DL02 CDP), `profile59`(Fore DL02 CDP), `profile88`(Fore DL02 ABP) 等  
    - 参考スクリプトではこの種に対して **Mold="-" + Thickness=12 + FlangeType=262** を併用したブラケットがある（後述）

**補足（Surfaces 指定形式の読み替え）**
- `profileX[0]+",FL"` のように **Profile要素 + 参照面(FL/WF/WB)** をペアで指定するのが基本。
- `PLS` 形式（参考スクリプトで多用）は、**ソリッドやプロファイルを“平面（法線ベクトル）”として扱う特殊指定**で、1505系やフランジ付き1501系で出てくる。  
  - 例：`["PLS","False","False","0","-0","-1",solid1]`（Deckソリッドの下面方向を法線に取る）

### ループ構造方針
- **DL（縦方向）系ブラケット（BracketType=1501、両FL）**
  - 対象：Fore/Aft の `DL01/DL03/DL04/DL05` で、**Wall側CDP** と **Deck.D側P/FP/AP** のペアを作る。
  - ループ案：
    - `(wall_profile, deck_profile)` のリストを作り、順番に `CreateBracketParam()` → `CreateBracket()`。
- **FR（フレーム）系ブラケット（BracketType=1505、Deckソリッド PLS – Side FRxx CDP のFL）**
  - 対象：Side の `FR07/08/10/11/12/14/15` の各CDP。
  - ループ案：
    - `fr_profiles = [profile16, profile12, profile44, profile111, profile52, profile113, profile3]` のように **BaseElement=各profile[0]** でループし、Surfaces1 は共通で `PLS(..., solid2)` を使う。
- **例外（DL02 フランジ付き）**
  - `ProfileType=1201`（`profile59`, `profile19` など）については、参考スクリプト同様に
    - Mold="-"、Thickness="12"
    - Surfaces1/2 を PLS 指定
    - FlangeType=262 を併用
  - これはループ化する場合も、対象要素が限定的（Fore DL02 / Aft DL02）なので2件程度の固定定義が妥当。

### パラメータ依存関係
- **BracketType**
  - 1501：一般的な **2面（Surfaces1/Surfaces2）を FL/WF/WB などで結ぶ**ブラケット
  - 1505：Deckソリッド（PLS）と FR材(FL)を結ぶ **片持ち/定寸型**（`BracketParams=["200"]` が付く）
- **Thickness（板厚）**
  - 参考スクリプトより実質的に2系統：
    - **t ≈ 8（"7.9999999999999964"）**：L形(7mm)系の接続（DL01/DL03/DL04/DL05 の 1002系が多い）
    - **t = 10（"9.9999999999999982"）**：Side FRxx の 1505 ブラケットで多い
    - **t = 12**：`ProfileType=1201`（DL02フランジ付き）接続ブラケット
- **Sf1DimensionType / Sf2DimensionType**
  - 1501（両FL）では `1531` を使用し、`Sf1DimensonParams=["200","15"]`（または FR/Side系で `"250","15"`）のような **脚長/逃げ量** 指定。
  - 1505 では Deck側を `1541` とし、`Sf1DimensonParams=["0","100"]` のように **オフセット距離**を与える。  
- **Sf1EndElements / Sf2EndElements**
  - 参考スクリプトで 1505 のときに `Sf1EndElements` が出現：  
    - Deck側の “端をどの要素で止めるか” を指定し、ブラケットが無限に伸びたり向きが反転するのを防ぐ目的。
  - 元スクリプトに適用する場合も、1505 では **必須**扱いとし、端要素に **Deck側の代表Profile（例：Deck側のDL05材やDeck上の基準材）** を入れる。

### 注意事項
- **Surfaces1 / Surfaces2 は「面ペア」であり、要素単体指定ではない**
  - `profileX[0]+",FL"` のように、必ず “要素 + 面種(FL/WF/WB)” をセットにする。
  - PLS 指定の場合は `["PLS", ..., 法線, 要素]` という配列形式になる（参考スクリプト踏襲）。
- **BaseElement の選び方が重要**
  - 参考スクリプトでは、ブラケット名が「Wall～Deck」でも **BaseElement は Wall側 profile** になっているケースが多い（例：`BaseElement=profile7[0]` など）。  
  - 元スクリプトでも、基本は **壁側（またはFR材側）を BaseElement** とし、Deckソリッドは Surfaces 側に回す。
- **Mold（"+" / "-"）の整合**
  - 1501 の多くは Mold="+"、DL02フランジ付き接続は Mold="-" が採用されている（参考スクリプト）。
  - 元スクリプトでも、DL02（1201）絡みは **Mold="-" + t=12 + FlangeType=262** を優先。
- **後処理（BodyDivideByCurves）**
  - 参考スクリプトは、作成したブラケットを `skt_pl4` などで分割している例が多い。  
  - ただし編集指示は「ブラケットをつけてください」なので、仕様としては **分割工程は必須ではない**が、仕上げ品質を合わせる場合は「ブラケット作成後に同一スケッチ平面で分割」を検討対象にする。
- **元スクリプトの Deck.D ソリッドは `solid2`**
  - 参考スクリプトで 1505 の Surfaces1 に使われる `solid1` は「HK.Casing.Deck.D」ソリッド。元スクリプトでは同等が `solid2` なので、1505 の PLS 参照は `solid2` を使う前提で整理する。
