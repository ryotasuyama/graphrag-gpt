## ブラケット配置分析

### 対象部材とループ変数
- **参照スクリプト側のブラケット母材（BaseElement）候補の傾向**
  - `profileXX[0]`（主に ProfileType=1002/1003/1201 の **スティフナ/ガーダ相当のプロファイル**）を BaseElement にしてブラケットを生成。
  - 取り合いは **「Surfaces1 / Surfaces2 の面ペア」**で定義され、面指定は以下の2系統が混在：
    - **プロファイル面指定**: `profileN[0]+",FL"` / `",WF"` / `",WB"` / `",FR"` など
    - **PLS 指定（平面・板要素側）**: `["PLS","False","False", nx, ny, nz, element]` の形式（法線ベクトル＋参照要素）
- **元のスクリプトに存在するブラケット対象となり得る部材群（抜粋・分類）**
  - **Deck/Wall の板（Sheet/Solid）系**
    - `extrude_sheet1..5,7`（Deck A/B/C/D、UpperDeck）
    - `extrude_sheet3,8`（Wall Aft / Wall Fore）
    - `solid1..4`（Deck A/B/C/D のソリッド化）
    - `solid5..16`（Wall Fore/Aft/Side のソリッド化：AB/BC/CD/OA など）
  - **プロファイル（ブラケットの BaseElement 候補）**
    - Fore/Aft 壁側の縦材相当：`profile6,7,9,25,26,32,35,36,53,62,67,72,75,83,87,94,100,105,115,117,119,121,123...`
    - Deck 側の縦材相当：`profile17,18,23,33,35,47,50,61,68,71,74,85,93,109,114...`
    - Side 壁側のフレーム相当：`profile3,5,10,12,16,20,21,24,34,37,43,44,45,51,52,55,57,58,65,76,77,79,82,89,91,92,95,101,102,103,106,111,113,116,118,122...`
- **ループ変数（実装方針として抽出できる単位）**
  - 参照スクリプトではループで生成していないが、構造上は以下のように「同型パターン」をループ化可能：
    - **Side 壁フレーム群**（FR07/08/10/11/12/14/15 など）  
      → BracketType=1505 のパターンが多い（Deck ソリッド/板とフレーム FL の取り合い）。
    - **Fore/Aft 壁の DL01〜DL05 周り**  
      → BracketType=1501 が多い（プロファイル FL 同士、あるいは PLS で壁/デッキ面を与える）。
    - **Deck の FR09/FR13 などの大断面プロファイル**  
      → BracketType=1501 + FlangeType=262 + Sf1EndElements を使うパターンが存在。

### ループ構造方針
- **ループ化の核（20個以上を確実に満たすための最短ルート）**
  1. **Side 壁フレーム群（FR系）を配列化**して、Deck ソリッド（例：`solid2=Deck.D`）に対して **BracketType=1505** を一括生成する。
     - 例：`frames_cdp = [profile16[0], profile12[0], profile44[0], profile111[0], profile52[0], profile113[0], profile3[0]]`（7個以上）
     - 各要素に対して `Surfaces1 = PLS(solid2)`, `Surfaces2 = frame+",FL"`, `Sf1EndElements = PLS(profile17[0])` を共通適用
  2. **Fore/Aft の DL縦材（DL01〜DL05 + DL00/03/04等）をペア化**して、**BracketType=1501（FL-FL）**を一括生成する。
     - 例：`pairs = [(profile32[0], profile17[0]), (profile72[0], profile17[0]), (profile100[0], profile86[0]), ...]`
- **配置漏れ回避**
  - 参照スクリプトは「都度手書き」だが、元スクリプトは部材数が多く対称ミラーも大量に存在するため、**片舷で生成→MirrorCopy で複製**という方針も有効。
  - ただし参照スクリプトではブラケット自体を MirrorCopy していないため、仕様としては「P/S それぞれに生成」でも整合する。

### パラメータ依存関係
- **Thickness（板厚）**
  - 参照スクリプトの実績：
    - BracketType=1501：`Thickness="7.9999..."` や `"12"`、プロファイル種別（1002/1201等）に連動している様子
    - BracketType=1505：`Thickness="9.9999..."`（Sideフレーム系で多用）
  - 元スクリプトへの適用方針：
    - Sideフレーム（FRxx、ProfileType=1002の典型）：**t=10**
    - DL縦材（ProfileType=1002の典型）：**t=8**
    - 1201（フランジ付き）周り：**t=12**
- **BracketType**
  - 参照スクリプトで確認できる主要タイプ：
    - **1501**：基本のブラケット（FL-FL、WF-FL、WB-FL など多用途）
    - **1505**：Deckソリッド（PLS）とフレーム（FL）接続で多用。`BracketParams=["200"]` を併用。
- **Sf1/Sf2 寸法（Sf1DimensionType / Sf2DimensionType）**
  - 参照スクリプトでは主に：
    - `Sf1DimensionType=1531`（例：`["200","15"]`、`["250","15"]`、`["800","15"]`）
    - `Sf1DimensionType=1541`（例：`["0","100"]` や `["0","80"]`）＋ `Sf1EndElements` 指定がセット
  - 元スクリプトへの適用方針：
    - 1505（PLS→FL）では **Sf1=1541 + EndElements指定**、Sf2=1531（長さ＋オフセット）を踏襲
    - 1501（FL-FL）では Sf1=1531、Sf2=1531 を踏襲
- **FlangeType / FlangeParams**
  - 参照スクリプトの一部（1501）で `FlangeType=262` が付く（大型フレーム接続：FR09/FR13系に多い）
  - 元スクリプトでは以下に適用候補：
    - `profile64/profile42` など Deck の大断面（ProfileType=1003）と取り合うブラケット（#22相当）

### 注意事項
- **Surfaces1 / Surfaces2 は「面ペア」定義であり、要素名だけでなく面種別（FL/WF/WB/FR）まで一致が必要**
  - 参照スクリプトでは、同じ BaseElement でも接続対象により `WF`/`WB` を使い分けている（例：Deck FR13P の `WF`、同じ名前で `WB` 側を使うケース）。
- **PLS 指定の形式を厳密に踏襲すること**
  - `["PLS","False","False", nx, ny, nz, element]` の並びを崩すと参照面が解釈されない可能性がある。
- **Sf1EndElements / Sf2EndElements の有無で形状が破綻しやすい**
  - 参照スクリプトでは 1505 の多くが `Sf1EndElements` を持つ（Deck側の終端規制）。
  - 1501 でも `Sf1EndElements=[profile4[1]+",FR"]` のように「端部要素」を与える例がある（大型フレーム周り）。
- **ブラケット生成後の後処理（BodyDivideByCurves）**
  - 参照スクリプトは多数のブラケットに対して `BodyDivideByCurves(..., [skt_pl4], ...)` を実行して分割している。  
    元スクリプトでも同等の「仕上げ分割」を要求される場合、ブラケット生成直後に同処理を入れる設計になる。  
- **本タスクの制約**
  - 依頼は「20個以上ブラケットをつけてください」だが、元スクリプト自体にはブラケットAPI呼び出しが無いため、ここで示したのは**配置仕様（どの要素のどの面を接続し、BracketType/寸法系をどう踏襲するか）**である。実装時はこのリストを `CreateBracketParam()` 群として起こす。
