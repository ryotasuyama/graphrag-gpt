## ブラケット配置分析

### 対象部材とループ変数
本スクリプト（元のスクリプト）内で「ブラケット（CreateBracket）」は**一切作成されていません**。そのため、編集指示「できるだけ多くのブラケットをつけてください」を満たすには、参考スクリプトに倣って **BracketParam を追加し、主要な交差部（Deck×Wall、Deck×Side、Wall×補強材）へ網羅的に配置**する方針が必要です。

ただし、現状の元スクリプトは「部材（Sheet/Solid/Profile）」の生成が非常に多く、命名規則も `ABP/BCP/OAP/CDP` 等で多数存在するため、ブラケット対象は次のように整理して扱うのが現実的です。

- **Deck系シート（Plate相当）**
  - `extrude_sheet1` = HK.Casing.Deck.B
  - `extrude_sheet2` = HK.Casing.Deck.A
  - `extrude_sheet4` = HK.Casing.Deck.C
  - `extrude_sheet5` = HK.Casing.Deck.D
  - `extrude_sheet7` = HK.General.Deck.UpperDeck（上甲板）
- **Wall系シート（Plate相当）**
  - `extrude_sheet3` = HK.Casing.Wall.Aft
  - `extrude_sheet8` = HK.Casing.Wall.Fore
  - `extrude_sheet6` = HK.Casing.Wall.SideP
  - `extrude_sheet9` = HK.Casing.Wall.SideS
- **Solid（板厚付け済み、PLS指定に使用されうる“板の実体”）**
  - Deck: `solid1`(Deck.C), `solid2`(Deck.D), `solid3`(Deck.B), `solid4`(Deck.A)
  - Wall/Side: `solid5`(Wall.Aft.CD), `solid10`(Wall.Fore.CD), `solid12`(Wall.Side.CDP), `solid8`(Wall.Side.OAP), `solid15`(Wall.Side.BCP), `solid16`(Wall.Side.ABP) など
- **Profile（補強材：Stiffener/FB/L等）**
  - 例：`profile6`(Fore DL05), `profile7`(Aft DL05), `profile62`(Fore DL01), `profile75`(Aft DL01), `profile19`(Aft DL02), `profile59`(Fore DL02), `profile36`(Aft DL03), `profile117`(Fore DL03), `profile87`(Aft DL04), `profile100`(Fore DL04)
  - Sideフレーム：`profile16`(FR07), `profile12`(FR08), `profile65`(FR09), `profile44`(FR10), `profile111`(FR11), `profile52`(FR12), `profile43`(FR13), `profile113`(FR14), `profile23/profile3/profile89/profile118`(FR15系) 等

**ループ変数化の候補（ブラケットを増やすためのグルーピング）**
- `fore_dl_profiles = [profile62, profile59, profile117, profile100, profile32]` 等（Fore側：DL01/DL02/DL03/DL04/DL05）
- `aft_dl_profiles  = [profile75, profile19, profile36, profile87, profile72]` 等（Aft側：DL01/DL02/DL03/DL04/DL05）
- `side_fr_profiles = [profile16, profile12, profile65, profile44, profile111, profile52, profile43, profile113, profile3]`（Side：FR07〜FR15）
- `deck_dl_profiles = [profile68, profile47, profile8, profile17, profile86, profile35, profile23]`（Deck上のDL系補強）
- `deck_fr_profiles = [profile64, profile42, profile22, profile60, profile78, profile41, profile90, profile96, profile110, profile112]`（Deck上のFR系/横材）

**寸法（DimensionType）・端部拘束（EndElements）の踏襲指針**
- **1501（汎用三角ブラケット）**
  - `Sf1DimensionType=1531` / `Sf1DimensonParams=["200","15"]` が基本（小型）
  - 大きめは `["250","15"]` や `["300","15"]`、DL02など幅広は `["800","15"]`
- **1505（スロット/カット付き等のブラケット、BracketParamsあり）**
  - `BracketParams=["200"]` を使用
  - `Surfaces1` は **PLS指定（solidを参照）**、`Surfaces2` は **profile,FL**
  - `Sf1DimensionType=1541` / `Sf1DimensonParams=["0","100"]` を使用し、`Sf1EndElements` を付ける（参考スクリプトの型）
- **Flange付きブラケット（DL02で使用）**
  - `FlangeType=262` / `FlangeParams=["100","30","30","30","30","1"]` 等（参考に準拠）
  - Surfaces1/2 は PLSベースで方向ベクトルを含む指定パターンがある（後述）

### ループ構造方針
「できるだけ多く」付けるため、参考スクリプトの“個別ベタ書き”ではなく、**Profile群をカテゴリ化して総当たり的に配置**する設計が必要です（ただし誤接続防止のため組み合わせは制限）。

- **方針A：DL（Fore/Aft）× Deck.DL の 1対1接続をループ**
  - `[(fore_wall_dl_profile, deck_dl_profile), ...]` をDL番号で対応付けてループ
  - 各ペアに対し 1501（FL–FL）を作成
- **方針B：Side FR（CDP系）× Deck（solid）をループ**
  - `side_fr_profiles` を回し、1505（PLS–FL）を作成
  - `Surfaces1` は Deck側の solid（例：Deck.D solid2 や Wall.Fore.CD solid10）を代表として選ぶ
  - `Sf1EndElements` に“端基準”となる profile を入れる（参考: `Sf1EndElements=["PLS",..., profile1[0]]` のような形式）
- **方針C：FR系横材（Deck上のFR09/FR13等）× Side FR（FL–FL）を追加**
  - `[(profile64, profile65), (profile42, profile43) ...]` のような代表ペアだけ追加し、過剰な総当たりは避ける

### パラメータ依存関係
参考スクリプトから読み取れる「ブラケットパラメータ決定ロジック」を、元スクリプトへ持ち込む際の仕様です。

- **Thickness（板厚）**
  - 1501の小型：`Thickness ≒ 8.0`（例: 7.9999999999999964）
  - Side FRの1505：`Thickness ≒ 10.0`
  - DL02フランジ材絡み：`Thickness = 12`
  - 目安として「BaseElement の板厚（ProfileParams内の t）」に近い値を採用している傾向  
    - 例：ProfileParamsに `...,"9",...` → Thickness 9〜10
- **BracketType**
  - **1501**：FL面同士（profile–profile）の一般ブラケット（壁縦材×甲板縦材、横材×縦材など）
  - **1505**：PLS面（solidの面）× profile FL の取り合い（Side FR×Deck板）で多用
- **Surfaces1/Surfaces2 の型**
  - **profile参照**：`profileX[0]+",FL"`、必要に応じて `WF/WB` を使用（参考: `profile4[0]+",WF"` / `",WB"`）
  - **PLS参照**：`["PLS","False","False", nx, ny, nz, solid_or_profile]` の形式  
    - 方向ベクトル（nx,ny,nz）は参考では `0,-0,-1` 等が使われ、部材の向きに依存
- **Sf1DimensionType / Sf2DimensionType**
  - **1531**：一般寸法（幅・オフセット）で、`["200","15"]` / `["250","15"]` / `["300","15"]` / `["800","15"]`
  - **1541**：端基準・距離指定（`["0","80"]` や `["0","100"]`）で、`Sf1EndElements` と組で使用（1505で多い）
- **Sf1EndElements / Sf2EndElements**
  - 参照スクリプトでは1501でも局所的に使用（例: WF面側で端基準をFR面に取る）
  - 1505ではほぼ必須級（基準部材を与えないと位置決めが不安定になるため）

### 注意事項
- **元スクリプトにはブラケット作成が存在しない**ため、「追加する仕様」を定義するしかない。実装時は `part.CreateBracketParam()` → 各プロパティ設定 → `part.CreateBracket()` の一連を必ず行う。
- **Surfaces1/Surfaces2 は“面ペア定義”**であり、`FL/WF/WB/FR` の選択を誤ると生成失敗・ねじれ・裏返りが起きやすい。参考スクリプトでは以下のパターンが安定：
  - profile同士：`FL–FL`（1501）
  - solid（PLS）× profile：`PLS–FL`（1505）
  - 梁ウェブ絡み：`WF` or `WB` を使い分け（1501 + flange/EndElements併用が多い）
- **Mold（+/-）は部材の左右・内外で変わる**。参考では Fore/Aftやフランジ材で `Mold="-"` が出る。元スクリプトは左右ミラーが大量にあるため、最終的には**ミラー側でMoldを反転**する仕様が必要になり得る。
- **DL02などフランジ材絡みは特殊（FlangeType=262）**で、PLS指定のベクトルやフランジパラメータが必要。単純なFL–FLの1501を乱発すると干渉しやすい。
- 参考スクリプトはブラケット後に `BodyDivideByCurves` で分割処理を多用しているが、これは「ブラケットを取り付けただけだと形状が大きすぎる/不要部分を切る」運用に相当する。元スクリプトへ同様処理を入れる場合、**分割用スケッチ平面（例: Sketch3 / skt_pl4）を共通利用**する設計が必要。
