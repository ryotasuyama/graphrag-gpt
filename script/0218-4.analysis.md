### ループ構造方針
- **方針A（DL系：Fore/Aft 壁 × Deck 系）**
  - DLごとに「壁側CDP（BaseElement候補）」「甲板側DL*P/FP/AP（相手側）」を **辞書/配列でペア化**して列挙し、同一テンプレ（BracketType=1501, FL×FL, Sf1/Sf2=1531）で一括配置する。
  - ただし **DL02のみ例外**になりやすく、参考スクリプト同様に
    - `Surfaces1/Surfaces2` を `PLS` 形式で与える
    - `FlangeType=262` を付加
    - 寸法（Sf1Dim/Sf2Dim）が `800,15` クラス
    のテンプレを別扱いにする。
- **方針B（FR系：Side 壁FR(CDP) × Deck.D ソリッド）**
  - FR07/08/10/11/12/14/15 等の「同断面の側壁縦通材(CDP)」を配列化し、**BracketType=1505** のテンプレで一括配置する。
  - このとき必ず参考同様に
    - `Surfaces1` は `["PLS", ... , solid(deck)]`（デッキ側をソリッドで受ける）
    - `Surfaces2` は `[profileX[0]+",FL"]`
    - `Sf1DimensionType=1541` + `Sf1EndElements` で端部要素（例：DL05P等）を与える
    を踏襲する。
- **方針C（FR13/FR09 等の「1501（FL×FL）」系）**
  - Side FR13.CDP ↔ Deck.D FR13P、Side FR09.CDP ↔ Deck.D FR09P は参考で 1501（250,15）系。FR配列に含めて一括配置可能。

### パラメータ依存関係
- **共通（BracketParam の骨格）**
  - `DefinitionType=1`
  - `BracketName`: 「部材名.接続先.～」規則（例：`HK.Casing.Wall.Fore.DL05.Deck.DP` 等）
  - `MaterialName="SS400"`
  - `UseSideSheetForPlane=False`
  - `RevSf1/RevSf2/RevSf3` は基本 False（参考に準拠、向きが必要な箇所のみ変更）
- **Thickness（板厚）**
  - 参考スクリプトでは概ね以下に収束：
    - ブラケット板厚 8mm級（`7.9999...`）: 小形L/FB（ProfileParamsに t=7 付近）に対応する箇所（DL01/DL03/DL04/DL05 など）
    - ブラケット板厚 10mm級（`9.9999...`）: Side FR の 1505 ブラケット（FR材が t=9 付近、ProfileParamsの板厚9に対応）
    - ブラケット板厚 12mm: DL02 のフランジ付き（ProfileType=1201, t=12）に対応する箇所
  - したがって元スクリプトでも、**接続先Profileの板厚に追従**させるのが整合的（DL02=12、Side FR系=10、その他=8）。
- **BracketType**
  - **1501**: FL×FL（Profile同士）で三角ブラケットを作る基本型。DL系の「壁縦通材 ↔ 甲板縦通材」接続の主力。
  - **1505**: `BracketParams=["200"]` を伴う型で、`Surfaces1` に PLS（多くは deck solid）を与え、`Sf1DimensionType=1541` + `Sf1EndElements` で端部拘束する（Side FR縦通材 ↔ Deck の典型）。
- **Scallop / Flange**
  - `Scallop1Type=1801`
    - パラメータ `"0"` または `"50"` が混在。参考では DL02 や FR系などで `"50"` が入りやすい。
  - **DL02（ProfileType=1201絡み）**は参考で `FlangeType=262` を付与し `FlangeParams` を設定しているため、元スクリプトでも **DL02のブラケットのみ FlangeType/Params を必須**とするのが安全。
- **寸法（Sf1DimensionType / Sf2DimensionType）**
  - 1501の基本：`Sf1DimensionType=1531`, `Sf1DimensonParams=["200","15"]`（またはFR系で `["250","15"]` / `["300","15"]`）
  - 1505の基本：`Sf1DimensionType=1541`, `Sf1DimensonParams=["0","100"]`（端基準）、`Sf2DimensionType=1531`, `Sf2DimensonParams=["200","15"]`
  - 1505では **`Sf1EndElements` を必ず与える**（参考では `["PLS",... , profile1[0]]` のように、端決め要素を指定）。

### 注意事項
- **Surfaces の与え方が2系統ある**
  1. `["element,FL"]` のように「要素+面名」で直接指定（主に Profile 同士の FL/WF/WB）
  2. `["PLS","False","False", nx,ny,nz, element]` のような **PLS形式**で平面（法線ベクトル）と要素を与える  
     → 参考では DL02 系・1505 系で多用。元スクリプトでも同条件の箇所はこの形式を踏襲しないと、面解釈違いで失敗しやすい。
- **Side FR × Deck のブラケットは 1505 を優先**
  - 参考スクリプトでは FR07/08/10/11/12/14/15 に対して一貫して 1505 を採用し、`Surfaces1=PLS + solid(deck)`、`Surfaces2=profile,FL`、`Sf1EndElements` で端位置を拘束している。  
  - 元スクリプト側でも Deck.D のソリッドは `solid2` として存在するため、同様に `solid2` を使って受けるのが最も再現性が高い。
- **DL02 は例外的にフランジ付きブラケット（FlangeType=262）になりやすい**
  - 参考では Fore DL02 / Aft DL02 のブラケットが、他DL（1501の単純FL×FL）と異なり、PLS指定 + FlangeType を伴う。  
  - 元スクリプトでも DL02 の部材は `ProfileType=1201` が多く、同様の例外扱いが必要。
- **BaseElement は「壁側縦通材」または「側壁FR材」を選ぶ**
  - 参考では「Wall.Fore.*」「Wall.Aft.*」「Wall.Side.FR*」側が BaseElement になっているケースが多い。元スクリプトも同方針に合わせると命名・寸法系の整合が取りやすい。
- **分割（BodyDivideByCurves）を行う前提がある場合がある**
  - 参考スクリプトでは作成したブラケットを `BodyDivideByCurves` で切っている箇所がある（同一スケッチ平面で切断して見た目/干渉を調整）。  
  - 元スクリプトにこの工程がない場合、干渉や余長が残る可能性があるため、最終的に「ブラケットも同じカーブで分割する」運用を想定しておく。
