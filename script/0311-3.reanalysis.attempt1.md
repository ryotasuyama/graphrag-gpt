#### 失敗候補の修正結果
- **除外対象**
  - 旧 #22  
    `profile40[0]+",WB"` × `profile41[0]+",FL"`  
    `BracketName="HK.Casing.Deck.A.FR13P"`
- **除外理由**
  - 実行時に COM 例外が発生しており、`Deck.A.DL04P` の `WB` 面と `Deck.A.FR13P` の `FL` 面による 1501 接続が有効でない可能性が高い
  - `profile41` は `profile40` を端部要素に持つ `FR13P` だが、Deck.D の成功例と異なり、`WF + Sf1EndElements` 付きの参照スクリプト型ではない
  - 参考スクリプトにある成功パターンは **Deck.D の FR13P / FR09P に対するWFまたはWBの使い分け**であり、Deck.A に機械的に横展開すると失敗する
- **代替方針**
  - 成功済みの Deck.D 系 FR ブラケットを優先維持
  - 30個以上の候補要件は、Fore整合済みの Deck.A/B/C の DL 系ブラケット候補で補完する

### ループ構造方針
- **Wall.Fore × Deck(F/FP/P)** を DL ごとにループ
  - 対象DL: `DL00, DL01, DL03, DL04, DL05`
  - 許可ペア:
    - `Wall.Fore.DLxx.*` → `Deck.X.DLxx.F` / `Deck.X.DLxx.FP`
    - 実データ上、`DL04P`, `DL05P` も Fore 系で使われているが、少なくとも **A/AP 系でないこと**を確認して採用する
  - 実際の安全候補:
    - Deck.A/B/C の `DL00.F`, `DL01.FP`, `DL03.FP`, `DL05P`
    - Deck.C/D の `DL04P`
    - Deck.D の `DL03P` も成功済み実績あり

- **Wall.Aft × Deck(A/AP)** を DL ごとにループ**
  - 許可ペア:
    - `Wall.Aft.DLxx.*` → `Deck.X.DLxx.A` / `Deck.X.DLxx.AP`
  - 現スクリプト上で明確に成立している安全候補:
    - `Deck.D.DL00.A`
    - `Deck.D.DL01.AP`
  - `Deck.A/B/C` 側には A/AP 明示プロファイルが不足しているため、**それらとのAftブラケットは候補除外**

- **Wall.Side(FR) × Deck面 or Deck FRプロファイル** を FR ごとにループ
  - `FR07, 08, 10, 11, 12, 14, 15` は参考スクリプト同様 **BracketType=1505**
  - `FR09, FR13` は Deck側FR材との **BracketType=1501**
  - Side Wall には Fore/Aft 制約を適用しないが、**接続先 Deck プロファイルまたは Deck ソリッド面が実在すること**を必須条件とする

- **FB/PL系のDL02接続** は通常ループから分離
  - `profile59/profile19` と `profile18` を組み合わせる
  - `Surfaces1/2` は `PLS` 配列形式
  - Flange付きブラケットを使用

- **Deck FR 系ブラケットは成功実績ベースで限定**
  - 維持してよい成功パターン:
    - Deck.D FR13P
    - Deck.D FR09P
    - Deck.D FR13C
  - **Deck.A FR13P のような未確立のWB接続横展開は除外**
  - Deck.A/B/C の FR09P / FR13P は、再度採用する場合でも別途の面種別検証が必要

### パラメータ依存関係
- **BracketType の選択**
  - **1501**
    - 壁縦通材 `CDP` と Deck縦通材 `F / FP / P / A / AP` の **FL–FL 接続**
    - Deck FR材と Wall.Side FR材の **FL–FL 接続**
    - Deck梁の `WF/WB` 面と FR材 `FL` の接続
    - DL02 の **PLS–PLS 接続**
  - **1505**
    - Wall.Side FR材と Deck板ソリッド面の接続
    - 参考スクリプト準拠で
      - `Surfaces1=["PLS",...,solid2]`
      - `Surfaces2=[profileX[0]+",FL"]`
      - `Sf1EndElements=["PLS",...,profile17[0]]`

- **Thickness**
  - 一般DLブラケット（125x75系）: **8t**
  - DL02 FB/PL系: **12t**
  - Side FR のDeck板直付け（1505）: **10t**
  - FR09/FR13 系との接続: **9t 前後**
  - 失敗した Deck.A FR13P 候補も 9t だったが、**厚みより面指定/接続方式が失敗要因の可能性が高い**

- **Sf1 / Sf2 寸法**
  - FL–FL の標準:
    - `Sf1DimensionType=1531`
    - `Sf2DimensionType=1531`
    - `Sf1DimensonParams=["200","15"]`
    - `Sf2DimensonParams=["200","15"]`
  - FR13 / FR09 の大きめ材:
    - `["250","15"]` または `["300","15"]`
  - 1505 板面接続:
    - `Sf1DimensionType=1541`
    - `Sf1DimensonParams=["0","100"]`
    - `Sf1EndElements` 必須
    - `Sf2DimensionType=1531`
    - `Sf2DimensonParams=["200","15"]`

- **FlangeType / FlangeParams**
  - DL02 の PLS型で必要
  - 参考値:
    - `FlangeType=262`
    - `FlangeParams=["100","30","29.999999999999996","30","30","1"]`
  - Deck.D の FR09/FR13 の WF接続では
    - `FlangeType=262`
    - `FlangeParams=["75","30","29.999999999999996","30","50","1"]`
    - `Sf1EndElements` を併用する成功例あり

- **BaseElement**
  - Wall.Fore / Wall.Aft 接続 → 壁側 `CDP`
  - Wall.Side × Deck面接続 → Side側 `CDP`
  - Deck FR × Wall.Side FR 接続 → 成功例では Deck側 `FR` を `BaseElement`
  - 失敗した Deck.A FR13P 候補も Deck側を `BaseElement` にしていたが、これ自体は否定要因ではなく、**面ペアの不整合が主因と判断**

### 注意事項
- **Fore/Aft 整合ルールは必須**
  - `Wall.Fore.DLxx.*` → `Deck.X.DLxx.F / FP` 系のみ許可
  - `Wall.Aft.DLxx.*` → `Deck.X.DLxx.A / AP` 系のみ許可
  - `Wall.Aft` に対して `Deck.A/B/C` の `F/FP/P` を接続候補に入れてはいけない
  - 今回の再分析ではこのルールを再適用し、**Aft側のDeck.C/B/Aミラー候補は除外維持**

- **失敗した Deck.A FR13P 候補は除外**
  - `profile40[0]+",WB"` × `profile41[0]+",FL"` の 1501 は COM例外発生
  - 参考スクリプトに同等の成功根拠がないため、修正採用ではなく **候補除外** とする

- **成功済み bracket 1〜21 は維持**
  - 再分析では成功実績を最優先し、既存成功候補は変更しない
  - 修正対象は失敗候補のみ

- **候補数要件は維持**
  - 旧 #22 を除外しても、Fore整合済みの Deck.A/B/C の DL 接続候補を追加管理することで **30件以上** を維持可能
  - 実装優先度としては
    1. 成功済み 1〜21 を固定
    2. 成功実績のある Deck.D FR 系（#22〜24相当の代替）を追加
    3. Fore整合済み Deck.A/B/C の DL 系を追加
  - この順で採用すれば、安全性と件数要件を両立できる

- **PLS形式を使うケースを落とさない**
  - DL02 は `FL` 指定ではなく `PLS` 指定が必要
  - `profile18`, `profile19`, `profile59` のような FB/PL 系は通常の FL–FL と分けて扱う

- **1505 の Sf1EndElements を必ず継承**
  - Side系 1505 では `Sf1EndElements` の省略は避ける
  - 参考スクリプト準拠の終端拘束が必要

- **WF / WB / FL の使い分けは成功実績ベースで制限**
  - Deck.D FR13P / FR09P は `WF` 側成功例あり
  - Deck.D FR13C は `WB` 側成功例あり
  - Deck.A FR13P の `WB` は失敗したため、**同型横展開禁止**
  - Deck.A/B/C の FR ブラケットは、別途の検証が終わるまで「分析候補」止まりとするのが安全

- **推奨の実装優先順位**
  1. Side FR の 1505 ブラケット全件
  2. Wall.Fore / Wall.Aft の Deck.D 直結ブラケット
  3. Deck.D の FR09/FR13 系成功パターン
  4. Fore整合済み Deck.A/B/C の DL 拡張候補
  5. DL02 の PLS型
