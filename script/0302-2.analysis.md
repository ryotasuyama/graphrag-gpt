### ループ構造方針
元スクリプトは自動生成コード的に“逐次命名（profile1〜）”されており、**部材グルーピングがコード上の連番でしか表現されていません**。そのため、確実に 50個以上を漏れなく付与するには、以下のループ方針が適します。

- **方針A（推奨）: ブラケット対象 Profile を明示リスト化してループ**
  - 例: `side_cdp_profiles = [profile16[0], profile12[0], profile44[0], profile52[0], profile113[0], profile3[0], profile111[0] ...]`
  - 各 profile について、接続先の deck solid（solid2/solid1/solid3/solid4）を割り当てて `BracketType=1505` を作る
  - mirror 側は `MirrorCopy` 済み要素が存在するため、同様に `mirror_copiedXX[0]` を対象に追加ループ

- **方針B: ルールベースで“Wall縦材×Deck縦材”を片端ずつ 1501 で接続**
  - 例: Fore縦材（profile62, profile100, profile32, profile117...）と Deck縦材（profile61, profile86, profile17, profile35...）のペアを定義し、`Surfaces1=[wallProfile+",FL"]`, `Surfaces2=[deckProfile+",FL"]`

- **方針C: 片舷作成→MirrorCopy によるブラケット自体のミラー複製**
  - 参考スクリプトはブラケット自体の MirrorCopy はしていないが、元スクリプトでは MirrorCopy が大量に使われているため、**ブラケットもミラー複製して数量を確保**するのが簡単。
  - ただし、MirrorCopy は“要素のソリッド/シート形状”を複製するので、**ベース要素に依存するブラケット生成より後**にミラーするか、先に対象部材をミラーしてから反対舷ブラケットを作るかを統一する必要がある。

### パラメータ依存関係
参考スクリプトから読み取れる、ブラケット生成パラメータの基本ルールは以下です。

- **共通**
  - `bracketPram.DefinitionType = 1`
  - `MaterialName = "SS400"`
  - `UseSideSheetForPlane = False`
  - `RevSf1/RevSf2/RevSf3` は通常 False（ただし面法線が逆のときのみ True を使う）

- **BracketType**
  - **1501**: 基本ブラケット（FL×FL、あるいは WF/WB×FL のような“部材面同士”）
    - `Surfaces1=[profileX[0]+",FL"]` のような直接面指定が多い
    - `Sf1DimensionType=1531`（例: `["200","15"]` や `["250","15"]`）
  - **1505**: “面（PLS）×部材（FL）”のような取り合いで使用
    - `Surfaces1=["PLS",... , solidX]` で**ソリッド（Deck厚肉等）を参照面**にする例が多い
    - `BracketParams=["200"]` が付く（参考: bracketPram4/7/15/17/19/21/22）
    - `Sf1DimensionType=1541` + `Sf1EndElements` を組み合わせる頻度が高い（例: `Sf1DimensonParams=["0","100"]`）

- **Thickness（板厚）**
  - 参考では接続相手のプロファイル板厚に近い値が選ばれている
    - 例: Side縦材ブラケット（1505）で `Thickness="9.999..."`（≈10mm）
    - Fore/Aftの小型縦材同士（1501）で `Thickness="7.999..."`（≈8mm）
    - DL02 系のフランジ付き（1201）相手は `Thickness="12"` が使われる
  - 元スクリプトに合わせた推奨:
    - 一般: 8mm（1002/1003系の薄板縦材）
    - FR系Side縦材に対して: 10mm
    - DL02（ProfileType=1201）近傍: 12mm

- **スカラップ**
  - `Scallop1Type=1801`
  - `Scallop1Params=["0"]` または `["50"]`
  - 1501で多用される（端部干渉回避）

- **フランジ（ブラケットに折り返し等）**
  - 参考では一部 1501 で `FlangeType=262` を付与している（例: bracketPram2/3/5/12）
  - 元スクリプトで 50個以上を優先するなら、まずは **フランジ無し（標準）**で安定化し、必要箇所（DL02など）だけフランジを適用するのが合理的。

- **End1/End2 の指定**
  - ブラケットパラメータ自体には End1/End2 は基本なく、代わりに
    - `Sf1EndElements` / `Sf2EndElements`
    - `Sf1DimensionType=1541` のような端基準寸法
  - 参考で `Sf1EndElements` が登場するのは主に 1505 または 1541 指定のケース。

### 注意事項
- **元のスクリプトにはブラケット生成が存在しないため**、「50個以上ブラケットを追加」の編集は、必然的に `CreateBracketParam()` / `CreateBracket()` ブロックを新設する設計になります（参考スクリプトのAPI構造を踏襲する必要がある）。
- **Surfaces1/Surfaces2 は“面ペア”で成立**するため、接続相手の要素が
  - Profile の FL/WF/WB/FR 面を持つか
  - あるいは `PLS` 形式でソリッド/シートを参照可能か
  を必ず揃える必要があります。
- **参照面の向き（Mold, RevSf1/2）**は失敗要因になりやすいので、最初は参考と同じ `Mold="+"`（または必要箇所のみ `"-"`）で統一し、干渉や裏返りが出た箇所だけ `RevSf*` を調整する方針が安全です。
- 参考スクリプトではブラケット作成後に `BodyDivideByCurves` で形状分割している例が多いですが、元スクリプトには分割基準スケッチ（skt_pl4 等）が存在するため、**同様にブラケットにも分割処理を適用する設計**が望ましい（ただし「50個以上」という数量要件のみなら、分割は後回しでも成立）。
- **左右（PL,Y）対称**が大量にあるため、ブラケットも
  - 片舷に集中配置 → ブラケット要素を MirrorCopy  
  または
  - 片舷の対象部材（profile）と mirror_copied（相手）に対して同一ロジックで再生成  
  のどちらかに統一すること（混在すると参照関係が崩れやすい）。
