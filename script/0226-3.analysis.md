## ブラケット配置分析

### 対象部材とループ変数
- **壁（Wall Sheet）**
  - `extrude_sheet8`: HK.Casing.Wall.Fore（前壁）
  - `extrude_sheet3`: HK.Casing.Wall.Aft（後壁）
  - `extrude_sheet6`: HK.Casing.Wall.SideP（舷側P）
  - `extrude_sheet9`: HK.Casing.Wall.SideS（舷側S）
- **甲板（Deck Sheet）**
  - `extrude_sheet1`: HK.Casing.Deck.B
  - `extrude_sheet2`: HK.Casing.Deck.A
  - `extrude_sheet4`: HK.Casing.Deck.C
  - `extrude_sheet5`: HK.Casing.Deck.D
  - `extrude_sheet7`: HK.General.Deck.UpperDeck
- **ブラケット接続の母材になり得る Profile（BaseElement候補）**
  - 壁側の縦通・小骨的 profile（`profile62`/`profile94`/`profile100`/`profile105`/`profile75`/`profile72`/`profile87`/`profile121` 等）：主に `ProfileType=1002`
  - Deck側の縦通・梁的 profile（`profile61`/`profile60`/`profile23`/`profile27`/`profile64`/`profile19`/`profile18` 等）：`1002`/`1003`/`1201`
- **本スクリプト（元のスクリプト）内の「ブラケット配置対象となる代表的な交差」**
  - Fore壁（`extrude_sheet8`）× Deck（`extrude_sheet5`/`extrude_sheet4`/`extrude_sheet2`/`extrude_sheet1`/`extrude_sheet7`）付近に、DL00/DL01/DL02/DL03/DL04/DL05 の profile群が存在
  - Aft壁（`extrude_sheet3`）× Deck群にも同様に DL00〜DL05 の profile群が存在
  - Side壁（`extrude_sheet6`）× Deck.D（`solid2` 相当のDeckソリッドが作られている）に FR07/08/10/11/12/14/15 等の Side profile が存在

> 参考スクリプトでは、ブラケットは「壁側縦材（CDP等）と、甲板側縦材（FP/AP/Deck側材）のフランジ面（FL/WF/WB等）の面ペア」で作られ、必要に応じて `PLS`（参照平面/疑似面）でソリッド面を指定しています。  
> 元のスクリプトにはブラケット生成が一切ないため、追加仕様として「代表的な交差点（Wall↔Deck、Side↔Deck）にブラケットを入れる」方針を、参考スクリプトの面指定・寸法ロジックに合わせて設計する必要があります。

### ループ構造方針
参考スクリプトは**明示ループ無し（逐次作成）**ですが、仕様として整理すると以下の2グループでループ化できます。

- **グループA：Wall（Fore/Aft）× Deck のDL系列ブラケット（BracketType=1501）**
  - 対象：`DL01/DL03/DL04/DL05` は「FL↔FL」の直指定が多い
  - ループ単位例：  
    - `[(wall_profile, deck_profile, thickness, mold)]` のタプル列を作り、順に `CreateBracketParam` → `CreateBracket`
- **グループB：Side（FRxx）× Deckソリッド（BracketType=1505）**
  - 対象：`FR07/08/10/11/12/14/15` 等（参考では1505を多用）
  - ループ単位例：  
    - `[(side_profile, deck_solid, sf1_end_element_profile)]` のタプル列を作る  
    - `Surfaces1` を `["PLS",... , deck_solid]`、`Surfaces2` を `[side_profile+",FL"]` として作成

> 重要：参考スクリプトでは 1505 のとき `BracketParams=["200"]` が必須で、さらに `Sf1EndElements` を入れて「端部拘束（終端要素）」を明示し、寸法タイプを 1541 にしています。これを踏襲すると配置の破綻が起きにくいです。

### パラメータ依存関係
参考スクリプトから読み取れる「ブラケットパラメータ決定ロジック（実質的な規約）」は以下です。

- **BracketType**
  - **1501**：一般的なブラケット（Wall↔Deck の FL↔FL、WF/WB↔FL）
  - **1505**：Deckソリッド（面）に取り付けるタイプ（Side FRxx ↔ Deck）で使用。`BracketParams=["200"]` が付随。
- **Thickness**
  - DL01/DL03/DL04/DL05 の多く：`7.999...`（≒8mm）
  - DL02（フランジ付き：ProfileType=1201絡み）の多く：`12mm`
  - Side FRxx 1505：`9.999...`（≒10mm）
  - Deck FR09P/FR13P のWF接続など：`8.999...`（≒9mm）
- **Mold（向き）**
  - Fore壁系：`Mold="+"` が多い（ただしDL02などで `"-"` もあり）
  - Aft壁系：ケースにより `"+"` と `"-"` が混在（DL02は `"-"` が多い）
  - Side系1505：`"+"`
- **Surfaces1 / Surfaces2 の指定規約**
  - **Profile同士**：`[element_id + ",FL"]` や `",WF"`, `",WB"` を使用
  - **ソリッド面（Deckなど）**を使うとき：`["PLS","False","False", nx, ny, nz, solid_or_profile]` 形式（法線ベクトルを含める）
- **寸法指定（Sf1DimensionType / Sf2DimensionType）**
  - 標準：`1531` with `["200","15"]` または `["250","15"]`、DL02は `["800","15"]`
  - ソリッド面基準で端部拘束を入れるとき：`1541` with `["0","100"]` や `["0","80"]`
- **Sf1EndElements / Sf2EndElements**
  - 1505のDeck面基準ブラケットで頻出：`Sf1EndElements=["PLS",... , profile1[0]]` のように「終端を決める要素」を入れている
  - 1501でも、WF面側に対して `Sf1EndElements=[profile4[1]+",FR"]` のように端部要素を追加するケースがある

### 注意事項
- **ブラケットは Surfaces1 / Surfaces2 の“面ペア”定義が必須**  
  - Profile要素は `",FL"`, `",WF"`, `",WB"` のように面タイプを文字列で指定しており、この指定を誤ると生成失敗しやすい。
- **ソリッド面に取り付ける場合は PLS 指定が実質必須**  
  - 参考ではDeckソリッドに対して `Surfaces1=["PLS",..., solid1]` の形式を用い、さらに `Sf1EndElements` で終端を拘束している。元スクリプトでも Side×Deck はこの方式を踏襲すべき。
- **DL02などフランジ付き（ProfileType=1201）絡みは、フランジ設定が入るパターンがある**  
  - 参考では FlangeType=262/FlangeParams をブラケット側に設定しているケースがあり（#2, #5）、DL02相当の接続ではこの有無が重要。
- **寸法タイプの使い分け**  
  - 1531（長さ/オフセット系）と1541（端部基準系）を混在使用しており、特に1505は1541に寄せている。ここを揃えると再現性が上がる。
- **後工程の BodyDivideByCurves はブラケット形状トリムに利用**  
  - 参考では多くのブラケットに対して `BodyDivideByCurves(..., bracketX, [skt_pl4], ...)` を実行しており、完成形を整形している。  
  - 元スクリプトで「ブラケットをつける」だけが目的でも、同様のトリムが必要になる可能性が高い（干渉回避・端部整理）。ただし仕様としては「ブラケット作成→必要なら分割」をセットで設計しておく。
