## 総合実装計画

### 計画1: `0310-1.py` を上書きしない

**問題**: リトライのたびに `0310-1.py` が上書きされ、初回生成（attempt 0）の内容が失われる。

**修正内容**:
- `save_script_file`: `attempt == 0` のときだけ `output_path` に書き込み、`attempt > 0` では `.attemptN.py` にのみ保存
- `process_generation_loop`: リトライ時の `run_script` を `.attemptN.py` のパスで実行

**修正箇所**: `query.py` の `save_script_file`（line 956-959）、`process_generation_loop`（line 1173-1174）

---

### 計画2: `single_bracket_mode` を廃止する

**問題**: `attempt >= 2` で「1個だけ生成」モードが発動し、ユーザの要件（20個以上）を放棄する。attempt 0→1 で 13→22 個と進捗していたのに、attempt 2 で全て捨てて1個にしている。

**修正内容**:
- `query.py:1118` の `"single_bracket_mode": attempt >= 2` を削除
- `prompts/loader.py:218-225` の「単一ブラケットモード」ブロックを削除

**修正箇所**: `query.py`（line 1118）、`prompts/loader.py`（line 218-225）

---

### 計画3: 成功したブラケットを保持し、失敗箇所以降のみ再生成する

**問題**: リトライ時に `merge_bracket_section(original_code, ...)` でブラケットセクション全体を差し替えている。成功した bracket 1〜22 も巻き添えで再生成され、結果が不安定になる。

**修正内容**:
- `extract_bracket_failure_context` の `error_line_info` からエラー行番号を取得
- 現在の `current_code` のエラー行**より前**（成功部分）を保持する
- LLM にはエラー行以降のブラケットだけを再生成させる
- `merge_bracket_section` の代わりに、成功部分 + 再生成部分を結合する新関数を作る

**具体的な流れ**:
```
current_code の line 1〜(エラー行-1) を保持
 ↓
LLM にエラー行以降のブラケット定義だけを再生成させる
（「bracket N まで成功。bracket N+1 以降を修正して」と指示）
 ↓
保持部分 + 再生成部分を結合
```

**修正箇所**: `query.py` の `process_generation_loop` リトライ部分（line 1092-1137）、新関数追加

---

### 計画4: エラーコンテキストに「どこまで成功したか」を含める

**問題**: `error_ctx` に失敗情報しかなく、LLM が「どこまで動いたか」を知らない。

**修正内容**:
- エラー行番号から、成功したブラケット数を算出する（`current_code` から `CreateBracket` の呼び出しをカウントし、エラー行より前のものを「成功」とする）
- `error_ctx` に `successful_bracket_count` と `failed_bracket_name` を追加
- プロンプトに「bracket 1〜N は成功済み。bracket N+1 のパラメータを修正し、残りも生成してください」と明示

**修正箇所**: `query.py` の `process_generation_loop`（line 1106-1118）、`prompts/loader.py` の `_build_error_context_block`

---

### 計画5: 実行成功後にブラケット数を検証する

**問題**: 実行成功判定が「ランタイムエラーなし」のみ。1個のブラケットでも「成功」扱いになる。

**修正内容**:
- 実行成功後、生成されたスクリプト内の `CreateBracket` 呼び出し数をカウント
- ユーザの指示から要求ブラケット数を抽出（「20個以上」→ 20）
- カウントが要求数に満たない場合、警告を表示してリトライを継続する
- 要求数が明示されていない場合は検証をスキップ（従来通り）

**修正箇所**: `query.py` の `process_generation_loop`（line 1176-1180 の成功判定部分）

---

### 実装順序

| 順序 | 計画 | 理由 |
